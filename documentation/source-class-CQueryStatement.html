  50: /**
  51:  *  Query statement.
  52:  *
  53:  * This class implements a query statement, such as those that populate the
  54:  * {@link CQuery query} class. This class concentrates on building and managing these
  55:  * statements, which are structured as follows:
  56:  *
  57:  * <ul>
  58:  *  <li><i>{@link kAPI_QUERY_SUBJECT kAPI_QUERY_SUBJECT}</i>: The query subject. It refers
  59:  *      to the object element that we are considering in the query.
  60:  *  <li><i>{@link kAPI_QUERY_OPERATOR kAPI_QUERY_OPERATOR}</i>: The query predicate. This
  61:  *      element represents the predicate or comparaison operator, it can take the following
  62:  *      values:
  63:  *   <ul>
  64:  *      <li><i>{@link kOPERATOR_DISABLED kOPERATOR_DISABLED}</i>: Disabled, it means that
  65:  *          the current statement is disabled.
  66:  *      <li><i>{@link kOPERATOR_EQUAL kOPERATOR_EQUAL}</i>: Equality (=), this operator
  67:  *          implies that the statement has also an {@link kAPI_QUERY_DATA object}.
  68:  *      <li><i>{@link kOPERATOR_EQUAL_NOT kOPERATOR_EQUAL_NOT}</i>: Inequality
  69:  *          (!=), negates the {@link kOPERATOR_EQUAL kOPERATOR_EQUAL}
  70:  *          operator; this operator implies that the statement has also an
  71:  *          {@link kAPI_QUERY_DATA object}.
  72:  *      <li><i>{@link kOPERATOR_LIKE kOPERATOR_LIKE}</i>: Like, it is an accent and
  73:  *          case insensitive equality filter, this operator implies that the statement has
  74:  *          also an {@link kAPI_QUERY_DATA object}.
  75:  *      <li><i>{@link kOPERATOR_LIKE_NOT kOPERATOR_LIKE_NOT}</i>: The negation of
  76:  *          the {@link kOPERATOR_LIKE LIKE} operator, this operator implies that the
  77:  *          statement has also an {@link kAPI_QUERY_DATA object}.
  78:  *      <li><i>{@link kOPERATOR_PREFIX kOPERATOR_PREFIX}</i>: Starts with, or prefix
  79:  *          match, this operator implies that the statement has also an
  80:  *          {@link kAPI_QUERY_DATA object}.
  81:  *      <li><i>{@link kOPERATOR_CONTAINS kOPERATOR_CONTAINS}</i>: Contains, selects
  82:  *          all elements that contain the match string, this operator implies that the
  83:  *          statement has also an {@link kAPI_QUERY_DATA object}.
  84:  *      <li><i>{@link kOPERATOR_SUFFIX kOPERATOR_SUFFIX}</i>: Ends with, or suffix
  85:  *          match, this operator implies that the statement has also an
  86:  *          {@link kAPI_QUERY_DATA object}.
  87:  *      <li><i>{@link kOPERATOR_REGEX kOPERATOR_REGEX}</i>: Regular expression, this
  88:  *          operator implies that the statement has also an {@link kAPI_QUERY_DATA object}.
  89:  *      <li><i>{@link kOPERATOR_LESS kOPERATOR_LESS}</i>: Smaller than (\<), this operator
  90:  *          implies that the statement has also an {@link kAPI_QUERY_DATA object}.
  91:  *      <li><i>{@link kOPERATOR_LESS_EQUAL kOPERATOR_LESS_EQUAL}</i>: Smaller than
  92:  *          or equal (\<=), this operator implies that the statement has also an
  93:  *          {@link kAPI_QUERY_DATA object}.
  94:  *      <li><i>{@link kOPERATOR_GREAT kOPERATOR_GREAT}</i>: Greater than (\>), this operator
  95:  *          implies that the statement has also an {@link kAPI_QUERY_DATA object}.
  96:  *      <li><i>{@link kOPERATOR_GREAT_EQUAL kOPERATOR_GREAT_EQUAL}</i>: Greater than
  97:  *          or equal (\>=), this operator implies that the statement has also an
  98:  *          {@link kAPI_QUERY_DATA object}.
  99:  *      <li><i>{@link kOPERATOR_IRANGE kOPERATOR_IRANGE}</i>: Range inclusive,
 100:  *          matches \>= value \<=; this operator implies that the statement has also an
 101:  *          {@link kAPI_QUERY_DATA object} which should contain the two range values.
 102:  *      <li><i>{@link kOPERATOR_ERANGE kOPERATOR_ERANGE}</i>: Range exclusive,
 103:  *          matches \> value \<, this operator implies that the statement has also an
 104:  *          {@link kAPI_QUERY_DATA object} which should contain the two range values.
 105:  *      <li><i>{@link kOPERATOR_NULL kOPERATOR_NULL}</i>: Is <i>NULL</i> or element
 106:  *          is missing.
 107:  *      <li><i>{@link kOPERATOR_NOT_NULL kOPERATOR_NOT_NULL}</i>:Not <i>NULL</i> or
 108:  *          element exists.
 109:  *      <li><i>{@link kOPERATOR_IN kOPERATOR_IN}</i>: In, or belongs to set, this operator
 110:  *          implies that the statement has also an {@link kAPI_QUERY_DATA object} that will
 111:  *          contain the list of choices.
 112:  *      <li><i>{@link kOPERATOR_NI kOPERATOR_NI}</i>: Not in, the negation of
 113:  *          {@link kOPERATOR_IN kOPERATOR_IN}, this operator implies that the statement has
 114:  *          also an {@link kAPI_QUERY_DATA object} which contains the list of choices.
 115:  *      <li><i>{@link kOPERATOR_ALL kOPERATOR_ALL}</i>: All, or match the full set, this
 116:  *          operator implies that the statement has also an {@link kAPI_QUERY_DATA object}
 117:  *          which will contain the set.
 118:  *      <li><i>{@link kOPERATOR_NALL kOPERATOR_NALL}</i>: Not all, the negation of
 119:  *          the {@link kOPERATOR_ALL kOPERATOR_ALL} operator, this operator implies that the
 120:  *          statement has also an {@link kAPI_QUERY_DATA object} which contains the set.
 121:  *      <li><i>{@link kOPERATOR_EX kOPERATOR_EX}</i>: Expression, indicates a
 122:  *          complex expression, this operator implies that the statement has also an
 123:  *          {@link kAPI_QUERY_DATA object} which will contain the expression.
 124:  *   </ul>
 125:  *  <li><i>{@link kAPI_QUERY_TYPE kAPI_QUERY_TYPE}</i>: The statement object type, or data
 126:  *      type of the {@link kAPI_QUERY_DATA kAPI_QUERY_DATA} element, if the latter is
 127:  *      required:
 128:  *   <ul>
 129:  *      <li><i>{@link kTYPE_STRING kTYPE_STRING}</i>: String, we assume
 130:  *          in UTF8 character set.
 131:  *      <li><i>{@link kTYPE_INT32 kTYPE_INT32}</i>: 32 bit signed integer.

 133:  *      <li><i>{@link kTYPE_FLOAT kTYPE_FLOAT}</i>: Floating point number.
 134:  *      <li><i>{@link kTYPE_DATE kTYPE_DATE}</i>: A date.
 135:  *      <li><i>{@link kTYPE_TIME kTYPE_TIME}</i>: A date and time.
 136:  *      <li><i>{@link kTYPE_STAMP kTYPE_STAMP}</i>: A native timestamp.
 137:  *      <li><i>{@link kTYPE_BOOLEAN kTYPE_BOOLEAN}</i>: An on/off switch.
 138:  *      <li><i>{@link kTYPE_BINARY kTYPE_BINARY}</i>: A binary string.
 139:  *   </ul>
 140:  *  <li><i>{@link kAPI_QUERY_DATA kAPI_QUERY_DATA}</i>: The statement object or test data.
 141:  * </ul>
 142:  *
 143:  * The main goal of this class is to provide an interface that may ease the construction of
