Class COntologyEdge

Ontology graph edge node.

This class implements an ontology graph edge node.

The class is derived from CGraphEdge CGraphEdge and implements the exact same functionality as COntologyNode COntologyNode, it adds two elements: the SubjectTerm() subject and ObjectTerm() object COntologyTerm terms which are linked with the Subject() subject and Object() object nodes.

Instances of this class represent predicate nodes and they hold the referenced nodes.

# _IndexTerms( Everyman\Neo4j\Client $theContainer )

Create node indexes.

Create node indexes.

This method will save node indexes after the node was _Commit() committed, it will perform the following selections:

  • kINDEX_EDGE_TERM kINDEX_EDGE_TERM: The Term() term kTAG_GID global identifier (RelationshipIndex). container, it must be a Everyman\Neo4j\Client instance.
  • kINDEX_EDGE_NAME kINDEX_EDGE_NAME: The Term() term COntologyTermObject::Name() names in all languages (RelationshipIndex).
  • kINDEX_EDGE_TERMS kINDEX_EDGE_TERMS: The node relations, this index records the relation terms, that is, the combination of the subject SubjectTerm() term, predicate Term() term and the object ObjectTerm() term, this can be used to retrieve existing relations.

The following index tags are set:

  • kTAG_GID kTAG_GID: The Term() term kTAG_GID global identifier.
  • kTAG_NAME kTAG_NAME: The Term() term kTAG_NAME names.
  • kTAG_EDGE_TERM kTAG_EDGE_TERM: The relationships between terms taken from the Node() node's kTAG_EDGE_TERM kTAG_EDGE_TERM property, formatted as a SUBJECT/PREDICATE/OBJECT string, in which each element is the kTAG_GID global COntologyTerm term identifier.
  • kTAG_NODE kTAG_NODE: The relationships between Node() nodes, expressed as a SUBJECT/PREDICATE/OBJECT string, in which the subject and object are the Node() node identifiers and the predicate is the kTAG_GID global Term() term identifier.
