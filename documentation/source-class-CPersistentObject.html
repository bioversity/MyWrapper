  31: /**
  32:  * Persistent unit objects ancestor.
  33:  *
  34:  * This class is the ancestor of all persistent objects in this library, it implements the
  35:  * common interfaces and workflow that persistent objects will use to be
  36:  * {@link __construct() instantiated}, {@link __construct() retrieved} or
  37:  * {@link Commit() stored} in object {@link CContainer containers}.
  38:  *
  39:  * This class interacts closely with {@link CContainer CContainer} derived classes so that
  40:  * concrete classes derived from this one need only to be concerned with their specific
  41:  * functionality, while persistent issues are handled foth by this class and by the
  42:  * concrete classes derived from {@link CContainer CContainer}.
  43:  *
  44:  * The class declares three main persistence operations whose workflow is as follows:
  45:  *
  46:  * <ul>
  47:  *  <li><i>{@link __construct() Create}</i>: The {@link __construct() constructor} can
  48:  *      instantiate an object from data, as its {@link CArrayObject ancestor}, a protected
  49:  *      {@link _Create() method} may be overloaded by derived classes to implement
  50:  *      specific functionality.
  51:  *  <li><i>{@link __construct() Retrieve}</i>: The {@link __construct() constructor} is
  52:  *      also used to retrieve objects stored in a {@link CContainer container}, this
  53:  *      operation follows these steps:
  54:  *   <ul>
  55:  *      <li><i>{@link _PrepareLoad() Prepare}</i>: This step is used to check parameters and
  56:  *          prepare the resources needed to locate and retrieve the object.
  57:  *      <li><i>{@link _Load() Find}</i>: In this step the object will be searched for
  58:  *          and retrieved from the {@link CContainer container}.
  59:  *   </ul>
  60:  *  <li><i>{@link Commit() Store}</i>: This operation will store the object in a
  61:  *      {@link CContainer container}, the followed steps are:
  62:  *   <ul>
  63:  *      <li><i>{@link _PrepareCommit() Prepare}</i>: This step is used to check parameters
  64:  *          and prepare the resources needed to save the object.
  65:  *      <li><i>{@link _Commit() Store}</i>: This operation will perform the actual
  66:  *          storage, this step should be delegated to the {@link CContainer container}.
  67:  *   </ul>
  68:  * </ul>
  69:  *
  70:  * This class implements the default behaviour, in derived classes you should overload these
  71:  * methods if necessary, without changing the public interface.
  72:  *
  73:  * This class extends its {@link CStatusObject ancestor} behaviour to record persistence
  74:  * status and accepts only instances derived from {@link CContainer CContainer} as
  75:  * containers.
  76:  *
  77:  *  @package    MyWrapper
  78:  *  @subpackage Persistence
  79:  */
  80: class CPersistentObject extends CStatusObject

  96:     /**
  97:      * Instantiate class.
  98:      *
  99:      * The constructor can be used to instantiate an empty object, instantiate an object
 100:      * from some content, or load an object from a container.
 101:      *
 102:      * The parameters to this method are:
 103:      *
 104:      * <ul>
 105:      *  <li><b>$theContainer</b>: This parameter represents either the <i>contents</i> of
 106:      *      the object, if the next parameter is missing, or the
 107:      *      <i>{@link CContainer container}</i> in which the object resides. If missing, we
 108:      *      assume we want to create an empty object.
 109:      *   <ul>
 110:      *      <li><i>NULL</i>: In this case it is assumed you want to instantiate an empty
 111:      *          object and the next parameter will be ignored.
 112:      *      <li><i>Array</i> or <i>ArrayObject</i>: In this case we assume the parameter
 113:      *          represents either the object contents, if the next parameter is <i>NULL</i>,
 114:      *          or the {@link CContainer container} in which the object is
 115:      *          {@link Commit() stored}, in which case we interpret the next parameter to be
 116:      *          the object's identifier.
 117:      *      <li><i>Other</i>: Any other type will raise an
 118:      *          {@link kERROR_UNSUPPORTED exception}, or it should be handled by derived
 119:      *          classes.
 120:      *   </ul>
 121:      *  <li><b>$theIdentifier</b>: This parameter represents the key or query that will be
 122:      *      used to locate the object in the container provided in the first parameter.
 123:      *   <ul>
 124:      *      <li><i>NULL</i>: This indicates that the first parameter represents the object
 125:      *          <i>contents</i>.
 126:      *      <li><i>other</i>: Any other type is considered as a key or query used to locate
 127:      *          the object in the container provided in the first parameter.
 128:      *   </ul>
 129:      *  <li><b>$theModifiers</b>: This bitfield parameter can be used to provide a series of
 130:      *      options to the object. The available options can be consulted in
 131:      *      {@link Flags.inc.php this} file. Most of these options are managed automatically
 132:      *      and should not be provided explicitly. This class handles one option that has
 133:      *      the following meaning:
 134:      *   <ul>
 135:      *      <li><i>{@link kFLAG_STATE_ENCODED kFLAG_STATE_ENCODED}</i>: If the option is on,
 136:      *          the object will be passed through a
 137:      *          {@link CContainer::UnserialiseObject() conversion} process that will convert
 138:      *          all {@link CDataType standard} data types into native container custom data
 139:      *          types {@link CContainer::_PrepareCommit() before}
 140:      *          {@link Commit() committing} the object; the opposite
 141:      *          {@link CDataType::SerialiseObject() conversion} process will take place
 142:      *          {@link CContainer::_FinishLoad() after} {@link __construct() loading} the
 143:      *          object from a container. It is a good idea to keep this flag ON, because the
 144:      *          converted object will be compatible with any {@link CContainer container}
 145:      *          and it will make data transfers from one database to the other easy.
 146:      *          This process is automatic and transparent.
 147:      *   </ul>
 148:      * </ul>
 149:      *
 150:      * Depending on the results of this method we set the {@link _IsCommitted() committed}
 151:      * status {@link kFLAG_STATE_COMMITTED flag} as follows:
 152:      *
 153:      * <ul>
 154:      *  <li><i>Empty object</i>: By omitting the first parameter we indicate that we want to
 155:      *      start from scratch, the object will not have its
 156:      *      {@link _IsCommitted() committed} {@link kFLAG_STATE_COMMITTED flag} set.
 157:      *  <li><i>Filled object</i>: In this case we provide in the <i>$theContainer</i>
 158:      *      parameter the object's <i>contents</i>. In this case we do not consider the
 159:      *      object as persistent, so the object will not have its
 160:      *      {@link _IsCommitted() committed} {@link kFLAG_STATE_COMMITTED flag} set.
 161:      *  <li><i>Selected object</i>: In this case we search the <i>$theContainer</i> with
 162:      *      the provided <i>$theIdentifier</i> as a key or query, and if:
 163:      *   <ul>
 164:      *      <li><i>Found</i>: We set the {@link _IsCommitted() committed}
 165:      *          {@link kFLAG_STATE_COMMITTED flag} on.
 166:      *      <li><i>Not found</i>: We instantiate an empty object and ignore the
 167:      *          {@link _IsCommitted() committed} {@link kFLAG_STATE_COMMITTED flag}.
 168:      *   </ul>
 169:      * </ul>
 170:      *
 171:      * The {@link _IsInited() inited} status {@link kFLAG_STATE_INITED flag} is not set by
 172:      * default, this will prevent the object from being {@link Commit() stored} and it is
 173:      * the responsibility of derived concrete instances to manage this status.
 174:      *
 175:      * This method takes advantage of a protected interface which should be overloaded by
 176:      * derived classes instead of overloading this method:
 177:      *
 178:      * <ul>
 179:      *  <li><i>Retrieve object</i>: If the second parameter was provided, it implies that
 180:      *      the object is to be retrieved from the first parameter which represents the
 181:      *      {@link CContainer container} in which the object contents are stored.
 182:      *   <ul>
 183:      *      <li><i>{@link _PrepareLoad() _PrepareLoad}()</i>: This method should check and
 184:      *          normalise the container and identifier. In this class we ensure that the
 185:      *          container is derived from {@link CContainer CContainer}.
 186:      *      <li><i>{@link _Load() _Load}()</i>: This method will delegate the
 187:      *          {@link CContainer container} the responsibility of locating and retrieving
 188:      *          the object using the provided identifier.
 189:      *   </ul>
 190:      *  <li><i>{@link _Create() _Create}()</i>: This method should instantiate
 191:      *      the object from the contents returned by {@link _Load() _Load} or
 192:      *      by the data provided in the first parameter in the case the second parameter
 193:      *      was omitted.
 194:      * </ul>
 195:      *
 196:      * Derived classes should overload the above interface rather than overloading this
 197:      * method and should do so only if necessary, because this workflow should be sufficient
 198:      * for persisting objects.
 199:      *
 200:      * @param mixed                 $theContainer       Persistent container.
 201:      * @param mixed                 $theIdentifier      Object identifier.
 202:      * @param bitfield              $theModifiers       Create modifiers.
 203:      *
 204:      * @access public
 205:      *
 206:      * @uses _PrepareLoad()
 207:      * @uses _Load()
 208:      * @uses _Create()
 209:      * @uses _IsCommitted()
 210:      * @uses _FinishLoad()
 211:      * @uses _FinishCreate()
 212:      */
 213:     public function __construct( $theContainer = NULL,
 214:                                  $theIdentifier = NULL,
 215:                                  $theModifiers = kFLAG_DEFAULT )
 216:     {
 217:         //
 218:         // Prepare create.
 219:         //
 220:         $this->_PrepareCreate( $theContainer, $theIdentifier, $theModifiers );
 221:         
 222:         //
 223:         // Provided identifier.
 224:         //
 225:         if( $theIdentifier !== NULL )
 226:         {
 227:             //
 228:             // Prepare load.
 229:             //
 230:             $this->_PrepareLoad( $theContainer, $theIdentifier, $theModifiers );
 231:             
 232:             //
 233:             // Find object.
 234:             //
 235:             $found = $this->_Load( $theContainer, $theIdentifier, $theModifiers );
 236:             
 237:             //
 238:             // Create object and set status.
 239:             //
 240:             $this->_IsCommitted( $this->_Create( $found ) );
 241:             
 242:             //
 243:             // Finish.
 244:             //
 245:             $this->_FinishLoad( $theContainer, $theIdentifier, $theModifiers );
 246:         }
 247:         
 248:         //
 249:         // Provided content.
 250:         //
 251:         else
 252:         {
 253:             //
 254:             // Load content.
 255:             //
 256:             $this->_Create( $theContainer );
 257:             
 258:             //
 259:             // Initialise object.
 260:             //
 261:             $this->_FinishCreate( $theContainer );
 262:         }

 280:     /**
 281:      * Commit the object into a container.
 282:      *
 283:      * This method should be used to commit the object to a container, the method accepts
 284:      * two parameters:
 285:      *
 286:      * <ul>
 287:      *  <li><b>$theContainer</b>: This parameter represents the <i>container</i> in which
 288:      *      the object is to be stored. By default we enforce concrete instances of
 289:      *      {@link CContainer CContainer}, derived classes may overload the protected
 290:      *      interface to initialise this value.
 291:      *  <li><b>$theIdentifier</b>: This parameter represents the value by which the current
 292:      *      object will be uniquely identified in the first parameter. By default this
 293:      *      value is required, derived classes may overload the protected interface to
 294:      *      initialise this value.
 295:      *  <li><b>$theModifiers</b>: This parameter represents the commit operation options, by
 296:      *      default we assume you want to {@link kFLAG_PERSIST_REPLACE replace} an object,
 297:      *      but you may change this value if you want to perform specific commit operations:
 298:      *   <ul>
 299:      *      <li><i>{@link kFLAG_PERSIST_INSERT kFLAG_PERSIST_INSERT}</i>: The provided
 300:      *          object will be inserted in the container, it is assumed that no other
 301:      *          element in the container shares the same identifier, in that case the
 302:      *          container {@link CContainer::Commit() method} must raise an
 303:      *          {@link kERROR_DUPLICATE exception}.
 304:      *      <li><i>{@link kFLAG_PERSIST_UPDATE kFLAG_PERSIST_UPDATE}</i>: The provided
 305:      *          object will replace the existing object. In this case the method expects
 306:      *          the container to have an entry with the same key as the provided identifier,
 307:      *          if this is not the case the container {@link CContainer::Commit() method}
 308:      *          must raise an {@link kERROR_NOT_FOUND exception}. With this option it is
 309:      *          assumed that the provided object's attributes will replace all the existing
 310:      *          object's ones.
 311:      *      <li><i>{@link kFLAG_PERSIST_MODIFY kFLAG_PERSIST_MODIFY}</i>: The provided
 312:      *          object is assumed to contain a subset of an existing object's attributes,
 313:      *          these provided attributes will be appended or replace the existing ones.
 314:      *          In this case the method expects the container to have an entry with the
 315:      *          same key as the provided identifier, if this is not the case the method must
 316:      *          raise an {@link kERROR_NOT_FOUND exception}.
 317:      *      <li><i>{@link kFLAG_PERSIST_REPLACE kFLAG_PERSIST_REPLACE}</i>: The provided
 318:      *          object will be {@link kFLAG_PERSIST_INSERT inserted}, if the identifier
 319:      *          doesn't match any container elements, or it will
 320:      *          {@link kFLAG_PERSIST_UPDATE replace} the existing object. As with
 321:      *          {@link kFLAG_PERSIST_UPDATE update}, it is assumed that the provided
 322:      *          object's attributes will replace all the existing object's ones.
 323:      *      <li><i>{@link kFLAG_PERSIST_DELETE kFLAG_PERSIST_DELETE}</i>: This option
 324:      *          assumes you want to remove the object from the container, although the
 325:      *          container features a specific {@link CContainer::Delete() method} for this
 326:      *          purpose, this option may be used to implement a <i>deleted state</i>, rather
 327:      *          than actually removing the object from the container.
 328:      *      <li><i>{@link kFLAG_STATE_ENCODED kFLAG_STATE_ENCODED}</i>: If the option is on,
 329:      *          the object will be passed through a
 330:      *          {@link CContainer::UnserialiseObject() conversion} process that will convert
 331:      *          all {@link CDataType standard} data types into native container custom data
 332:      *          types {@link CContainer::_PrepareCommit() before} committing the object. It
 333:      *          is a good idea to keep this flag ON, because the converted object will be
 334:      *          compatible with any {@link CContainer container} and it will make data
 335:      *          transfers from one database to the other easy. This process is automatic and
 336:      *          transparent.
 337:      *   </ul>
 338:      * </ul>
 339:      *
 340:      * The method will only operate if either the {@link _IsCommitted() committed}
 341:      * {@link kFLAG_STATE_COMMITTED flag} is <i>not set</i>, or the
 342:      * {@link _IsDirty() dirty} {@link kFLAG_STATE_DIRTY flag} is <i>set</i>, if none of
 343:      * these two conditions are satisfied, the method will do nothing.
 344:      *
 345:      * The method will by default <i>set</i> the {@link _IsCommitted() committed}
 346:      * {@link kFLAG_STATE_COMMITTED flag} and <i>reset</i> the {@link _IsDirty() dirty}
 347:      * {@link kFLAG_STATE_DIRTY flag} if the operation was successful.
 348:      *
 349:      * This method should raise an exception if any error occurs.
 350:      *
 351:      * The method should return the unique identifier of the object or <i>NULL</i> if the
 352:      * operation was not performed.
 353:      *
 354:      * In derived classes you should not overload this method, instead, you should overload
 355:      * the protected interface:
 356:      *
 357:      * <ul>
 358:      *  <li><i>{@link _PrepareCommit() _PrepareCommit}()</i>: This method can be used to
 359:      *      initialise or check the parameters and resources.
 360:      *  <li><i>{@link _Commit() _Commit}()</i>: This method will perform the
 361:      *      actual commit.
 362:      *  <li><i>{@link _FinishCommit() _FinishCommit}()</i>: This method can be used to perform
 363:      *      eventual post-flight adjustments.
 364:      * </ul>
 365:      *
 366:      * @param mixed                 $theContainer       Persistent container.
 367:      * @param mixed                 $theIdentifier      Object identifier.
 368:      * @param bitfield              $theModifiers       Commit modifiers.
 369:      *
 370:      * @access public
 371:      * @return mixed
 372:      *
 373:      * @uses _IsDirty()
 374:      * @uses _IsCommitted()
 375:      * @uses _PrepareCommit()
 376:      * @uses _Commit()
 377:      * @uses _FinishCommit()
 378:      *
 379:      * @see kFLAG_PERSIST_INSERT kFLAG_PERSIST_UPDATE kFLAG_PERSIST_MODIFY
 380:      * @see kFLAG_PERSIST_REPLACE kFLAG_STATE_ENCODED
 381:      */
 382:     public function Commit( $theContainer = NULL,
 383:                             $theIdentifier = NULL,
 384:                             $theModifiers = kFLAG_PERSIST_REPLACE )
 385:     {
 386:         //
 387:         // Check if we need to do it.
 388:         //
 389:         if( $this->_IsDirty()
 390:          || (! $this->_IsCommitted())
 391:          || ($theModifiers & kFLAG_PERSIST_DELETE) )
 392:         {
 393:             //
 394:             // Prepare.
 395:             //
 396:             $this->_PrepareCommit( $theContainer, $theIdentifier, $theModifiers );
 397:             
 398:             //
 399:             // Store object.
 400:             //
 401:             $theIdentifier
 402:                 = $this->_Commit
 403:                     ( $theContainer, $theIdentifier, $theModifiers );
 404:             
 405:             //
 406:             // Finish.
 407:             //
 408:             $this->_FinishCommit( $theContainer, $theIdentifier, $theModifiers );
 409:             
 410:             return $theIdentifier;                                                  // ==>
 411:         }
 412:         
 413:         return NULL;                                                                // ==>
 414:         
 415:     } // Commit.

 460:     /**
 461:      * Manage committed status.
 462:      *
 463:      * This method can be used to get or set the object's committed state.
 464:      *
 465:      * A committed object is one that has either been loaded from a container or committed
 466:      * to a container. This state indicates that the object is persistent. This state,
 467:      * combined with the {@link _IsDirty() dirty} status, can determine if an object needs
 468:      * to be committed in a container or not.
 469:      *
 470:      * The method features a single parameter:
 471:      *
 472:      * <ul>
 473:      *  <li><i>NULL</i>: The method will return <i>TRUE</i> if the object is committed, or
 474:      *      <i>FALSE</i> if the object is not committed.
 475:      *  <li><i>TRUE</i>: The method will set the object to committed.
 476:      *  <li><i>FALSE</i>: The method will reset the object's committed state.
 477:      * </ul>
 478:      *
 479:      * In all cases the method will return the state <i>after</i> it was eventually
 480:      * modified.
 481:      *
 482:      * @param mixed                 $theState           TRUE, FALSE or NULL.
 483:      *
 484:      * @access protected
 485:      * @return boolean
 486:      *
 487:      * @uses _ManageBitField()
 488:      *
 489:      * @see kFLAG_STATE_COMMITTED
 490:      */
 491:     protected function _IsCommitted( $theState = NULL )
 492:     {
 493:         return $this->_ManageBitField( $this->mStatus,
 494:                                        kFLAG_STATE_COMMITTED,
 495:                                        $theState );                                 // ==>
 496:     
 497:     } // _IsCommitted.

 501:      *  _IsEncoded                                                                      *
 502:      *==================================================================================*/
 503: 
 504:     /**
 505:      * Manage encoded status.
 506:      *
 507:      * This method can be used to get or set the object's encoded state.
 508:      *
 509:      * This flag determines whether custom data types are to be converted to
 510:      * {@link CDataType standard} types when working with the object. This is especially
 511:      * useful when transmitting the object through the network or when managing custom
 512:      * database types.
 513:      *
 514:      * The conversion to {@link CDataType standard} types is
 515:      * {@link CDataType::SerialiseObject() done} when {@link __construct() loading} the
 516:      * object from a {@link CContainer container} and the opposite is
 517:      * {@link CContainer::UnserialiseObject() done} before {@link Commit() storing} the
 518:      * object into a {@link CContainer container}, these operations are performed by the
 519:      * {@link CContainer container} object that takes care of object persistence.
 520:      *
 521:      * This {@link kFLAG_STATE_ENCODED kFLAG_STATE_ENCODED} flag can be provided at
 522:      * {@link __construct() instantiation} or directly to the persistence methods, although
 523:      * it is a better idea to provide it in the {@link __construct() constructor} to have
 524:      * this conversion happen transparently.
 525:      *
 526:      * The method features a single parameter:
 527:      *
 528:      * <ul>
 529:      *  <li><i>NULL</i>: The method will return <i>TRUE</i> if the object is encoded, or
 530:      *      <i>FALSE</i> if the object is not encoded.
 531:      *  <li><i>TRUE</i>: The method will set the object to encoded.
 532:      *  <li><i>FALSE</i>: The method will reset the object's encoded state.
 533:      * </ul>
 534:      *
 535:      * In all cases the method will return the state <i>after</i> it was eventually
 536:      * modified.
 537:      *
 538:      * @param mixed                 $theState           TRUE, FALSE or NULL.
