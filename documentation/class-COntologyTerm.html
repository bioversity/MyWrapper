Class COntologyTerm

Generic term.

This kind of COntologyTermObject term represents a generic ontology term.

This class handles the _IsInited() inited status by checking the following values in its Kind() kinds list:

  • Default: By default the object must have its kTAG_CODE code set.
  • kTYPE_NAMESPACE kTYPE_NAMESPACE: It uses the default requirements.
  • kTYPE_ROOT kTYPE_ROOT: It requires the kTAG_NAME name to be set.
  • kTYPE_PREDICATE kTYPE_PREDICATE: It requires the kTAG_NAME name.
  • kTYPE_ATTRIBUTE kTYPE_ATTRIBUTE: It requires the kTAG_NAME name and kTAG_CARDINALITY cardinality.
  • kTYPE_MEASURE kTYPE_MEASURE: It requires the kTAG_NAME name and kTAG_TYPE type.
  • kTYPE_ENUMERATION kTYPE_ENUMERATION: It requires the kTAG_NAME name and kTAG_ENUM enumeration.

In this class we ensure that the term has both the Code() code and Name() name in order to have an _IsInited() inited kFLAG_STATE_INITED status.

# Image( mixed $theKind, mixed $theType, mixed $theData = NULL, boolean $getOld = FALSE )

Manage images.

Manage images.

This method can be used to manage the term's list of kOFFSET_IMAGE images, this kOFFSET_IMAGE offset is represented by an array of items holding three elements:

  • kTAG_KIND kTAG_KIND: This element represents the kind or qualifier of the image, the element is required.
  • kTAG_TYPE kTAG_TYPE: This element represents the data type of the image, this element is required.
  • kTAG_DATA kTAG_DATA: This element represents the image data which should be expressed in the data type declared in the kTAG_TYPE kTAG_TYPE element.

The method expects the following parameters:

  • $theOffset: The main offset to manage.
  • $theKind: The item kTAG_KIND kind; it should be able to cast this value to a string which represents an index.
  • $theType: The item kTAG_TYPE type; it should be able to cast this value to a string which represents an index.
  • $theData: This parameter represents the item's kTAG_DATA data element, or the operation to be performed:
    • NULL: This indicates that we want to retrieve the data of the item with index matching the previous parameters.
    • FALSE: This indicates that we want to remove the item matching the index provided in the previous parameters.
    • other: Any other value indicates that we want to add or replace the kTAG_DATA data element of the item matching the previous parameters.
  • $getOld: Determines what the method will return:
    • TRUE: Return the element or list before it was eventually modified.
    • FALSE: Return the element or list after it was eventually modified.

The method makes use of a static CAttribute::ManageTypedOffset() method, please consult its reference for more information.

Manage term sources.

Manage term sources.

This method can be used to manage the term kTAG_SOURCE sources, it manages an array of strings with the following offsets:

  • kTAG_KIND kTAG_KIND: The source kind, this string can be used to define the type of the source, this element represents the array key, although technically it is implemented as an element to allow searching on all values.
  • kTAG_DATA kTAG_DATA: The source, this element should hold the actual source reference, which should be convertable to a string.

The parameters to this method are:

  • $theValue: The value or operation:
    • NULL: Return the current value selected by the second parameter.
    • FALSE: Delete the value selected by the second parameter.
    • other: Set value selected by the second parameter.
  • $theType: The element type, kind or index:
    • NULL: This value indicates that the phone has no type or kind, in general, when adding elements, this case applies to default elements.
    • other: All other types will be interpreted as the kind or type of the phone number.
  • $getOld: Determines what the method will return:
    • TRUE: Return the value before it was eventually modified.
