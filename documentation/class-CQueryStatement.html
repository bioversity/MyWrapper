Class CQueryStatement

Query statement.

This class implements a query statement, such as those that populate the CQuery query class. This class concentrates on building and managing these statements, which are structured as follows:

  • kAPI_QUERY_SUBJECT kAPI_QUERY_SUBJECT: The query subject. It refers to the object element that we are considering in the query.
  • kAPI_QUERY_OPERATOR kAPI_QUERY_OPERATOR: The query predicate. This element represents the predicate or comparaison operator, it can take the following values:
    • kOPERATOR_DISABLED kOPERATOR_DISABLED: Disabled, it means that the current statement is disabled.
    • kOPERATOR_EQUAL kOPERATOR_EQUAL: Equality (=), this operator implies that the statement has also an kAPI_QUERY_DATA object.
    • kOPERATOR_EQUAL_NOT kOPERATOR_EQUAL_NOT: Inequality (!=), negates the kOPERATOR_EQUAL kOPERATOR_EQUAL operator; this operator implies that the statement has also an kAPI_QUERY_DATA object.
    • kOPERATOR_LIKE kOPERATOR_LIKE: Like, it is an accent and case insensitive equality filter, this operator implies that the statement has also an kAPI_QUERY_DATA object.
    • kOPERATOR_LIKE_NOT kOPERATOR_LIKE_NOT: The negation of the kOPERATOR_LIKE LIKE operator, this operator implies that the statement has also an kAPI_QUERY_DATA object.
    • kOPERATOR_PREFIX kOPERATOR_PREFIX: Starts with, or prefix match, this operator implies that the statement has also an kAPI_QUERY_DATA object.
    • kOPERATOR_CONTAINS kOPERATOR_CONTAINS: Contains, selects all elements that contain the match string, this operator implies that the statement has also an kAPI_QUERY_DATA object.
    • kOPERATOR_SUFFIX kOPERATOR_SUFFIX: Ends with, or suffix match, this operator implies that the statement has also an kAPI_QUERY_DATA object.
    • kOPERATOR_REGEX kOPERATOR_REGEX: Regular expression, this operator implies that the statement has also an kAPI_QUERY_DATA object.
    • kOPERATOR_LESS kOPERATOR_LESS: Smaller than (\<), this operator implies that the statement has also an kAPI_QUERY_DATA object.
    • kOPERATOR_LESS_EQUAL kOPERATOR_LESS_EQUAL: Smaller than or equal (\<=), this operator implies that the statement has also an kAPI_QUERY_DATA object.
    • kOPERATOR_GREAT kOPERATOR_GREAT: Greater than (\>), this operator implies that the statement has also an kAPI_QUERY_DATA object.
    • kOPERATOR_GREAT_EQUAL kOPERATOR_GREAT_EQUAL: Greater than or equal (\>=), this operator implies that the statement has also an kAPI_QUERY_DATA object.
    • kOPERATOR_IRANGE kOPERATOR_IRANGE: Range inclusive, matches \>= value \<=; this operator implies that the statement has also an kAPI_QUERY_DATA object which should contain the two range values.
    • kOPERATOR_ERANGE kOPERATOR_ERANGE: Range exclusive, matches \> value \<, this operator implies that the statement has also an kAPI_QUERY_DATA object which should contain the two range values.
    • kOPERATOR_NULL kOPERATOR_NULL: Is NULL or element is missing.
    • kOPERATOR_NOT_NULL kOPERATOR_NOT_NULL:Not NULL or element exists.
    • kOPERATOR_IN kOPERATOR_IN: In, or belongs to set, this operator implies that the statement has also an kAPI_QUERY_DATA object that will contain the list of choices.
    • kOPERATOR_NI kOPERATOR_NI: Not in, the negation of kOPERATOR_IN kOPERATOR_IN, this operator implies that the statement has also an kAPI_QUERY_DATA object which contains the list of choices.
    • kOPERATOR_ALL kOPERATOR_ALL: All, or match the full set, this operator implies that the statement has also an kAPI_QUERY_DATA object which will contain the set.
    • kOPERATOR_NALL kOPERATOR_NALL: Not all, the negation of the kOPERATOR_ALL kOPERATOR_ALL operator, this operator implies that the statement has also an kAPI_QUERY_DATA object which contains the set.
    • kOPERATOR_EX kOPERATOR_EX: Expression, indicates a complex expression, this operator implies that the statement has also an kAPI_QUERY_DATA object which will contain the expression.
  • kAPI_QUERY_TYPE kAPI_QUERY_TYPE: The statement object type, or data type of the kAPI_QUERY_DATA kAPI_QUERY_DATA element, if the latter is required:
    • kTYPE_STRING kTYPE_STRING: String, we assume in UTF8 character set.
    • kTYPE_INT32 kTYPE_INT32: 32 bit signed integer.
    • kTYPE_INT64 kTYPE_INT64: 64 bit signed integer.
    • kTYPE_FLOAT kTYPE_FLOAT: Floating point number.
    • kTYPE_DATE kTYPE_DATE: A date.
    • kTYPE_TIME kTYPE_TIME: A date and time.
    • kTYPE_STAMP kTYPE_STAMP: A native timestamp.
    • kTYPE_BOOLEAN kTYPE_BOOLEAN: An on/off switch.
    • kTYPE_BINARY kTYPE_BINARY: A binary string.
  • kAPI_QUERY_DATA kAPI_QUERY_DATA: The statement object or test data.

The main goal of this class is to provide an interface that may ease the construction of complex CQuery queries by providing specialised methods for building statements that can then safely be CQuery::AppendStatement() appended to query CQuery objects.

Instantiate class.

Instantiate class.

The constructor can be used to instantiate a statement either by providing an existing statement structure, or by providing the statement elements:

  • $theSubject: The statement subject:
    • array or ArrayObject: Structures are interpreted as built statements, so the method will scan the structure and load the corresponding elements.
    • string: Any other type will be converted to a string and interpreted as the statement subject, or data element key.
  • $thePredicate: The statement operator or predicate:
    • kOPERATOR_DISABLED kOPERATOR_DISABLED: Disabled, it means that the current statement is disabled; the remaining parameters are ignored.
    • kOPERATOR_EQUAL kOPERATOR_EQUAL: Equality (=), this operator implies that the method expects the next two parameters.
    • kOPERATOR_EQUAL_NOT kOPERATOR_EQUAL_NOT: Inequality (!=), negates the kOPERATOR_EQUAL kOPERATOR_EQUAL operator; this operator implies that the method expects the next two parameters.
    • kOPERATOR_LIKE kOPERATOR_LIKE: Like, it is an accent and case insensitive equality filter, this operator implies that the method expects the next two parameters.
    • kOPERATOR_LIKE_NOT kOPERATOR_LIKE_NOT: The negation of the kOPERATOR_LIKE LIKE operator, this operator implies that the method expects the next two parameters.
    • kOPERATOR_PREFIX kOPERATOR_PREFIX: Starts with, or prefix match, this operator implies that the method expects the next two parameters.
    • kOPERATOR_CONTAINS kOPERATOR_CONTAINS: Contains, selects all elements that contain the match string, this operator implies that the method expects the next two parameters.
    • kOPERATOR_SUFFIX kOPERATOR_SUFFIX: Ends with, or suffix match, this operator implies that the method expects the next two parameters.
    • kOPERATOR_REGEX kOPERATOR_REGEX: Regular expression, this operator implies that the method expects the next two parameters.
    • kOPERATOR_LESS kOPERATOR_LESS: Smaller than (\<), this operator implies that the method expects the next two parameters.
    • kOPERATOR_LESS_EQUAL kOPERATOR_LESS_EQUAL: Smaller than or equal (\<=), this operator implies that the method expects the next two parameters.
    • kOPERATOR_GREAT kOPERATOR_GREAT: Greater than (\>), this operator implies that the method expects the next two parameters.

  • $theType: The statement object data type, this qualifies all remaining parameters. The allowed values are:
    • kTYPE_STRING kTYPE_STRING: String, we assume in UTF8 character set, the string is expected in the next parameter.
    • kTYPE_INT32 kTYPE_INT32: 32 bit signed integer, the number is expected in the next parameter, either as an integer, float or string; once received, it will be converted to a CDataTypeInt32 CDataTypeInt32 object.
    • kTYPE_INT64 kTYPE_INT64: 64 bit signed integer, the number is expected in the next parameter, either as an integer, float or string; once received, it will be converted to a CDataTypeInt64 CDataTypeInt64 object.
    • kTYPE_FLOAT kTYPE_FLOAT: Floating point number, the number is expected in the next parameter, either as an integer, float or string.
    • kTYPE_DATE kTYPE_DATE: A string date, it is treated as a string date with a YYYYMMDD format in which month and day may be omitted.
    • kTYPE_TIME kTYPE_TIME: A string time, it is treated as a string time with a YYYY-MM-DD HH:MM:SS format in which all elements are required; this element will be converted to a kTYPE_STAMP kTYPE_STAMP data type.
    • kTYPE_STAMP kTYPE_STAMP: A timestamp, optionally including microseconds, elements of this type will be converted to CDataTypeStamp CDataTypeStamp objects.
    • kTYPE_BOOLEAN kTYPE_BOOLEAN: An on/off switch, it will be converted to a 1/0 pair.
    • kTYPE_BINARY kTYPE_BINARY: A binary string, the string will be converted to a CDataTypeBinary CDataTypeBinary object.
  • $theObject: The statement object data, it should reflect the data type provided in the previous parameter.
  • $theRange: The statement range end or second element. This value must also reflect the data type provided in the previous parameter and will be automatically set if you provided a range and forgot to set it.
