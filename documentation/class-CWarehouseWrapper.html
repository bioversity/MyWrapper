Class CWarehouseWrapper

Warehouse Mongo data wrapper.

This class overloads its CMongoDataWrapper ancestor to implement a web-service customised to the germplasm data warehouse.

This class adds the following operations:

  • kAPI_OP_LOGIN kAPI_OP_LOGIN: Login, this operation expects the user kAPI_OPT_USER_CODE code and kAPI_OPT_USER_PASS password and will return the matching user.
  • kAPI_OP_GET_TERMS kAPI_OP_GET_TERMS: Get terms, this operation will return the list of ontology COntologyTerm terms matching the identifiers provided in the kAPI_OPT_IDENTIFIERS kAPI_OPT_IDENTIFIERS option. This list is expected to hold COntologyTerm term kTAG_GID identifiers.
  • kAPI_OP_GET_NODES kAPI_OP_GET_NODES: Get terms, this operation will return the list of ontology COntologyTerm terms matching the identifiers provided in the kAPI_OPT_IDENTIFIERS kAPI_OPT_IDENTIFIERS option. This list is expected to hold COntologyTerm term kTAG_GID identifiers.

The class adds the following options:

  • kAPI_OPT_USER_CODE kAPI_OPT_USER_CODE: User code, this option expects a string corresponding to the user CEntity::Code() code.
  • kAPI_OPT_USER_PASS kAPI_OPT_USER_PASS: User code, this option expects a string corresponding to the user CUser::Password() password.
  • kAPI_OPT_IDENTIFIERS kAPI_OPT_IDENTIFIERS: Identifiers list, this option expects an array of object identifiers. The actual type of these elements is determined by the operation.

# _Handle_GetRelations( )

Handle kAPI_OP_GET_RELS get-relations request.

Handle kAPI_OP_GET_RELS get-relations request.

This method will return the same structure as the kAPI_OP_GET_EDGES kAPI_OP_GET_EDGES service:

  • kAPI_RESPONSE_EDGES kAPI_RESPONSE_EDGES: The list of edges as an array structured as follows:
    • key: The edge identifier.
    • Value: An array structured as follows:
      • kAPI_RESPONSE_SUBJECT kAPI_RESPONSE_SUBJECT: The subject COntologyNode node ID.
      • kAPI_RESPONSE_PREDICATE kAPI_RESPONSE_PREDICATE: The predicate COntologyTerm term kTAG_GID identifier.
      • kAPI_RESPONSE_OBJECT kAPI_RESPONSE_OBJECT: The object COntologyNode node ID.
  • kAPI_RESPONSE_NODES kAPI_RESPONSE_NODES: The list of kAPI_RESPONSE_SUBJECT subject and kAPI_RESPONSE_OBJECT object found nodes as follows:
    • Key: The node ID.
    • Value: The node properties.
  • kAPI_RESPONSE_TERMS kAPI_RESPONSE_TERMS: The list of terms related to the list of found nodes and to the edge predicate as follows:
    • Key: The COntologyTerm term global kTAG_GID identifier.
    • Value: The contents of the COntologyTerm term.

Depending on the value of the kAPI_OPT_DIRECTION kAPI_OPT_DIRECTION parameter:

  • kAPI_DIRECTION_IN kAPI_DIRECTION_IN: The service will return all COntologyEdge edges that point to the COntologyNode nodes provided in the kAPI_OPT_IDENTIFIERS kAPI_OPT_IDENTIFIERS parameter.
  • kAPI_DIRECTION_OUT kAPI_DIRECTION_OUT: The service will return all COntologyEdge edges pointing from the COntologyNode nodes provided in the kAPI_OPT_IDENTIFIERS kAPI_OPT_IDENTIFIERS parameter.
  • kAPI_DIRECTION_ALL kAPI_DIRECTION_ALL: The service will return all COntologyEdge edges connected in any way to the COntologyNode nodes provided in the kAPI_OPT_IDENTIFIERS kAPI_OPT_IDENTIFIERS parameter.

The service also expects a kAPI_OPT_LEVELS kAPI_OPT_LEVELSparameter, a signed integer, that indicates how many levels to recurse the graph traversal, if this parameter is not provided, it will default to 1 level; to traverse all levels this parameter should be set to a negative number; a level of 0 will only return the list of involved nodes and terms.

If the kAPI_OPT_PREDICATES kAPI_OPT_PREDICATES parameter was provided, only those COntologyEdge edges whose type matches any of the predicate COntologyTerm term identifiers provided in that parameter will be selected.

If the kAPI_OPT_IDENTIFIERS kAPI_OPT_IDENTIFIERS parameter was not provided, the method will return the above structure with no content.

Note: this method handles the case in which the kAPI_OPT_DIRECTION direction is not provided: in that case it will treat the list of kAPI_OPT_IDENTIFIERS identifiers as a list of edge node IDs and will work as the kAPI_OP_GET_EDGES kAPI_OP_GET_EDGES service.
