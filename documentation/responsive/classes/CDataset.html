    Dataset object.

    Besides the inherited properties. datasets have the following attributes:

    • Title: The dataset title represents the dataset name or identifier provided by the dataset creator, this attribute is used in the object's identifier.
    • User: The dataset user represents the identifier of the user that created the dataset, this attribute is used in the object's identifier.
    • Name: The dataset name represents the name or label by which the dataset is referred to. Unlike the title which has an identification purpose, this attribute has a documentation purpose and can be expressed in several languages.
    • Description: The dataset description represents a description or definition of the dataset, it can be expressed in several languages.

    By default, the unique identifier of the object is its code, which is also its id.

    Objects of this class require at least the code offset to have an initialised status.

    Manage dataset description.

    Description(mixed $theValue, mixed $theLanguage, boolean $getOld) : string

    This method can be used to manage the description, it manages an array of structures with the following offsets:

    • kTAG_LANGUAGE: The description's language, this element represents the code of the language in which the next element is expressed in.
    • kTAG_DATA: The dataset description or comment.

    The parameters to this method are:

    • $theValue: The description or operation:
      • NULL: Return the current value selected by the second parameter.
      • FALSE: Delete the value selected by the second parameter.
      • other: Set value selected by the second parameter.
    • $theLanguage: The description's language code:
      • NULL: This value indicates that the description has no language, in general, when adding elements, this case applies to default elements.
      • other: All other types will be interpreted as the language code.
    • $getOld: Determines what the method will return:
      • TRUE: Return the value before it was eventually modified.
      • FALSE: Return the value after it was eventually modified.

    Store a file.

    StoreFile(string $theFile, \CMongoGridContainer $theContainer, mixed $theMetadata, \bitfield $theModifiers) : \CDatasetFile

    This method can be used to store a file in a grid container, it will take care of adding a reference to the current dataset in the provided file metadata record.

    This method accepts the following parameters:

    • $theFile: The file path or object (SplFileInfo).
    • $theContainer: The grid container in the form of a CMongoGridContainer or a MongoGridFS; any other type will raise an exception.
    • $theMetadata: An array or preferably a CDatasetFile record containing the file's metadata, which will be added to the MongoGridFS record under the "metadata" label. If the parameter is omitted, a CDatasetFile record will be initialised with a reference to the current dataset.
    • $theModifiers: A bitfield containing the operation options, the only relevant flag is kFLAG_STATE_ENCODED which determines whether custom data types are to be encoded (set by default in this class).

    The current dataset must have been committed before calling this method, or an exception will be raised: no files can be saved from this class without a valid dataset reference.

    The mnethod will return the newly created file reference ID.
