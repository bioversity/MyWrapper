    Institute.

    This class overloads its ancestor to implement an institute contact entity.

    Institutes feature specific properties that are added to the inherited ones:

    • kOFFSET_ACRONYM: This offset represents the institute list of acronyms.
    • kOFFSET_URL: This offset represents the institute list of URLs or web pages.

    The object is considered initialised only if it has its code, as its ancestor, its name, password and e-mail address.

    If the code has not been explicitly set, before committing the object it will be set to the value of the e-mail. Also in that phase, the kENTITY_USER constant will be set in the user type.

    The e-mail in this class is a scalar property, in other classes it will probably be a list of different e-mail types. In this class we want to link a single user with a single e-mail, possibly not shared by any other user, that is why we link by default the user code and e-mail.

    The unique identifier of this class is composed by the default index of thew object, prefixed by the kENTITY_INST token and the kTOKEN_CLASS_SEPARATOR token, this allows institutes and other types of entities to share the same code; this is enforced both in the HashIndex method, to which you only need to pass the user code, and in the protected _PrepareCommit method which will place the resulting string in the global identifier.

    Manage entity acronyms.

    Acronym(mixed $theValue, mixed $theOperation, boolean $getOld) : mixed

    This method can be used to handle the institute acronyms list, it uses the standard accessor method to manage the list of acronyms.

    Each element of this list should indicate an acronym by which one refers to the current institute, the nature and specifics of these elements is the responsibility of concrete classes.

    For a more thorough reference of how this method works, please consult the CAttribute::ManageArrayOffset method, in which the second parameter will be the constant kOFFSET_ACRONYM.

    Manage institute urls.

    URL(string $theValue, mixed $theType, boolean $getOld) : string

    This method can be used to manage the institute URL or web pages list, the method expects the following parameters:

    • $theValue: The value or operation:
      • NULL: Return the current value selected by the second parameter.
      • FALSE: Delete the value selected by the second parameter.
      • other: Set value selected by the second parameter.
    • $theType: The element type, kind or index:
      • NULL: This value indicates that the URL has no type or kind, in general, when adding elements, this case applies to default elements.
      • other: All other types will be interpreted as the kind or type of the URL.
    • $getOld: Determines what the method will return:
      • TRUE: Return the value before it was eventually modified.
      • FALSE: Return the value after it was eventually modified.
