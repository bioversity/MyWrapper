    Generic term.

    This kind of term represents a generic ontology term.

    This class handles the inited status by checking the following values in its kinds list:

    • Default: By default the object must have its code set.
    • kTYPE_NAMESPACE: It uses the default requirements.
    • kTYPE_ROOT: It requires the name to be set.
    • kTYPE_PREDICATE: It requires the name.
    • kTYPE_ATTRIBUTE: It requires the name and cardinality.
    • kTYPE_MEASURE: It requires the name and type.
    • kTYPE_ENUMERATION: It requires the name and enumeration.

    In this class we ensure that the term has both the code and name in order to have an inited status.

    Manage data cardinality.

    Cardinality(NULL | FALSE | string $theValue, boolean $getOld) : string

    This method can be used to manage the measure data type, it uses the standard accessor method to manage the offset:

    • $theValue: The value or operation:
      • NULL: Return the current value.
      • FALSE: Delete the value.
      • string: String values represent the cardinality enumeration:
        • kCARD_0_1: Zero or one, a scalar or none.
        • kCARD_1: Exactly one, a required scalar.
        • kCARD_ANY: Any, this implies that we either have an array or no data.
      • integer: An integer represents the exact cardinality, in that case we assume the data is an array of at most that number of elements.
    • $getOld: Determines what the method will return:
      • TRUE: Return the value before it was eventually modified.
      • FALSE: Return the value after it was eventually modified.

    Manage cross-references.

    Image(mixed $theKind, mixed $theType, mixed $theData, boolean $getOld) : string

    This method can be used to manage the term's list of images, this offset is represented by an array of items holding three elements:

    • kTAG_KIND: This element represents the kind or qualifier of the image, the element is required.
    • kTAG_TYPE: This element represents the data type of the image, this element is required.
    • kTAG_DATA: This element represents the image data which should be expressed in the data type declared in the kTAG_TYPE element.

    The method expects the following parameters:

    • $theOffset: The main offset to manage.
    • $theKind: The item kind; it should be able to cast this value to a string which represents an index.
    • $theType: The item type; it should be able to cast this value to a string which represents an index.
    • $theData: This parameter represents the item's data element, or the operation to be performed:
      • NULL: This indicates that we want to retrieve the data of the item with index matching the previous parameters.
      • FALSE: This indicates that we want to remove the item matching the index provided in the previous parameters.
      • other: Any other value indicates that we want to add or replace the data element of the item matching the previous parameters.
    • $getOld: Determines what the method will return:
      • TRUE: Return the element or list before it was eventually modified.
      • FALSE: Return the element or list after it was eventually modified.

    The method makes use of a static method, please consult its reference for more information.

    Manage term sources.

    Source(string $theValue, mixed $theType, boolean $getOld) : string

    This method can be used to manage the term sources, it manages an array of strings with the following offsets:

    • kTAG_KIND: The source kind, this string can be used to define the type of the source, this element represents the array key, although technically it is implemented as an element to allow searching on all values.
    • kTAG_DATA: The source, this element should hold the actual source reference, which should be convertable to a string.

    The parameters to this method are:

    • $theValue: The value or operation:
      • NULL: Return the current value selected by the second parameter.
      • FALSE: Delete the value selected by the second parameter.
      • other: Set value selected by the second parameter.
    • $theType: The element type, kind or index:
      • NULL: This value indicates that the phone has no type or kind, in general, when adding elements, this case applies to default elements.
      • other: All other types will be interpreted as the kind or type of the phone number.
    • $getOld: Determines what the method will return:
      • TRUE: Return the value before it was eventually modified.
      • FALSE: Return the value after it was eventually modified.

    _PrepareCommit(\reference $theContainer, \reference $theIdentifier, \reference $theModifiers) 

    We overload this method to set default kinds according to which data properties the object has:

    • Cardinality: We set the kind to attribute.
    • Data Type: We set the kind to measure.
    • Enumeration: We set the kind to enumeration.

    We also set by default the enumeration to the value of the code, if the term has the enumeration kind and the term enumerations are empty.
