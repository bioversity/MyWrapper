    User.

    This class overloads its ancestor to implement a user entity.

    Users are entities that are used for authentication purposes, they share the same attributes as their parent class and add one required element:

    • kOFFSET_PASSWORD: This offset represents the user access password, it will be used in the authentication process. The class features a member accessor method to manage this property.

    The object is considered initialised only if it has its code, as its ancestor, its name, password and e-mail address.

    If the code has not been explicitly set, before committing the object it will be set to the value of the e-mail. Also in that phase, the kENTITY_USER constant will be set in the user type.

    The e-mail in this class is a scalar property, in other classes it will probably be a list of different e-mail types. In this class we want to link a single user with a single e-mail, possibly not shared by any other user, that is why we link by default the user code and e-mail.

    Note: this class enforces the encoded status flag, because the object identifier is a binary string, so always use complex data type instances derived from the standard types.

    The unique identifier of this class is composed by the default index of the object, prefixed by the kENTITY_USER token and the kTOKEN_CLASS_SEPARATOR token, this allows users and other types of entities to share the same code; this is enforced both in the HashIndex method, to which you only need to pass the user code, and in the protected _PrepareCommit method which will place the resulting string in the global identifier.

    Hash index.

    HashIndex(string $theValue) : string

    This method can be used to format an identifier provided as a string, it will be used by the _id method to format the result of the _index method. One can consider this as the index hashing method for all derived classes.

    In this class we take the provided code and prefix it with the kENTITY_USER token, the result will be hashed.

    Manage user manager.

    Manager(NULL | FALSE | string $theValue, boolean $getOld) : string

    This method can be used to manage the user manager, it uses the standard accessor method to manage the offset:

    • $theValue: The value or operation:
      • NULL: Return the current value.
      • FALSE: Delete the value.
      • other: Set value.
    • $getOld: Determines what the method will return:
      • TRUE: Return the value before it was eventually modified.
      • FALSE: Return the value after it was eventually modified.

    The manager is either the user that created the current user or the user that is in charge of the current user.
