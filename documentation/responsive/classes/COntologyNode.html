    Ontology graph node.

    This class implements an ontology graph node.

    The class is derived from CGraphNode, it adds another required property, the term. This class will wrap the array access framework to the combination of the node properties and the term elements, except that the term elements will be read-only.

    This class introduces a new kind of container: it must be an array of two elements structured as follows:

    • kTAG_NODE: This element should hold the nodes container, it must be a Everyman\Neo4j\Client instance.
    • kTAG_TERM: This element should hold the terms container, it must be a CContainer instance.

    Note that the class will not cast to an array correctly, you must use the getArrayCopy method to get an array, if you know how to solve this, please do it!

    Manage data cardinality.

    Cardinality(NULL | FALSE | string $theValue, boolean $getOld) : string

    This method can be used to manage the measure data type, it uses the standard accessor method to manage the offset:

    • $theValue: The value or operation:
      • NULL: Return the current value.
      • FALSE: Delete the value.
      • string: String values represent the cardinality enumeration:
        • kCARD_0_1: Zero or one, a scalar or none.
        • kCARD_1: Exactly one, a required scalar.
        • kCARD_ANY: Any, this implies that we either have an array or no data.
      • integer: An integer represents the exact cardinality, in that case we assume the data is an array of at most that number of elements.
    • $getOld: Determines what the method will return:
      • TRUE: Return the value before it was eventually modified.
      • FALSE: Return the value after it was eventually modified.

    Return related {@link COntologyEdge edges}.

    GetRelated(\reference $theContainer, mixed $thePredicate, string $theDirection) : array

    This method will return all nodes that are related to the current one in an array of edge nodes whose elements are structured as follows:

    • Key: The edge ID.
    • Value: The edge object.

    The method accepts the following parameters:

    • $theContainer: The graph and term containers as an array:
      • kTAG_NODE: This element should hold the nodes container, it must be a Everyman\Neo4j\Client instance.
      • kTAG_TERM: This element should hold the terms container, it must be a CContainer instance.
    • $thePredicate: The predicate terms as an array of the following types:
      • COntologyNode: The node term global identifier.
      • COntologyTerm: The term global identifier will be used as the node type.
      • Everyman\Neo4j\Relationship: The relationship's type will be used as the predicate, all other elements of the provided edge node will be ignored.
      • string: Any other type will be converted to string and will be used as the node type.
    • $theDirection: The direction of the relation:
      • out: Outgoing relations, the current node is the subject.
      • in: Incoming relations, the current node is the object.
      • all: Incoming and outgoing relations (default), the current node is the object or the subject.

    Create a graph edge.

    RelateTo(\reference $theContainer, mixed $thePredicate, mixed $theObject, boolean $doDuplicate) : \COntologyEdge

    This method can be used to create a graph edge or relation between the current node and an object node using a predicate node. This method consideres the triplet subject node/predicate term/object node as the relationship identifier when considering duplicate relationships. For that reason the subject and object terms of the relationship must relate to committed nodes.

    The method accepts the following parameters:

    • $theContainer: The graph and term containers as an array:
      • kTAG_NODE: This element should hold the nodes container, it must be a Everyman\Neo4j\Client instance.
      • kTAG_TERM: This element should hold the terms container, it must be a CContainer instance.
    • $thePredicate: The predicate term:
      • COntologyNode: The node term global identifier.
      • COntologyTerm: The term global identifier will be used as the node type.
      • Everyman\Neo4j\Relationship: The relationship's type will be used as the predicate, all other elements of the provided edge node will be ignored.
      • string: Any other type will be converted to string and will be used as the node type.
    • $theObject: The destination node or relationship object node:
      • COntologyNode: The method will use it's node.
      • Everyman\Neo4j\Node: The method will use it as the relationship object.
      • integer: The method will search for the node corresponding to the provided number, if the node was not found, the method will raise an exception.
      • other: Any other type will raise an exception.
    • $doDuplicate: This flag determines whether cuplicate relationships are allowed: if FALSE, if a relationship with the same subject and object nodes, along with the same predicate term are found, it will be considered a duplicate, and this method will actually return that relationship.

    The method will return a COntologyEdge object, or raise an exception if the operation was not successful.

    _IndexNodes(Everyman\Neo4j\Client $theContainer) 

    This method will save node indexes after the node was committed, there are two main indexes for node properties:

    • kINDEX_NODE_NODE: This index (NodeIndex) links the node to its properties through the following keys:
      • kTAG_TYPE: This key links the current node to its type, which may either be inherited from its term or have been explicitly set.
      • kTAG_KIND: This key links the current node to its kinds, which may either be inherited from its term or have been explicitly set.
      • kTAG_DOMAIN: This key links the current node to its domains, which may either be inherited from its term or have been explicitly set.
      • kTAG_CATEGORY: This key links the current node to its categories, which may either be inherited from its term or have been explicitly set.

    _ManagePropertyArray(string $theOffset, mixed $theValue, mixed $theOperation, boolean $getOld) : mixed

    This method will set, retrieve and delete array element properties, it accepts the following parameters:

    • $theOffset: The property offset to manage.
    • $theValue: This parameter represents either the value to add, or the index of the element to operate on:
      • NULL: This value indicates that we want to operate on all elements, which means that we are retrieving the full list or deleting it.
      • array: This value indicates that we want to replace the whole list, this will only be tested if the next parameter evaluates to TRUE.
      • other: Any other type represents either the new value to be added or the index to the value to be returned or deleted. It must be possible to cast this value to a string, this is what will be used to compare elements.
    • $theOperation: This parameter represents the operation to be performed, it will be evaluated as a boolean and its scope depends on the value of the previous parameter:
      • NULL: Return the element or list.
      • FALSE: Delete the element or list.
      • TRUE: Add the element or list. Note that with this value, if you provide NULL in the previous parameter, it will be equivalent to deleting the whole list.
    • $getOld: Determines what the method will return:
      • TRUE: Return the element or list before it was eventually modified.
      • FALSE: Return the element or list after it was eventually modified.

    uses \Node()
    uses \_IsDirty()

    _PrepareCreate(\reference $theContainer, \reference $theIdentifier, \reference $theModifiers) 

    In this class we first check whether the container has the following structure:

    • kTAG_NODE: This element should hold the nodes container, it must be a Everyman\Neo4j\Client instance.
    • kTAG_TERM: This element should hold the terms container, it must be a COntologyTermObject instance.

    We then call the parent method and check if the container was replaced with the content: in that case we try to load the related term.
