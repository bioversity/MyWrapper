    Node index.

    This class implements a graph node index, it can be used to replicate a Neo4j node to a MongoDB collection.

    Neo4j uses Lucene as its standard indexing tool, in this library we use MongoDB as the standard database, so it is a logical choice to use Mongo as the indexing mechanism for Neo4j. Also, by storing nodes in Mongo, by dumping the database contents we also have the graph structure with it.

    The class features a single data member, the node which contains a Neo4j node reference. When instantiating this class you are required to provide a persistent instance of a node.

    The contents of the object will be the node properties as will be stored in the Mongo default collection for nodes:

    • kTAG_LID: This offset will hold the node ID.
    • kTAG_DATA: This offset will hold the node properties.

    The class provides a single member accessor method: Node, which can be used to set the referenced node.

    Note that the class will not cast to an array correctly, you must use the getArrayCopy method to get an array, if you know how to solve this, please do it!

    Commit the object into a container.

    Commit(mixed $theContainer, \bitfield $theModifiers) : mixed

    This method should be used to commit the object to a container, the method accepts two parameters:

    • $theContainer: This parameter represents the index container, the value will be fed to the _ResolveIndexContainer protected method which should return a CMongoContainer.
    • $theModifiers: This parameter represents the commit operation options, only the following options are considered:
      • kFLAG_PERSIST_REPLACE: The provided object will be inserted, if the identifier doesn't match any container elements, or it will replace the existing object. As with update, it is assumed that the provided object's attributes will replace all the existing object's ones.
      • kFLAG_PERSIST_DELETE: This option assumes you want to remove the object from the container, although the container features a specific method for this purpose, this option may be used to implement a deleted state, rather than actually removing the object from the container.

    Manage native node.

    Node(mixed $theValue, boolean $getOld) : mixed

    This method can be used to manage the native node reference, it uses the standard accessor method to manage the property:

    • $theValue: The value or operation:
      • NULL: Return the current value.
      • Everyman\Neo4j\PropertyContainer: Set value.
      • other: Raise exception.
    • $getOld: Determines what the method will return:
      • TRUE: Return the value before it was eventually modified.
      • FALSE: Return the value after it was eventually modified.

    Instantiate class.

    __construct(mixed $theNode, mixed $theContainer) 

    The constructor can be used to instantiate an object with a node, there is no provision for instantiating an empty object.

    The method expects the following parameters:

    • $theNode: The referenced node, this data can come in three flavours:
      • Neo4j node: This is the expected format, the node must be persistent, that is, it must have an ID. In this case the second parameter is ignored.
      • integer or numeric string: The value refers to the node ID, in this case the node will have to be loaded from a Neo4j client, which must be provided in the next parameter.
      • query: In this case we assume the value represents a query that will be sent to the index, only the first matching record will be used.
    • $theContainer: The nodes container, depending on the first parameter:
      • Neo4j node: The parameter will be ignored.
      • integer or numeric string: The parameter is expected to be an instance of a Neo4j client, it will be used to locate the node in the graph.
      • array: The parameter is expected to have two elements:
        • kTAG_NODE: The Neo4j client (graph container).
        • kTAG_TERM: The Mongo container (index container), which can take the following types:
          • MongoDB: The database will be used to instantiate a MongoCollection named as the kDEFAULT_CNT_NODES tag.
          • MongoCollection: The provided collection will be used.
          • CMongoContainer: The referenced Mongo collection will be used (be careful not to send the terms collection).

    If any of the above conditions is not met, or if the node was not found, an exception will be raised.

    Locate node in graph.

    _LocateNode(mixed $theNode, mixed $theContainer) : \Everyman\Neo4j\Node | NULL

    This method can be used to locate a node in the provided graph container, it accepts two parameters:

    • $theNode: The node reference, it can be either an integer or a numeric string referring to the seeked node ID.
    • $theContainer: The graph container, it must be a Neo4j client.

    The method will return the located node or NULL if not found.

    If the located node has an ID of zero, meaning that it is the default Neo4j node, the method will raise an exception, because most likely the node parameter was a string that resolved to zero.

    If any of the provided parameters is incorrect, the method will raise an exception.

    Resolve index container.

    _ResolveIndexContainer(mixed $theContainer) : \CMongoContainer

    This method can be used to resolve the provided parameter to the index container, the method will return a CMongoContainer which points to the index collection; the provided parameter can take the following types:

    • MongoDB: The method will return a container using a MongoCollection named after the value of the kDEFAULT_CNT_NODES tag.
    • MongoCollection: The method will return a container instantiated with the collection.
    • CMongoContainer: The method will use that value.

    If the provided parameter is of any other type, the method will raise an exception.
