    FAO/WIEWS institute.

    This class overloads its ancestor to implement an institute entity taken from the FAO WIEWS institutes database.

    This kind of institutes adds the following properties:

    • kENTITY_INST_FAO_EPACRONYM: This offset represents the institute ECPGR acronym.
    • kENTITY_INST_FAO_TYPE: This offset represents the institute set of FAO institute types.
    • kOFFSET_URL: This offset represents the institute URL.
    • kOFFSET_kENTITY_INST_FAO_LATLATITUDE: This offset represents the institute latitude, note this is an integer value.
    • kENTITY_INST_FAO_LON: This offset represents the institute longitude, note this is an integer value.
    • kENTITY_INST_FAO_ALT: This offset represents the institute altitude, note this is an integer value.

    Unlike its parent, this class will use the institute code as-is for the object unique identifier, the index hashing method will not transform the index since it is at most 7 characters thus cannot conflict with other entity identifiers.

    FAOType(mixed $theValue, mixed $theOperation, boolean $getOld) : mixed

    This method can be used to handle the institute FAO/WIEWS types list, it uses the standard accessor method to manage the list of acronyms.

    Each element of this list should indicate an acronym by which one refers to the current institute, the nature and specifics of these elements is the responsibility of concrete classes.

    Import institutes.

    Import(\CContainer $theContainer, string $theURL) : integer

    This method can be used to import the institutes list from the current FAO/WIEWS export file, the method expects two parameters:

    • $theContainer: The container in which the institutes are stored.
    • $theURL: The download URL of the FAO/WIEWS export file.

    The method will return the number of records added and replaced.

    Note that the method will commit the records using the kFLAG_PERSIST_REPLACE and kFLAG_STATE_ENCODED flags.
