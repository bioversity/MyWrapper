    Attribute class.

    This class is a static methods repository which is used to manage CArrayObject derived classe's attributes.

    The class is a collection of static methods which represent member accessor methods for objects that store their properties as array elements. These methods will handle insertion, extraction and deletion of attributes in either arrays or ArrayObject instances, these methods will then be used by CArrayObject derived classes as the base for their member accessor methods.

    The class features the following methods:

    • ManageOffset: Manage a scalar offset.
    • ManageArrayOffset: Manage elements of an array offset.
    • ManageTypedOffset: Manage elements of an array offset by category or type.
    • ManageTypedArrayOffset: Manage elements of an array offset by category or type and data element value.

    Hash closure.

    HashClosure() : \closure

    This static method returns the default hashing closure that will be used to compare list elements. This method will return a closure function that expects two arguments:

    • $theElement: This parameter represents the element to be hashed, it can take two forms depending on the presence or not of the next parameter:
      • Next parameter is not there: If you omit the next parameter, it means that the element belongs to a list of scalars and that it is the element itself to represent the key. In this case, the function will return the MD5 hash of the element itself.
      • Next parameter is there: If you provide the next parameter, it means that the provided offset or offsets point to the key data items contained in this parameter: in this case this parameter must be either an array or an ArrayObject, if you provide anything else, an exception will be raised. The hash will be generated by an MD5 hash of the following sequence: [Offset] kTOKEN_NAMESPACE_SEPARATOR [MD5( Value )], in which the value is the value referenced by the offset or NULL; if there are more than one offset, these blocks will be separated by the kTOKEN_INDEX_SEPARATOR token, the function will return the MD5 of the full sequence.
    • $theOffsets: If provided, it must be an array of strings that represent offsets to items in the previous parameter which constitute the element's key. Note that these offsets will be sorted prior to being used, to allow consistent results.

    When creating your custom closures, you should use this function as a model and be aware of the following rule: the static methods in this class invoke a single hash closure, but they might use this closure on both lists and matrices, so you should design the function as two separate functions depending on the presence or not of the offsets list.

    Manage an array offset.

    ManageArrayOffset(\reference $theReference, string $theOffset, mixed $theValue, mixed $theOperation, boolean $getOld, \closure $theClosure) : mixed

    An array offset is an array property, this method can be used to set, retrieve and delete the elements of this property, as opposed to ManageOffset, which is used to manage the property as a whole.

    The elements of this list are uniquely identified by a closure function which is either passed to this method or defaults to the HashClosure function.

    When deleting elements, if the list becomes empty, the whole offset will be deleted.

    ManageObjectList(\reference $theReference, string $theMainOffset, string $theTypeOffset, string $theDataOffset, mixed $theValue, mixed $theOperation, boolean $getOld) : mixed

    This method can be used to manage a list of object references, in which each element is either:

    • Scalar: A scalar or object representing:
      • The object: The actual referenced object.
      • The object reference: An object reference structure or a scalar representing the object's identifier.
      or:
    • Array: A structure composed of two items:
      • Kind: This offset represents the type or predicate of the reference.
      • Data: This offset represents the actual object or object reference.

    The reference list is numerically indexed array and this method will ensure it remains so.

    Manage a scalar offset.

    ManageOffset(\reference $theReference, string $theOffset, mixed $theValue, boolean $getOld) : mixed

    This method can be used to manage a scalar offset, its options involve setting, retrieving and deleting an offset of the provided array or ArrayObject.

    The method accepts the following parameters:

    • &$theReference: Reference to the attributes container, it may either refer to an array or an ArrayObject, any other type will trigger an exception.
    • $theOffset: The offset to the attribute contained in the previous parameter that is to be managed.
    • $theValue: The value or operation:
      • NULL: Return the offset's current value.
      • FALSE: Delete the offset.
      • other: Any other type represents the offset's new value.
    • $getOld: Determines what the method will return:
      • TRUE: Return the value of the offset before it was eventually modified.
      • FALSE: Return the value of the offset after it was eventually modified.

    Manage a typed array offset.

    ManageTypedArrayOffset(\reference $theReference, string $theMainOffset, string $theDataOffset, array $theTypeOffsets, array $theTypeValues, mixed $theData, mixed $theOperation, boolean $getOld, \closure $theClosure) : mixed

    The structures managed by this method are equivalent to the structures managed by the ManageTypedOffset method, except that the data item, in this case, is an array and the goal of this method is to manage the data item array elements rather than the data item itself.

    This method features many of the same parameters than the ManageTypedOffset method except for a couple of changes.

    Here is the list of arguments:

    • this parameter is one of those that have a different usage than in the ManageTypedOffset method:
      • array: An array is considered in two different ways, depending on the type of the $theDataOffset parameter:
        • scalar: In this case the array is considered the data value.
        • array: In this case the array is considered as the list of values to be managed: each element of this array will be identified by the corresponding element of the $theDataOffset array parameter.
      • NULL: This indicates that the operation, provided in the next parameter, applies to the whole data item list; this also means that if the operation is TRUE we raise here an exception, because one cannot add a NULL data item.
      • other: Any other value represents either the index of the data item to be retrieved or deleted, or the data item to be added. Note that if the value is an array, the array elements will be added individually, rather than setting the data to the actual array.
    • $theOperation: This parameter represents the operation to be performed, it will be evaluated as a boolean and its scope depends on the value of the previous parameter:
      • NULL: Return the element's data item or list.
      • FALSE: Delete the element's data item or list.
      • TRUE: Add or replace an element's data item or list.
      • array: This type is only considered if the $theDataOffset parameter is provided as an array: the method will be called for each element of the $theDataOffset parameter matched with the corresponding element of this parameter. This also implies that both parameters must share the same count.
    • $getOld: Determines what the method will return:
      • TRUE: Return the data item before it was eventually modified.
      • FALSE: Return the data item after it was eventually modified.
    • $theClosure: The hashing closure, if omitted, the default closure function will be used, for more details on how to create a custom closure please see the HashClosure reference in this class. If the $theDataOffset was provided as an array, this parameter may also be provided as a list of closures matching the $theDataOffset parameter count.

    Manage a typed offset.

    ManageTypedOffset(\reference $theReference, string $theMainOffset, string $theDataOffset, array $theTypeOffsets, array $theTypeValues, mixed $theData, boolean $getOld, \closure $theClosure) : mixed

    A typed offset is a list element that has at least an item containing the element data and a variable number of other items that determine the type or kind of the element: these latter items also determine the element's key and are used to retrieve, delete and create elements, no two elements may share the same combination of these items. Elements of this type must be arrays, ArrayObject instances will be converted to arrays.
