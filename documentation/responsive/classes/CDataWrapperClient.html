    Data wrapper client.

    This class represents a web-services data wrapper client, it facilitates the job of requesting information from servers derived from the CDataWrapper class.

    This class adds the following properties to its ancestor:

    • Database: This element represents the web-service database, it is stored in the kAPI_DATABASE offset.
    • Container: This element represents the web-service database container, it is stored in the kAPI_CONTAINER offset.
    • Page start: This element represents the start page requested from the web-service, it is stored in the kAPI_PAGE_START offset.
    • Page limit: This element represents the max page count requested from the web-service, it is stored in the kAPI_PAGE_LIMIT offset.
    • Query: This element represents the query that will be sent to the web-service, it is stored in the kAPI_DATA_QUERY offset.
    • Fields: This element represents the list of fields that should be returned by the web-service, it is stored in the kAPI_DATA_FIELD offset.
    • Sort: This element represents the list of fields that will be used to sort the reqults returned by the web-service, it is stored in the kAPI_DATA_SORT offset.
    • Object: This element represents the object to be sent to the web-service, it is stored in the kAPI_DATA_OBJECT offset.
    • Options: This element represents a list of options sent to the web-service, it is stored in the kAPI_DATA_OPTIONS offset.

    Manage operation.

    Operation(string $theValue, boolean $getOld) : mixed

    We overload this method to add the following allowed operations:

    • kAPI_OP_COUNT: COUNT web-service operation, used to return the total number of elements satisfying a query.
    • kAPI_OP_MATCH: This operation is equivalent to a read query, except that it will try to match one query clause at the time and will return a result on the first match.
    • kAPI_OP_GET: GET web-service operation, used to retrieve objects from the data store.
    • kAPI_OP_SET: SET web-service operation, used to insert or update objects in the data store.
    • kAPI_OP_UPDATE: UPDATE web-service operation, used to update existing objects in the data store.
    • kAPI_OP_INSERT: INSERT web-service operation, used to insert new objects in the data store.
    • kAPI_OP_BATCH_INSERT: This service is equivalent to the kAPI_OP_INSERT command, except that in this case you provide a list ov objects to insert.
    • kAPI_OP_MODIFY: MODIFY web-service operation, used to modify partial contents of objects in the data store.
    • kAPI_OP_DEL: DELETE web-service operation, used to delete objects from the data store.

    Manage options.

    Options(mixed $theOption, mixed $theValue, boolean $getOld) : mixed

    This method can be used to manage the query data options, with it you can add, retrieve and delete elements of the options list, to operate on the whole list you should use its offset.

    The method accepts the following parameters:

    • $theOption: This parameter represents the option, it can take the following values:
      • kAPI_OPT_SAFE: Can be a boolean or integer, defaults to FALSE. If FALSE, a database query will not wait for a response. If TRUE, the program will wait for the database response and throw an exception if the operation did not succeed.
      • kAPI_OPT_FSYNC: Boolean, defaults to FALSE. Forces the update to be synced to disk before returning success. If TRUE, a safe update is implied and will override setting safe to FALSE.
      • kAPI_OPT_TIMEOUT: Integer, if "safe" is set, this sets how long (in milliseconds) for the client to wait for a database response. If the database does not respond within the timeout period, an exception will be thrown.
      • kAPI_OPT_SINGLE: Boolean, used in the delete or update operatiosn: if TRUE, only the first object will be affected; if not, all matching objects will be considered.
    • $theValue: This parameter represents either the option value, or the operation to perform:
      • NULL: This value indicates that we want to retrieve the value of the option provided in the previous parameter.
      • FALSE: This value indicates that we want to delete the option provided in the previous parameter.
      • other: Any other value is interpreted as the value of the option selected by the first parameter. Note that, in general, missing options are equivalent to setting the option to FALSE, when the value is a boolean, so turning off an option is the same as removing it.
    • $getOld: Determines what the method will return:
      • TRUE: Return the element or list before it was eventually modified.
      • FALSE: Return the element or list after it was eventually modified.
