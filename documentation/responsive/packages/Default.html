      Autoloader.

      Neo4jAutoload(string $theClassName) 

      This section allows automatic inclusion of the library classes.

      Parameters

      $theClassName

      string

      name of class to load.

      Constants

       

      Count tag.

      kAPI_AFFECTED_COUNT 

      This tag will hold the total number of elements affected by the operation.

       

      Web-service database container.

      kAPI_CONTAINER 

      This is the tag that represents the database container on which we want to operate.

      Cardinality: one.

       

      Web-service database.

      kAPI_DATABASE 

      This is the tag that represents the database on which we want to operate.

      Cardinality: one.

       

      Data store object fields.

      kAPI_DATA_FIELD 

      This is the tag that represents the data store object elements that should be returned: if omitted it is assumed that the whole object is to be returned.

      Cardinality: one or zero.

       

      Data store object.

      kAPI_DATA_OBJECT 

      This is the tag that represents the data store object, this value is used when committing data back to the data store.

      Cardinality: one or zero.

       

      Data store options.

      kAPI_DATA_OPTIONS 

      This is the tag that represents the data store options, this value is used to provide additional options to the operation. It is structured as a key/value pair having the following default key elements:

      • kAPI_OPT_SAFE: Safe commit option.
      • kAPI_OPT_FSYNC: Safe and sync commit option.
      • kAPI_OPT_TIMEOUT: Operation timeout.
      • kAPI_OPT_SINGLE: Single object selection.

      Cardinality: one or zero.

       

      Paging.

      kAPI_DATA_PAGING 

      This offset represents the collector tag for request paging parameters.

      Note: this is a reserved offset tag.

       

      Data store filter.

      kAPI_DATA_QUERY 

      This is the tag that represents the data store filter or query.

      Cardinality: one or zero.

       

      Request.

      kAPI_DATA_REQUEST 

      This tag holds the request block.

       

      Response.

      kAPI_DATA_RESPONSE 

      This tag holds the response block.

       

      Data store sort order.

      kAPI_DATA_SORT 

      This is the tag that represents the data store sort elements that should be used for sorting the results.

      Cardinality: one or zero.

       

      Status.

      kAPI_DATA_STATUS 

      This tag holds the operation status block.

       

      Timing.

      kAPI_DATA_TIMING 

      This offset represents the collector tag for request timing parameters.

      Note: this is a reserved offset tag.

       

      Incoming and outgoing.

      kAPI_DIRECTION_ALL 

      This tag indicates incoming and outgoing relationships, in other words, all elements that are related to the current object, both pointing to it and pointed by it.

       

      Incoming.

      kAPI_DIRECTION_IN 

      This tag indicates an incoming relationship, in other words, all elements that point to the current object.

       

      Outgoing.

      kAPI_DIRECTION_OUT 

      This tag indicates an outgoing relationship, in other words, all elements to which the current object points to.

       

      Web-service format.

      kAPI_FORMAT 

      This is the tag that represents the web-service request and response format.

      This applies both to the request and the response; if the parameter is not provided we assume the JSON format.

      Cardinality: one.

       

      Web-service operation.

      kAPI_OPERATION 

      This is the tag that represents the web-service operation.

      Cardinality: one.

       

      Relationship direction.

      kAPI_OPT_DIRECTION 

      This option is used when retrieving edges: it indicates the direction of the relationships in regard to the node identifiers provided in the kAPI_OPT_IDENTIFIERS parameter:

      • kAPI_DIRECTION_IN: Incoming relationships, this will select all elements that point to the objects provided in the kAPI_OPT_IDENTIFIERS parameter.
      • kAPI_DIRECTION_OUT: Outgoing relationships, this will select all elements pointed to by the objects provided in the kAPI_OPT_IDENTIFIERS parameter.
      • kAPI_DIRECTION_ALL: All relationships, this will select all elements both pointing to and pointed to by the objects provided in the kAPI_OPT_IDENTIFIERS parameter.

       

      FSYNC option.

      kAPI_OPT_FSYNC 

      Boolean, defaults to FALSE. Forces the update to be synced to disk before returning success. If TRUE, a safe update is implied and will override setting safe to FALSE.

       

      Identifiers option.

      kAPI_OPT_IDENTIFIERS 

      This option refers to a list of object identifiers, this option is used by assorted operations to receive the list of objects to be retrieved; the type of the list's elements is determined by the operation.

       

      Relationship levels.

      kAPI_OPT_LEVELS 

      This option is used when retrieving for relationships: it indicates the amount of levels to follow.

      If the integer parameter is omitted, the service will force a one level step, if the parameter is negative, it means that the service will continue until all levels have been reached.

       

      Log reauest.

      kAPI_OPT_LOG_REQUEST 

      If the value of this tag resolves to TRUE, the request will be logged in the response.

      Cardinality: one or zero.

       

      Trace errors.

      kAPI_OPT_LOG_TRACE 

      If the value of this tag resolves to TRUE, eventual errors will bare the trace.

      Cardinality: one or zero.

       

      No response option.

      kAPI_OPT_NO_RESP 

      Can be a boolean or integer, defaults to FALSE. If TRUE, the response section will not be included in the result. This can be useful when you are only interested in the status of the operation and not in the response.

       

      Predicates option.

      kAPI_OPT_PREDICATES 

      This option refers to a list of predicate term identifiers, this option is used by operations involving the selection of edge nodes: only those edge nodes referring to any of the provided predicate term identifiers will be selected.

      Note that this parameter depends on the kAPI_OPT_PREDICATES_INC parameter: if the latter is 1, it means that only edges whose predicate is among this list will be selected; if the value is 0, only edges whose predicate is not among this list will be selected.

       

      Predicates inclusion option.

      kAPI_OPT_PREDICATES_INC 

      This option refers to the list of predicates: it is a boolean (1/0) flag that determines whether the predicates list is to be considered as inclusive or exclusive. If omitted it defaults to 1 (inclusion).

       

      SAFE option.

      kAPI_OPT_SAFE 

      Can be a boolean or integer, defaults to FALSE. If FALSE, the program continues executing without waiting for a database response. If TRUE, the program will wait for the database response and throw an exception if the operation did not succeed.

       

      SINGLE option.

      kAPI_OPT_SINGLE 

      Boolean, used in the delete operation: if TRUE, only one object will be deleted; if not, all matching objects will be deleted.

      Get relationships web-service.

      kAPI_OP_GET_RELS 

      This is the tag that represents the get relationships web service, it will locate all edge nodes related to the node identifiers provided in the kAPI_OPT_IDENTIFIERS parameter, following the direction provided in the kAPI_OPT_DIRECTION parameter.

      Depending on the value of the kAPI_OPT_DIRECTION parameter:

      • kAPI_DIRECTION_IN: The service will return all edges that point to the nodes provided in the kAPI_OPT_IDENTIFIERS parameter.
      • kAPI_DIRECTION_OUT: The service will return all edges pointing from the nodes provided in the kAPI_OPT_IDENTIFIERS parameter.
      • kAPI_DIRECTION_ALL: The service will return all edges connected in any way to the nodes provided in the kAPI_OPT_IDENTIFIERS parameter.
      • kAPI_OPT_DIRECTION parameter not provided: In this case the service will assume that the identifiers provided in the kAPI_OPT_IDENTIFIERS parameter are edge node IDs, and it will simply return the matching edges.

      The service also expects a kAPI_OPT_LEVELS parameter, a signed integer, that indicates how many levels to recurse the graph traversal, if this parameter is not provided, it will default to 1 level; to traverse all levels this parameter should be set to a negative number; a level of 0 will only return the list of involved nodes and terms.

      The service will return the same structure as the kAPI_OP_GET_EDGES service:

      • kAPI_RESPONSE_EDGES: The list of edges as an array structured as follows:
        • key: The edge identifier.
        • Value: An array structured as follows:
          • kAPI_RESPONSE_SUBJECT: The subject node ID.
          • kAPI_RESPONSE_PREDICATE: The predicate term identifier.
          • kAPI_RESPONSE_OBJECT: The object node ID.
      • kAPI_RESPONSE_NODES: The list of subject and object found nodes as follows:
        • Key: The node ID.
        • Value: The node properties.
      • kAPI_RESPONSE_TERMS: The list of terms related to the list of found nodes and to the edge predicate as follows:
        • Key: The term global identifier.
        • Value: The contents of the term.

      If you provide the kAPI_OPT_PREDICATES parameter, only those edges whose type matches any of the predicate term identifiers provided in that parameter will be selected.

      If you omit the kAPI_OPT_IDENTIFIERS parameter, no elements will be returned. The service does not use paging options.

      The record count refers to the edges count.

      Note that the container is not required, if omitted it will be initialised to the kDEFAULT_CNT_TERMS constant.

      Set tags web-service.

      kAPI_OP_SET_TAGS 

      This is the tag that represents the set tags web service, it expects the kAPI_OPT_IDENTIFIERS parameter to hold a list of elements that can take the following forms:

      • Scalar: A scalar is expected if the annotation element is a single node being both a trait and a measure, in this case the scalar should be the term identifier.
      • Array: An array is expected if the tag is comprised of a chain of terms.
        • Terms: The list of terms must be represented by an array of term identifiers in which the number of elements must be odd, the odd elements represent node term references and the even elements represent edge predicate terms.
        • Edges: The list of edges must be represented as an array of integers representing edge identifiers. This edge sequence will ultimately be transformed in a chain of terms that will be matched in the database.

      The service will check if the provided chain of terms exists, in that case it will return the found record; if not, it will create a new record.

      Note that this service will instantiate term paths, so call this service only if you are sure you need to do it; if any error occurs, the operation will be aborted (but the eventual created annotations will not).

      The service will return the result in the same format as the kAPI_OP_GET_TAGS service.
