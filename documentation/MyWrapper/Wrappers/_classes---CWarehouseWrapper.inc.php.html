Identifiers option.

This option refers to a list of object identifiers, this option is used by assorted operations to receive the list of objects to be retrieved; the type of the list's elements is determined by the operation.

kAPI_OPT_NODE_SELECTORS = ':@node-selectors' (line 156)

Attribute selectors option.

This option refers to a list of attributes and matching values, this option is used by the nodes query operation. The contents are an array indexed by the node attributes whose value is an array of matching values. The resulting query will be composed in AND mode.

kAPI_OP_GET_NODES = '@GET_NODES' (line 74)

Get nodes web-service.

This is the tag that represents the get nodes web service, it will locate all nodes matching the provided identifiers in the kAPI_OPT_IDENTIFIERS parameter and return the following structure:

  • kAPI_RESPONSE_TERMS: The list of terms related to the list of nodes as follows:
    • Index: The term identifier.
    • Value: The term properties.
  • kAPI_RESPONSE_NODES: The list of nodes as follows:
    • Index: The node ID.
    • Value: The node properties.

If you omit the kAPI_OPT_IDENTIFIERS parameter, no elements will be returned. The service does not use paging options.

kAPI_OP_GET_TERMS = '@GET_TERMS' (line 46)

Get terms web-service.

This is the tag that represents the get terms web service, it will locate all terms matching the provided identifiers in the kAPI_OPT_IDENTIFIERS parameter and return an array whose key is the provided identifier and as value the matched object or NULL. If you omit the kAPI_OPT_IDENTIFIERS parameter it mis assumed that you want all terms, in that case the service will enforce the use of paging options.

kAPI_OP_INCOMING_NODES = '@NODES_IN' (line 117)

Get incoming relations web-service.

This is the tag that represents the get incoming relations web service, it will locate all nodes that point to the nodes provided in the kAPI_OPT_IDENTIFIERS parameter having as predicate the terms listed in the kAPI_OPT_PREDICATES parameter. This service is equivalent to requesting all child nodes of the nodes provided in the kAPI_OPT_IDENTIFIERS parameter. If you omit the kAPI_OPT_PREDICATES parameter it is assumed that all predicates will be considered.

kAPI_OP_LOGIN = '@LOGIN' (line 33)

LOGIN web-service.

kAPI_OP_QUERY_ONTOLOGIES = '@QUERY_ONTOLOGIES' (line 103)

Query ontologies web-service.

This is the tag that represents the query ontologies web service, it will locate all ontology nodes matching the provided attributes in the kAPI_OPT_NODE_SELECTORS parameter and return the following structure:

  • kAPI_RESPONSE_TERMS: The list of terms related to the list of ontologies as follows:
    • Index: The term identifier.
    • Value: The term properties.
  • kAPI_RESPONSE_NODES: The list of ontologies as follows:
    • Index: The node ID.
    • Value: The node properties.

If you omit the kAPI_OPT_NODE_SELECTORS parameter, all nodes with ontology kind will be returned.
