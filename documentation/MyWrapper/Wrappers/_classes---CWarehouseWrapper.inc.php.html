kAPI_OPT_ATTRIBUTES = ':@attributes' (line 270)

Attribute selectors option.

This option is used by operations that query nodes, its content is an array structured as follows:

  • Index: The array element index is the attribute key.
  • Value: The value is an array of attribute values to be matched.

The resulting query will be composed in AND mode.

kAPI_OPT_DIRECTION = ':@direction' (line 291)

Relationship direction.

This option is used when retrieving edges: it indicates the direction of the relationships in regard to the node identifiers provided in the kAPI_OPT_IDENTIFIERS parameter:

  • kAPI_DIRECTION_IN: Incoming relationships, this will select all elements that point to the objects provided in the kAPI_OPT_IDENTIFIERS parameter.
  • kAPI_DIRECTION_OUT: Outgoing relationships, this will select all elements pointed to by the objects provided in the kAPI_OPT_IDENTIFIERS parameter.
  • kAPI_DIRECTION_ALL: All relationships, this will select all elements both pointing to and pointed to by the objects provided in the kAPI_OPT_IDENTIFIERS parameter.

kAPI_OP_GET_EDGES = '@GET_EDGES' (line 144)

Get edges web-service.

This is the tag that represents the get edges web service, it will locate all edges matching the provided identifiers in the kAPI_OPT_IDENTIFIERS parameter and the direction provided in the kAPI_OPT_DIRECTION parameter.

Depending on the presence or not of the kAPI_OPT_DIRECTION parameter:

  • kAPI_OPT_DIRECTION not provided: In this case we assume that the kAPI_OPT_IDENTIFIERS parameter contains the list of edge identifiers to match.
  • kAPI_OPT_DIRECTION provided: In this case we assume that the kAPI_OPT_IDENTIFIERS parameter contains the list of node identifiers for which we want to retrieve connected edges in the direction provided in the kAPI_OPT_DIRECTION parameter:
    • kAPI_DIRECTION_IN: The service will return all edges that point to the nodes provided in the kAPI_OPT_IDENTIFIERS parameter.
    • kAPI_DIRECTION_OUT: The service will return all edges pointing from the nodes provided in the kAPI_OPT_IDENTIFIERS parameter.
    • kAPI_DIRECTION_ALL: The service will return all edges connected in any way to the nodes provided in the kAPI_OPT_IDENTIFIERS parameter.

The service will return the following structure:

  • kAPI_RESPONSE_TERMS: The list of terms related to the list of subject and object nodes and the list of predicate terms as follows:
    • Index: The term identifier.
    • Value: The term properties.
  • kAPI_RESPONSE_NODES: The list of subject and object nodes as follows:
    • Index: The node ID.
    • Value: The node properties.
  • kAPI_RESPONSE_EDGES: The list of edges as an array of elements structured as follows:
    • kAPI_RESPONSE_SUBJECT: The subject node ID.
    • kAPI_RESPONSE_PREDICATE: The predicate term identifier.
    • kAPI_RESPONSE_OBJECT: The object node ID.

If you provide the kAPI_OPT_PREDICATES parameter, only those edges whose type matches any of the predicate term identifiers provided in that parameter will be selected.

If you omit the kAPI_OPT_IDENTIFIERS parameter, no elements will be returned. The service does not use paging options.

kAPI_OP_GET_RELS = '@GET_RELS' (line 187)

Get relations web-service.

This is the tag that represents the get relations web service, it will locate all edges related to the provided identifiers in the kAPI_OPT_IDENTIFIERS parameter, in the direction provided in the kAPI_OPT_DIRECTION parameter and for the kAPI_OPT_LEVELS levels.

The service expects the same parameters as the kAPI_OP_GET_EDGES service and returns a similar structure in which the only difference is that the kAPI_RESPONSE_EDGES element will be structured as follows:

  • kAPI_RESPONSE_EDGES: The list of edges will be an array structured as follows:
    • Index: The node identifier provided in the kAPI_OPT_IDENTIFIERS parameter.
    • Value: An array of elements structured as follows:
      • kAPI_RESPONSE_SUBJECT: The subject node ID.
      • kAPI_RESPONSE_PREDICATE: The predicate term identifier.
      • kAPI_RESPONSE_OBJECT: The object node ID.

For more information consult the kAPI_OP_GET_EDGES command.

If the kAPI_OPT_DIRECTION parameteris not provided, the service will set it by default to kAPI_DIRECTION_OUT, which translates to traversing the graph following the subclass predicate towards the root.

If the kAPI_OPT_LEVELS parameter is not provided, it is assumed that no limit is set, which means care should be taken. A negative value means no limit.

kAPI_OP_GET_TERMS = '@GET_TERMS' (line 46)

Get terms web-service.

This is the tag that represents the get terms web service, it will locate all terms matching the provided identifiers in the kAPI_OPT_IDENTIFIERS parameter and return an array whose key is the provided identifier and as value the matched object or NULL. If you omit the kAPI_OPT_IDENTIFIERS parameter it mis assumed that you want all terms, in that case the service will enforce the use of paging options.

kAPI_OP_QUERY_ROOTS = '@QUERY_ROOTS' (line 216)

Query roots web-service.

This is the tag that represents the query roots web service, it will locate all root nodes matching the provided attributes in the kAPI_OPT_ATTRIBUTES parameter and return the following structure:

  • kAPI_RESPONSE_TERMS: The list of terms related to the list of root nodes as follows:
    • Index: The term identifier.
    • Value: The term properties.
  • kAPI_RESPONSE_NODES: The list of root nodes as follows:
    • Index: The node ID.
    • Value: The node properties.

If you omit the kAPI_OPT_ATTRIBUTES parameter, all nodes with root kind will be returned.
