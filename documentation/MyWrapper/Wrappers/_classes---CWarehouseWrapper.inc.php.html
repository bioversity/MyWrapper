kAPI_OPT_IDENTIFIERS = ':@identifiers' (line 97)

Identifiers option.

This option refers to a list of object identifiers, this option is used by assorted operations to receive the list of objects to be retrieved; the type of the list's elements is determined by the operation.

kAPI_OPT_NODE_SELECTORS = ':@node-selectors' (line 107)

Attribute selectors option.

This option refers to a list of attributes and matching values, this option is used by the nodes query operation. The contents are an array indexed by the node attributes whose value is an array of matching values. The resulting query will be composed in AND mode.

kAPI_OP_GET_NODES = '@GET_NODES' (line 56)

Get nodes web-service.

This is the tag that represents the get nodes web service, it will locate all nodes matching the provided identifiers in the kAPI_OPT_IDENTIFIERS parameter and return an array whose key is the node ID and as value is the merged attributes of the node's term and the node's properties, or NULL.

kAPI_OP_GET_TERMS = '@GET_TERMS' (line 44)

Get terms web-service.

This is the tag that represents the get terms web service, it will locate all terms matching the provided identifiers in the kAPI_OPT_IDENTIFIERS parameter and return an array whose key is the provided identifier (converted to a string) and as value the matched object or NULL.

kAPI_OP_LOGIN = '@LOGIN' (line 33)

LOGIN web-service.

This is the tag that represents the LOGIN operation, it will check for the user and password and match both with a user record.

kAPI_OP_QUERY_ONTOLOGIES = '@QUERY_ONTOLOGIES' (line 68)

Query ontologies web-service.

This is the tag that represents the query ontologies web service, it will locate all ontology nodes matching the provided attributes in the kAPI_OPT_NODE_SELECTORS parameter and return an array whose key will be the node ID and as value is the merged attributes of the node's term and the node's properties, or NULL.
