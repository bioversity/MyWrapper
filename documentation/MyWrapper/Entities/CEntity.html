Entity.

An entity can be an individual, and organisation or a legal entity that exists as a unit, rather than being embedded in another object.

In this class we add to the parent the following properties:

  • kTAG_NAME: This offset represents the entity name, the class features a member accessor method to manage this property.
  • kOFFSET_EMAIL: This offset represents the entity e-mail address, it is a scalar property and the class features a member accessor method to manage it.

The class also features a static method that returns the default container name for objects of this type.

Name (line 99)

Manage entity name.

This method can be used to handle the entity name, it uses the standard accessor method to manage the offset.

This value should be a string that can be used as a label or as a short definition of the entity. The name may be language dependent, so the type of data stored in this offset is the responsibility of concrete classes.

For a more in-depth reference of this method, please consult the _ManageOffset method, in which the first parameter will be the constant kTAG_NAME.
