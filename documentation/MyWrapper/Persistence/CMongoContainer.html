UnserialiseData (line 219)

Unserialise provided data element.

We implement this method to convert all standard types into custom Mongo data types.

In this class we parse the following types and offsets:

  • CDataTypeMongoId object or kTYPE_MongoId offset: We return a MongoId object.
  • CDataTypeMongoCode object or kTYPE_MongoCode offset: We return a MongoCode object.
  • CDataTypeStamp object or kTYPE_STAMP offset: We return a MongoDate object.
  • CDataTypeRegex object or kTYPE_REGEX offset: We return a MongoRegex object.
  • CDataTypeInt32 object or kTYPE_INT32 offset: We return a MongoInt32 object.
  • CDataTypeInt64 object or kTYPE_INT64 offset: We return a MongoInt64 object.
  • CDataTypeBinary object or kTYPE_BINARY offset: We return a MongoBinData object.

  • access: public

_Commit (line 346)

Commit provided object.

We implement this method to handle MongoCollection object stores, this method will store the object in the current container.

The method will check if the current container is a MongoCollection, if this is not the case, it will raise an exception.

If the provided modifiers indicate a modify operation, the method will return the modified object, in all other cases the method will return the object identifier.

  • see: kFLAG_PERSIST_REPLACE kFLAG_PERSIST_UPDATE
  • see: kFLAG_PERSIST_INSERT kFLAG_PERSIST_MODIFY

Load object.

We implement this method to handle MongoCollection object stores, this method will retrieve the object from the current container.

The caller will have resolved references andeventually extracted the identifier from the provided parameter.
