Default offsets.

This file contains the definitions of the default offsets or tags used by objects in this library, whenever choosing offsets for persistent objects, you should first make sure that they are not among those defined in this file.

kOFFSET_REFS = ':REFS' (line 329)

References tag.

This is the tag that represents the list of references of an object, it is an array of object references in which each element may either be the reference itself or the following structure:

  • kTAG_KIND: Relation predicate, it can either be an object reference or a string.
  • kTAG_DATA: Relation object, it will be a reference to an object in which the following elements may appear:
    • kTAG_REFERENCE_ID: The unique identifier of the referenced object.
    • kTAG_REFERENCE_CONTAINER: The container name.
    • kTAG_REFERENCE_DATABASE: The database name.
    • kTAG_CLASS: The object class name.

kTAG_ID = '_id' (line 40)

Local unique identifier offset.

This is the tag that represents the object's unique identifier, this offset should hold a scalar value which uniquely identifies the object within the collection that holds it.

This should not be confused with the unique global identifier, which represents the value or values used by the public to refer to that object.

This value should be tightly integrated with the database.

kTAG_VERSION = ':VERSION' (line 90)

Version tag.

This tag is an offset that should be used to represent the object's version, the version is a value that should change each time the object is saved: it can be used to check whether an object was modified since the last time it was read.

By default it is an integer incremented each time the object is saved.
