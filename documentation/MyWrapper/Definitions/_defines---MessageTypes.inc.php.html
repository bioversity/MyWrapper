kMESSAGE_TYPE_BUG = 60 (line 97)

Bug.

This code indicates a bug.

Bugs, as opposed to errors, result from internal causes independant from external factors. A bug indicates that an operation will never execute as stated, it does not necessarily mean that it is fatal, but rather that the behaviour of an operation does not correspond to its declaration.

kMESSAGE_TYPE_ERROR = 40 (line 74)

Error.

This code indicates an error.

Errors indicate that something prevented an operation from being performed, this does not necessarily mean that the whole process is halted, but that the results of an operation will not be as expected.

kMESSAGE_TYPE_FATAL = 50 (line 85)

Fatal.

This code indicates a fatal error.

Fatal errors are errors that result in stopping the whole process: in this case the error will prevent other operations from being performed and the whole process should be halted.

kMESSAGE_TYPE_IDLE = 0 (line 32)

OK.

This code indicates no errors.

This state can be equated to an idle state.

kMESSAGE_TYPE_MESSAGE = 20 (line 52)

Message.

This code indicates a message.

A message is an informative message that is addressed to somebody, although it does not imply an error or warning, it was issued to a receiving party.

kMESSAGE_TYPE_NOTICE = 10 (line 42)

Notice.

This code indicates a notice.

A notice is an informative message that does not imply an error, nor a situation that should be handled; it can be considered as statistical data.

kMESSAGE_TYPE_WARNING = 30 (line 63)

Warning.

This code indicates a warning.

Warnings are informative data that indicate a potential problem, although they do not imply an error, they indicate a potential problem or an issue that should be addressed at least at a later stage.
