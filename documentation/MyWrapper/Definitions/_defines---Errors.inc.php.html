kERROR_BAD_METHOD_CALL = -4 (line 61)

Bad method call.

This error indicates a call to a non existing method. In general this will happen when mapping functions to the class and the called function does not exist.

kERROR_DUPLICATE = -9 (line 103)

Duplicate.

This error is raised when a duplicate element is found in a collection which is supposed to have unique elements.

In general this is the error that will be raised when inserting a duplicate element in a container.

kERROR_INVALID_PARAMETER = -2 (line 46)

Invalid parameter.

This error indicates that an invalid parameter was passed to a method or function.

kERROR_INVALID_STATE = -1 (line 39)

Invalid state.

This error indicates that the current state is invalid or not fit in order to perform the requested information.

kERROR_NOT_FOUND = -8 (line 92)

Not found.

This error is raised when an expected object is missing from a container.

kERROR_NOT_IMPLEMENTED = -10 (line 115)

Unimplemented.

This error is raised when the requested operation is not yet implemented. This is an indication that there is the intention of implementing the requested feature, but that this was not yet done.

This error is raised by operations rather than by data; in the latter case you would most likely get an unsupported error.

kERROR_NOT_INITED = -5 (line 69)

Object not ready.

This error indicates the attempt to operate with an object that has not yet been properly initialised.

kERROR_OK = 0 (line 31)

This code indicates successful operation or idle state.

kERROR_OPTION_MISSING = -3 (line 53)

Missing option.

kERROR_PROTECTED = -7 (line 85)

Protected.

This error indicates the attempt to modify a protected attribute, this usually occurs when modifying a significative offset of an object that uses it as an identifier.

kERROR_UNSUPPORTED = -6 (line 77)

Unsupported.

This error indicates the attempt to either perform an unsupported operation, or that an unsupported data type was provided to a method.
