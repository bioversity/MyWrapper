Term.

A term represents a concept identified by a pair of elements: the namespace and the Code(), the namespace groups all Code() belonging to a specific category, the Code() identifies the term within the namespace.

Terms are used in ontologies and to tag data elements, We add to the CGraphUnitObject the following properties:

  • kTAG_NAMESPACE: This attribute contains the namespace of the current term; note that this string is the identifier of the namespace term.
  • kTAG_NAME: This attribute represents the current term's name.
  • kTAG_DESCRIPTION: This attribute represents the current term's description or definition.

The unique identifier of instances from this class are formed by hashing the combination of the namespace and @link Code() code}, separated by a separator token; if the term has no namespace, the token is omitted.

Definition (line 221)

Manage term definition.

This method can be used to manage the term definition, it manages an array of structures with the following offsets:

  • kTAG_LANGUAGE: The definition's language, this element represents the code of the language in which the next element is expressed in.
  • kTAG_DATA: The term definition or description.

The parameters to this method are:

  • $theValue: The definition or operation:
    • NULL: Return the current value selected by the second parameter.
    • FALSE: Delete the value selected by the second parameter.
    • other: Set value selected by the second parameter.
  • $theLanguage: The definition's language code:
    • NULL: This value indicates that the definition has no language, in general, when adding elements, this case applies to default elements.
    • other: All other types will be interpreted as the language code.
  • $getOld: Determines what the method will return:
    • TRUE: Return the value before it was eventually modified.
    • FALSE: Return the value after it was eventually modified.

  • see: kTAG_DEFINITION kTAG_LANGUAGE

NS (line 107)

Manage term namespace.

This method can be used to handle the term namespace, it uses the standard accessor _ManageOffset() to manage the offset.

The namespace represents the global identifier of a term which represents the current term's namespace, which groups terms according to a specific category. This property and the current term's Code() represent the term's unique identifier.

For a more in-depth reference of this method, please consult the _ManageOffset() method, in which the first parameter will be the constant kTAG_CODE.

  • access: public
  • uses: _ManageOffset
