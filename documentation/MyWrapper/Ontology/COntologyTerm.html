Generic term.

This kind of term represents a generic ontology term.

This class handles the inited status by checking the following values in its kinds list:

  • Default: By default the object must have its code set.
  • kTYPE_NAMESPACE: It uses the default requirements.
  • kTYPE_ONTOLOGY: It requires the name to be set.
  • kTYPE_PREDICATE: It requires the name.
  • kTYPE_ATTRIBUTE: It requires the name and cardinality.
  • kTYPE_MEASURE: It requires the name and type.
  • kTYPE_ENUMERATION: It requires the name and enumeration.

In this class we ensure that the term has both the code and name in order to have an inited status.

Cardinality (line 351)

Manage data cardinality.

This method can be used to manage the measure data type, it uses the standard accessor method to manage the offset:

  • $theValue: The value or operation:
    • NULL: Return the current value.
    • FALSE: Delete the value.
    • string: String values represent the cardinality enumeration:
      • kCARD_0_1: Zero or one, a scalar or none.
      • kCARD_1: Exactly one, a required scalar.
      • kCARD_ANY: Any, this implies that we either have an array or no data.
    • integer: An integer represents the exact cardinality, in that case we assume the data is an array of at most that number of elements.
  • $getOld: Determines what the method will return:
    • TRUE: Return the value before it was eventually modified.
    • FALSE: Return the value after it was eventually modified.

  • access: public
