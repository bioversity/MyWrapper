Ontology graph node.

This class implements an ontology graph root node.

The class is derived from CGraphNode, it adds another required property, the term. This class will wrap the array access framework to the combination of the node properties and the term elements, except that the term elements will be read-only.

This class introduces a new kind of container: it must be an array of two elements structured as follows:

  • kTAG_NODE: This element should hold the nodes container, it must be a Everyman\Neo4j\Client instance.
  • kTAG_TERM: This element should hold the terms container, it must be a CContainer instance.

Note that the class will not cast to an array correctly, you must use the getArrayCopy method to get an array, if you know how to solve this, please do it!

Term (line 177)

Manage node term.

This method can be used to manage the node term reference, it uses the standard accessor method to manage the property:

  • $theValue: The value or operation:
    • NULL: Return the current value.
    • FALSE: Delete the value.
    • COntologyTermObject: Set value.
    • other: Raise exception.
  • $getOld: Determines what the method will return:
    • TRUE: Return the value before it was eventually modified.
    • FALSE: Return the value after it was eventually modified.

The method will also set the dirty status and the inited status if the node is provided.

_IndexTerms (line 865)

Create node/term indexes.

This method will save node indexes after the node was committed, there are two main indexes for nodes:

  • kINDEX_NODE_TERM: This index (NodeIndex) links the node to its term through the following keys:
    • kTAG_TERM: This key represents the term identifier.
    • kTAG_NAME: This key represents the term names in all languages.
  • kINDEX_NODE_TERM_GEN: This index (NodeFulltextIndex) links the node to its term through the following keys:
    • kPROP_REF_TERM_WORD: This key represents the term words found in the following properties:
      • kTAG_DEFINITION: Term definitions in all languages.

_PrepareCreate (line 508)

Normalise parameters of a create.

In this class we first check whether the container has the following structure:

  • kTAG_NODE: This element should hold the nodes container, it must be a Everyman\Neo4j\Client instance.
  • kTAG_TERM: This element should hold the terms container, it must be a COntologyTermObject instance.

We then call the parent method and check if the container was replaced with the content: in that case we try to load the related term.
