COntologyNode

Description
Description | Vars | Methods (details) Constants

Ontology node.

This class overloads its ancestor to add a method that can be used to relate the current node to another one using a predicate term and a graph edge.

The parent class does not feature this method because nodes of that class are supposed to be root nodes.

RelatedTo (line 348)

Return all edges pointing to this node.

This method can be used to retrieve the list of nodes that point to the current one. The method will return an array structured as follows:

  • Key: The identifier of the predicate term.
  • Value: An array of object nodes structured as follows:
    • Index: The node ID.
    • Value: The term attributes merged with the node properties.

The method accepts the following parameters:

  • $theContainer: The graph and term containers as an array:
    • kTAG_NODE: This element should hold the nodes container, it must be a Everyman\Neo4j\Client instance.
    • kTAG_TERM: This element should hold the terms container, it must be a CContainer instance.
  • $thePredicate: The predicate terms as an array of the following types:
    • COntology: The node term global COntologyTerm::GID().
    • COntologyTerm: The term global identifier will be used as the node COntologyEdge::Type().
    • Everyman\Neo4j\Relationship: The relationship's type will be used as the predicate, all other elements of the provided edge node will be ignored.
    • string: Any other type will be converted to string and will be used as the node COntologyEdge::Type().

RelateTo (line 121)

Create a graph edge.

This method can be used to create a graph edge or relation between the current node and an object node using a predicate node. This method will not duplicate relationships between the same nodes and the predicate term: it uses the kINDEX_NODE_TERM relationship index and its kTAG_EDGE_NODE key to locate existing relationships. For that reason the subject and object terms of the relationship must relate to committed nodes.

The method accepts the following parameters:

  • $theContainer: The graph and term containers as an array:
    • kTAG_NODE: This element should hold the nodes container, it must be a Everyman\Neo4j\Client instance.
    • kTAG_TERM: This element should hold the terms container, it must be a CContainer instance.
  • $thePredicate: The predicate term:
    • COntology: The node term global COntologyTerm::GID().
    • COntologyTerm: The term global identifier will be used as the node COntologyEdge::Type().
    • Everyman\Neo4j\Relationship: The relationship's type will be used as the predicate, all other elements of the provided edge node will be ignored.
    • string: Any other type will be converted to string and will be used as the node COntologyEdge::Type().
  • $theObject: The destination node or relationship object node:
    • COntologyNode: The method will use it's node.
    • Everyman\Neo4j\Node: The method will use it as the relationship object.
    • integer: The method will search for the node corresponding to the provided number, if the node was not found, the method will raise an exception.
    • other: Any other type will raise an exception.

The method will return a COntologyEdge object, or raise an exception if the operation was not successful.
