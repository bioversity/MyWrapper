Ontology term ancestor.

This class defines a term that resides in an ontology, it overloads ita parent to hash the identifier into a binary string and enforce the encoded status flag.

The class also adds the following properties:

  • kTAG_REFERENCE_SYNONYM: This property can be used to store synonyms as strings.
  • kTAG_REFERENCE_XREF: This property can be used to store cross references to other terms in the ontology.
  • kTAG_SOURCE: This property can be used to provide the source of the term.

We declare this class abstract to force the creation of specific ontology term types.

Manage object references.

We override this method to handle references structured as follows:

  • kTAG_KIND: This offset represents the reference predicate, it may be omitted if the reference has no type or when we want to define a default reference. By default we expect here a term reference.
  • kTAG_DATA: This offset represents the referenced term, it should be in the form of an object reference.

The main reason to override the method is to ensure references are standard binary strings.

Source (line 412)

Manage term sources.

This method can be used to manage the term sources, it manages an array of strings with the following offsets:

  • kTAG_KIND: The source kind, this string can be used to define the type of the source, this element represents the array key, although technically it is implemented as an element to allow searching on all values.
  • kTAG_DATA: The source, this element should hold the actual source reference, which should be convertable to a string.

The parameters to this method are:

  • $theValue: The value or operation:
    • NULL: Return the current value selected by the second parameter.
    • FALSE: Delete the value selected by the second parameter.
    • other: Set value selected by the second parameter.
  • $theType: The element type, kind or index:
    • NULL: This value indicates that the phone has no type or kind, in general, when adding elements, this case applies to default elements.
    • other: All other types will be interpreted as the kind or type of the phone number.
  • $getOld: Determines what the method will return:
    • TRUE: Return the value before it was eventually modified.
    • FALSE: Return the value after it was eventually modified.

  • access: public

_CheckReference (line 692)

Normalise reference parameter.

This method can be used to normalise a parameter that is supposed to be a reference to another term, that is, a binary string hash of the term's identifier converted to a binary standard type.

The method will perform the following conversions:

  • CDataTypeBinary: This is the default data type for the identifier.
  • MongoBinData: This type will be converted to the standard CDataTypeBinary type.
  • COntologyTermObject: Objects of the same class will have their identifier extracted.
  • NULL: NULL data will simply be passed.
  • other: Any other data type is assumed to be the term's identifier, so it will be hashed into a binary string and converted to the standard CDataTypeBinary type.
  • array: Arrays cannot be converted to string, so the method will raise an exception.

  • see: kFLAG_STATE_ENCODED
  • access: protected
  • uses: CPersistentObject::_IsEncoded()
