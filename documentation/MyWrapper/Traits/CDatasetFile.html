Dataset file.

This class implements a dataset file, it wraps the parent class around a structure that records a dataset file's attributes.

In general instances of this class will be embedded in a dataset objects to add lists of file references, the current class does not feature any persistence functions, it only concentrates in managing a file reference as a set of properties:

  • File(): This property defines a file, it represents the GridFS record identifier for the file.
  • Referenced: This property holds the list of file File(), this attribute all files that were generated taking the current file as a model. For instance, this list would contain all CSV files generated from all the workbooks of an Excel file.
  • Status: This property holds the list of states in which the current file is.
  • Kind: This property holds the list of kinds or types assigned to the file.
  • Columns: This property holds the list of column headers and associated metadata structured as follows:
    • kTAG_TAG: The tag associated with the data in the current column.
    • Title(): The original header title provided in the file and used for the annotation.

You should always instantiate this class from the dataset by using its CDataset::NewFile() method rather than instantiating the class on its own, this is because this way it is guaranteed that the record points to an existing file.

Column (line 335)

Manage the file columns.

This method can be used to handle the object's list of columns, it uses the standard accessor method to manage each of the file's column metadata.

Each element of this list represents a column of the file and holds the following information:

  • kTAG_TAG: This element holds the identifier of the ontology tag associated with the data contained in the current file's column.
  • kTAG_TITLE: This element holds the original header contents which were used to determine the column tag.

The method accepts three parameters:

  • $theColumn: The column index as an integer (zero based), or the operation to be performed on the whole list:
    • NULL: Return all columns metadata.
    • FALSE: Delete all columns metadata, the method will return the old value.
    • other: Any other type of value will be interpreted as an integer indicating the column index (zero based) to be considered.
  • $theTag: The identifier of the tag associated with the current column, or the operation to be performed:
    • NULL: Return the full contents of the current column, the next parameter is ignored in this case.
    • FALSE: Delete the current column, the next parameter is ignored in this case.
    • other: Any other type of value will be interpreted as the tag identifier associated with the column.
  • $theTitle: The original header contents used to annotate the column.

Manage dataset.

This method can be used to manage the file's dataset reference or the operation to be performed:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • other: Set the value with the provided parameter.

The second parameter is a boolean which if TRUE will return the old value when replacing values; if FALSE, it will return the currently set value.
