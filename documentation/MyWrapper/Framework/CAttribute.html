Attribute class.

This class is a static methods repository which is used to manage CArrayObject derived classe's attributes.

The class is a collection of static methods which represent member accessor methods for objects that store their properties as array elements. These methods will handle insertion, extraction and deletion of attributes in either arrays or ArrayObject instances, these methods will then be used by CArrayObject derived classes as the base for their member accessor methods.

The class features the following methods:

  • ManageOffset: Manage a scalar offset.
  • ManageArrayOffset: Manage elements of an array offset.
  • ManageTypedOffset: Manage elements of an array offset by category or type.
  • ManageTypedArrayOffset: Manage elements of an array offset by category or type and data element value.

Hash closure.


This static method returns the default hashing closure that will be used to compare  list elements. This method will return a closure function that expects two arguments:
  • $theElement: This parameter represents the element to be hashed, it can
		take two forms depending on the presence or not of the next parameter:
	 
    • Next parameter is not there: If you omit the next parameter, it means
			that the element belongs to a list of scalars and that it is the element
			itself to represent the key. In this case, the function will return the MD5
			hash of the element itself.
    • Next parameter is there: If you provide the next parameter, it means
			that the provided offset or offsets point to the key data items contained in
			this parameter: in this case this parameter must be either an array or an
			ArrayObject, if you provide anything else, an exception will be raised. The
			hash will be generated by an MD5 hash of the following sequence:
			[Offset]
			kTOKEN_NAMESPACE_SEPARATOR
            [MD5( Value )], in which the value is the value referenced by the
            offset or NULL; if there are more than one offset, these blocks will
            be separated by the kTOKEN_INDEX_SEPARATOR
            token, the function will return the MD5 of the full sequence.
  • $theOffsets: If provided, it must be an array of strings that represent
        offsets to items in the previous parameter which constitute the element's key.
        Note that these offsets will be sorted prior to being used, to allow consistent
        results.
When creating your custom closures, you should use this function as a model and be  aware of the following rule: the static methods in this class invoke a single hash  closure, but they might use this closure on both lists and matrices, so you should  design the function as two separate functions depending on the presence or not of  the offsets list.

Manage an array offset.


An array offset is an array property, this method can be used to set, retrieve and  delete the elements of this property, as opposed to  ManageOffset, which is used to manage the property as a whole.
The elements of this list are uniquely identified by a closure function which is  either passed to this method or defaults to the HashClosure  function.
When deleting elements, if the list becomes empty, the whole offset will be deleted.
The method accepts the following parameters:
  • &$theReference: Reference to the attributes container, it may either
        refer to an array or an ArrayObject, any other type will trigger an exception.
  • $theOffset: The offset to the attribute contained in the previous
        parameter that is to be managed. This referenced element is expected to be an
        array, if this is not the case, the method will raise an exception.
  • $theValue: Depending on the next parameter, this may either refer to the
        value to be set or to the index of the element to be retrieved or deleted:
     
    • NULL: This value indicates that we want to operate on all elements,
            which means, in practical terms, that we either want to retrieve or delete
            the full list. If the operation parameter resolves to TRUE, the
            method will default to retrieving the current list and no new element will
            be added.
    • array: An array indicates that we want to operate on a list of
            values and that other parameters may also be provided as lists. Note that
            ArrayObject instances are not considered here as arrays.
    • other: Any other type represents either the new value to be added or
            the index to the value to be returned or deleted. Note that this value will
            be hashed by the provided or default closure to
            determine if the element is new or not.
  • $theOperation: This parameter represents the operation to be performed
        whose scope depends on the value of the previous parameter:
     
    • NULL: Return the element or full list.
    • FALSE: Delete the element or full list.
    • array: This type is only considered if the $theValue parameter
            is provided as an array: the method will be called for each element of the
            $theValue parameter matched with the corresponding element of this
            parameter, which also means that both both parameters must share the same
            count.
    • other: Add the $theValue value to the list. If you provided
            NULL in the previous parameter, the operation will be reset to
            NULL.
  • $getOld: Determines what the method will return:
     
    • TRUE: Return the value before it was eventually modified.
    • FALSE: Return the value after it was eventually modified.
  • $theClosure: The hashing closure, the function should return a value
        representing the element's key. If omitted or NULL, the
        default closure will be used. If the $theValue
        parameter was provided as an array, you can provide an array of closures each
        applying to the corresponding element of $theValue list.

Manage a list of object references.


This method can be used to manage a list of object references, in which each element  is either:
  • Scalar: A scalar or object representing:
	 
    • The object: The actual referenced object.
    • The object reference: An object reference structure or a scalar
			representing the object's identifier.
    
        or:
  • Array: A structure composed of two items:
     
    • Kind: This offset represents the type or predicate of the reference.
    • Data: This offset represents the actual object or object reference.
The reference list is numerically indexed array and this method will ensure it  remains so.
The method accepts the following parameters:
  • &$theReference: Reference to an array or ArrayObject derived instance.
  • $theMainOffset: The offset to manage.
  • $theTypeOffset: The element's offset of the type or predicate.
  • $theDataOffset: The element's offset of the data.
  • $theValue: This parameter represents either the search key in the list
        when retrieving or deleting, or the reference when replacing or adding. If you
        provide an array, it means that the elements may have a kind offset and that the
        reference or object must be found in the data offset. When matching, if the kind
        offset is not provided, it means that only those elements that do not have a
        kind offset will be selected for matching. If the types match, the method will
        use the ObjectIdentifier
        method to match the references, please refer to its documentation for more
        information. If the provided value is not an array, it means that the reference
        list does not feature types, so matches will only be performed on the reference.
  • $theOperation: The operation to perform:
     
    • NULL: Return the element matched by the previous parameter.
    • FALSE: Delete the element matched by the previous parameter and
            return it.
    • other: Any other value means that we want to add to the list the
            element provided in the previous parameter, either appending it if there
            was no matching element, or by replacing a matching element. The method will
            return either the replaced element or the new one.
  • $getOld: Determines what the method will return when deleting or
        replacing:
     
    • TRUE: Return the deleted or replaced element.
    • FALSE: Return the replacing element or NULL when deleting.
The method used to match the list  elements expects identifiers in the references or objects, if these  are not there, there is no way to discern duplicates.

Manage a typed array offset.


The structures managed by this method are equivalent to the structures managed by the  ManageTypedOffset method, except that the data item, in  this case, is an array and the goal of this method is to manage the data item array  elements rather than the data item itself.
This method features many of the same parameters than the  ManageTypedOffset method except for a couple of changes.
Here is the list of arguments:
  • &$theReference: The reference to the attributes container, the object or
        array that holds the list of elements. The reference must point to an array or
        to an ArrayObject instance, any other type will trigger an exception.
  • $theMainOffset: The offset, in the previous parameter, to the list of
        elements to be managed, this referenced element is expected to be an array, if
        this is not the case, the method will raise an exception.
  • $theDataOffset: The offset to the element's data item as a string. If you
        provide an array, it means that you want to perform a series of operations, in
        other words, this method will be called once for each element of this array and
        the combination of parameters will depend on how the other arguments will have
        been provided. Note that an ArrayObject is not considered as a list in this
        case.
  • $theTypeOffsets: This parameter represents the offsets of the element
        items that represent the element's kind, type or key. In general you should
        provide an array of strings, if you provide a scalar, this will become an array
        of one element. If the previous parameter was provided as an array, it means
        that this parameter may take three forms:
     
    • scalar: The single offset will be used for each element provided in
            the previous parameter.
    • list: If you provide a list of strings, this list will be used for
            each element provided in the previous parameter.
    • matrix: If you provide a list of arrays, each element of the previous
            parameter will be matched with the corresponding array in this list, which
            also means that in this case the count of both parameters must be the same.
  • $theTypeValues: This parameter represents the the element kind, type or
        key values, this parameter is matched against the previous parameter to
        constitute the significant items of the element, meaning that each element of
        the previous parameter represents the offset referencing the corresponding
        element in this parameter.
  • $theData: This parameter represents the element's data value or the the
        data scope; this parameter is one of those that have a different usage than in
        the ManageTypedOffset method:
     
    • array: An array is considered in two different ways, depending on the
            type of the $theDataOffset parameter:
         
      • scalar: In this case the array is considered the data value.
      • array: In this case the array is considered as the list of values
                to be managed: each element of this array will be identified by the
                corresponding element of the $theDataOffset array parameter.
    • NULL: This indicates that the operation, provided in the next
            parameter, applies to the whole data item list; this also means that if the
            operation is TRUE we raise here an exception, because one cannot add
            a NULL data item.
    • other: Any other value represents either the index of the data item
            to be retrieved or deleted, or the data item to be added. Note that if the
            value is an array, the array elements will be added individually, rather
            than setting the data to the actual array.

Manage a typed offset.


A typed offset is a list element that has at least an item containing the element  data and a variable number of other items that determine the type or kind of the  element: these latter items also determine the element's key and are used to  retrieve, delete and create elements, no two elements may share the same combination  of these items. Elements of this type must be arrays, ArrayObject instances will be  converted to arrays.
The method accepts the following parameters:
  • &$theReference: The reference to the attributes container, the object or
		array that holds the list of elements. The reference must point to an array or
		to an ArrayObject instance, any other type will trigger an exception.
  • $theMainOffset: The offset, in the previous parameter, to the list of
		elements to be managed, this referenced element is expected to be an array, if
		this is not the case, the method will raise an exception.
  • $theDataOffset: The offset to the element's data item as a string. If you
		provide an array, it means that you want to perform a series of operations, in
		other words, this method will be called once for each element of this array and
		the combination of parameters will depend on how the other arguments will have
		been provided. Note that an ArrayObject is not considered as a list in this
		case.
  • $theTypeOffsets: This parameter represents the offsets of the element
		items that represent the element's kind, type or key. In general you should
		provide an array of strings, if you provide a scalar, this will become an array
		of one element. If the previous parameter was provided as an array, it means
		that this parameter may take three forms:
	 
    • scalar: The single offset will be used for each element provided in
			the previous parameter.
    • list: If you provide a list of strings, this list will be used for
			each element provided in the previous parameter.
    • matrix: If you provide a list of arrays, each element of the previous
			parameter will be matched with the corresponding array in this list, which
			also means that in this case the count of both parameters must be the same.
  • $theTypeValues: This parameter represents the the element kind, type or
		key values, this parameter is matched against the previous parameter to
		constitute the significant items of the element, meaning that each element of
		the previous parameter represents the offset referencing the corresponding
		element in this parameter.
  • $theData: This parameter represents the element's data value or the
		operation to be performed:
	 
    • array: An array is considered in two different ways, depending on the
			type of the $theDataOffset parameter:
		 
      • scalar: In this case the array is considered the data value.
      • array: In this case the array is considered as the list of values
				to be managed: each element of this array will be identified by the
				corresponding element of the $theDataOffset array parameter.
    • NULL: Retrieve the element's data item matching the provided types.
    • FALSE: Delete the element's data item matching the provided types.
    • other: Add or replace the element's data item matching the provided
			types with the current value.
  • $getOld: Determines what the method will return:
	 
    • TRUE: Return the data item before it was eventually modified.
    • FALSE: Return the data item after it was eventually modified.
  • $theClosure: The hashing closure, if omitted, the
		default closure function will be used, for more details on
        how to create a custom closure please see the HashClosure
        reference in this class. If the $theDataOffset was provided as an array,
        this parameter may also be provided as a list of closures matching the
        $theDataOffset parameter count.
