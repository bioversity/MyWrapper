Attribute class.

This class is a static methods holder that manage ArrayObject derived class attributes.

The main function of this class is to concentrate all attribute management methods in a static class which can be used for managing the attributes of any class derived from an ArrayObject.

The class features the following methods:

  • ManageOffset: Manage a scalar offset.
  • ManageArrayOffset: Manage elements of an array offset.
  • ManageTypedOffset: Manage elements of an array offset by category or type.
  • ManageTypedArrayOffset: Manage elements of an array offset by category or type and data element value.

Manage an array offset.


This method can be used to manage an array offset, this options involves setting,  retrieving and deleting elements of an offset which contains an array of values, this  method concentrates in managing the offset's elements, rather than  managing the offset itself.
The offset's array should be composed by elements that must be convertable to  strings: the string value represents the index of the element, which means that no  two elements can have the same string value.
When deleting elements, if the list becomes empty, the whole offset will be deleted.
The method accepts the following parameters:
  • &$theReference: Reference to an array or ArrayObject derived instance.
  • $theOffset: The offset to manage.
  • $theValue: This parameter represents the data element to be set, or the
        index to the data element to be deleted or retrieved:
     
    • NULL: This value indicates that we want to operate on all elements,
            which means, depending on the next parameter, that we are either retrieving
            or deleting the full list. If the operation parameter is TRUE, no
            element will be added.
    • array: This value indicates that we want to operate on a list of
            values: each of these values will be handled according to the operation
            parameter. Note that an ArrayObject is not considered in this scenario, so
            in that case you would have to convert it to an array.
    • other: Any other type represents either the new value to be added or
            the index to the value to be returned or deleted. It must be possible to
            cast this value to a string, this is what will be used to compare
            elements.
  • $theOperation: This parameter represents the operation to be performed,
        it will be evaluated as a boolean and its scope depends on the value of the
        previous parameter:
     
    • NULL: Return the element or list.
    • FALSE: Delete the element or list.
    • TRUE: Add the element or list. If you provided NULL as value,
            the operation will do nothing.
  • $getOld: Determines what the method will return:
     
    • TRUE: Return the value before it was eventually modified.
    • FALSE: Return the value after it was eventually modified.

		static ManageObjectList (line 1411)
	
 
	
	

Manage a list of object references.


This method can be used to manage a list of object references, in which each element  is either:
  • Scalar: A scalar or object representing:
	 
    • The object: The actual referenced object.
    • The object reference: An object reference structure or a scalar
			representing the object's identifier.
    
        or:
  • Array: A structure composed of two items:
     
    • Kind: This offset represents the type or predicate of the reference.
    • Data: This offset represents the actual object or object reference.
The reference list is numerically indexed array and this method will ensure it  remains so.
The method accepts the following parameters:
  • &$theReference: Reference to an array or ArrayObject derived instance.
  • $theMainOffset: The offset to manage.
  • $theTypeOffset: The element's offset of the type or predicate.
  • $theDataOffset: The element's offset of the data.
  • $theValue: This parameter represents either the search key in the list
        when retrieving or deleting, or the reference when replacing or adding. If you
        provide an array, it means that the elements may have a kind offset and that the
        reference or object must be found in the data offset. When matching, if the kind
        offset is not provided, it means that only those elements that do not have a
        kind offset will be selected for matching. If the types match, the method will
        use the ObjectIdentifier
        method to match the references, please refer to its documentation for more
        information. If the provided value is not an array, it means that the reference
        list does not feature types, so matches will only be performed on the reference.
  • $theOperation: The operation to perform:
     
    • NULL: Return the element matched by the previous parameter.
    • FALSE: Delete the element matched by the previous parameter and
            return it.
    • other: Any other value means that we want to add to the list the
            element provided in the previous parameter, either appending it if there
            was no matching element, or by replacing a matching element. The method will
            return either the replaced element or the new one.
  • $getOld: Determines what the method will return when deleting or
        replacing:
     
    • TRUE: Return the deleted or replaced element.
    • FALSE: Return the replacing element or NULL when deleting.
The method used to match the list  elements expects identifiers in the references or objects, if these  are not there, there is no way to discern duplicates.

Manage a typed array offset.


A typed array offset is structured as follows:
  • Type: This offset contains a scalar which determined the type or category
		of the element. This offset may be omitted.
  • Data: This offset contains the element data as an array of data elements,
		these elements are handled by the ManageArrayOffset
        method.
No two elements may share the same type or category and all data elements within a  category must be unique.
The method accepts the following parameters:
  • &$theReference: Reference to an array or ArrayObject derived instance.
  • $theMainOffset: The offset to manage.
  • $theTypeOffset: The element's offset of the type or category.
  • $theDataOffset: The element's offset of the data.
  • $theType: This parameter represents the index to the offset's elements.
  • $theData: This parameter represents the element's data element, if
        NULL or omitted, it implieas that the operation applies to the whole list
        of data elements.
  • $theOperation: This parameter represents the operation to be performed,
        it will be evaluated as a boolean and its scope depends on the value of the
        previous parameter:
     
    • NULL: Return the element or list.
    • FALSE: Delete the element or list.
    • TRUE: Add the element or list.
  • $getOld: Determines what the method will return:
     
    • TRUE: Return the value of the offset before it was eventually
            modified.
    • FALSE: Return the value of the offset after it was eventually
            modified.
The method will return the matched element's value of the data offset.

Manage a category and type offset.


A typed kind offset is structured as follows:
  • Kind: This offset contains a scalar which determined the kind or category
		of the element, this element may be omitted.
  • Type: This element represents the type of the next item, this element is
		required.
  • Data: This offset contains the element data, in this method we treat it
		as a scalar, this element is required.
No two elements of the list can share the same kind and type, these represent the  index of the array.
The parameters to this method are:
  • &$theReference: Reference to an array or ArrayObject derived instance.
  • $theMainOffset: The offset to manage.
  • $theKindOffset: The element's offset of the kind.
  • $theTypeOffset: The element's offset of the type.
  • $theDataOffset: The element's offset of the data.
  • $theKind: The item kind value; this value may be NULL, or one
		should be able to cast it to a string.
  • $theType: The item type value; one should be able to cast the value to a
		string.
  • $theData: This parameter represents the item's data element, or the
		operation to be performed:
	 
    • NULL: This indicates that we want to retrieve the data of the item
			with index matching the kind and type parameters.
    • FALSE: This indicates that we want to remove the item matching the
			kind and type parameters.
    • other: Any other value indicates that we want to add or replace the
			data element of the item matching the kind and type parameters.
  • $getOld: Determines what the method will return:
	 
    • TRUE: Return the element or list before it was eventually
			modified.
    • FALSE: Return the element or list after it was eventually
			modified.

Manage a typed offset.


A typed offset is structured as follows:
  • Type: This offset contains a scalar which determined the type or category
		of the element. This offset may be omitted.
  • Data: This offset contains the element data, in this method we treat it
		as a scalar. This offset may not be omitted.
No two elements may share the same type or category.
The method accepts the following parameters:
  • &$theReference: Reference to an array or ArrayObject derived instance.
  • $theMainOffset: The offset to manage.
  • $theTypeOffset: The element's offset of the type or category.
  • $theDataOffset: The element's offset of the data.
  • $theType: This parameter represents the value of the index element of the
		item, depending on the next parameter this value will be used for matching
		items in the list:
	 
    • NULL: An empty type means that we are looking for the item lacking
			the index element.
    • array: If you provide an array, it means that you are operating on a
			list of items: depending on the next parameter this will mean either
			retrieving the data elements of the items matching the array, deleting these
			items, or adding/replacing the items; in this lastcase, this means that the
			next parameter must also be an array and that each of its elements will be
			associated to the corresponding index element.
    • other: Any other value will be considered as the index to retrieve,
			remove or add/replace. You MUST be able to cast this value to a
			string.
  • $theData: This parameter represents the item's data element, or the
		operation to be performed:
	 
    • NULL: This indicates that we want to retrieve the data of the item
			with index matching the previous parameter.
    • FALSE: This indicates that we want to remove the item matching the
			index provided in the previous parameter.
    • other: Any other value indicates that we want to add or replace the
			data element of the item matching the previous parameter. Note that if the
			previous parameter is an array, this one must also be an array matching the
			latter's count.
  • $getOld: Determines what the method will return:
	 
    • TRUE: Return the value of the offset before it was eventually
			modified.
    • FALSE: Return the value of the offset after it was eventually
			modified.
The method will return the matched element's value of the data offset.
