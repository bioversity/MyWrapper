Attribute class.

This class is a static methods holder that manage ArrayObject derived class attributes.

The main function of this class is to concentrate all attribute management methods in a static class which can be used for managing the attributes of any class derived from an ArrayObject.

The class features the following methods:

  • ManageOffset: Manage a scalar offset.
  • ManageArrayOffset: Manage elements of an array offset.
  • ManageTypedOffset: Manage elements of an array offset by category or type.
  • ManageTypedArrayOffset: Manage elements of an array offset by category or type and data element value.

Manage an array offset.


This method can be used to manage an array offset, this options involves setting,  retrieving and deleting elements of an offset which contains an array of values, this  method concentrates in managing the offset's elements, rather than  managing the offset itself.
Note that the elements of the array must be scalars and must be convertable to a  string.
The method accepts the following parameters:
  • &$theReference: Reference to an array or ArrayObject derived instance.
  • $theOffset: The offset to manage.
  • $theValue: This parameter represents either the value to add, or the
        index of the element to retrieve or delete:
     
    • NULL: This value indicates that we want to operate on all elements,
            which means, depending on the next parameter, that we are either retrieving
            or deleting the full list.
    • array: This value indicates that we want to replace the whole list,
            this will only be tested if the next parameter evaluates to TRUE.
    • other: Any other type represents either the new value to be added or
            the index to the value to be returned or deleted. It must be possible to
            cast this value to a string, this is what will be used to compare
            elements.
  • $theOperation: This parameter represents the operation to be performed,
        it will be evaluated as a boolean and its scope depends on the value of the
        previous parameter:
     
    • NULL: Return the element or list.
    • FALSE: Delete the element or list.
    • TRUE: Add the element or list.
  • $getOld: Determines what the method will return:
     
    • TRUE: Return the value before it was eventually modified.
    • FALSE: Return the value after it was eventually modified.

Manage a typed array offset.


A typed array offset is structured as follows:
  • Type: This offset contains a scalar which determined the type or category
		of the element. This offset may be omitted.
  • Data: This offset contains the element data as an array of data elements,
		these elements are handled by the ManageArrayOffset
        method.
No two elements may share the same type or category and all data elements within a  category must be unique.
The method accepts the following parameters:
  • &$theReference: Reference to an array or ArrayObject derived instance.
  • $theMainOffset: The offset to manage.
  • $theTypeOffset: The element's offset of the type or category.
  • $theDataOffset: The element's offset of the data.
  • $theType: This parameter represents the index to the offset's elements.
  • $theData: This parameter represents the element's data element.
  • $theOperation: This parameter represents the operation to be performed,
        it will be evaluated as a boolean and its scope depends on the value of the
        previous parameter:
     
    • NULL: Return the element or list.
    • FALSE: Delete the element or list.
    • TRUE: Add the element or list.
  • $getOld: Determines what the method will return:
     
    • TRUE: Return the value of the offset before it was eventually
            modified.
    • FALSE: Return the value of the offset after it was eventually
            modified.
The method will return the matched element's value of the data offset.

Manage a typed offset.


A typed offset is structured as follows:
  • Type: This offset contains a scalar which determined the type or category
		of the element. This offset may be omitted.
  • Data: This offset contains the element data, in this method we treat it
		as a scalar. This offset may not be omitted.
No two elements may share the same type or category.
The method accepts the following parameters:
  • &$theReference: Reference to an array or ArrayObject derived instance.
  • $theMainOffset: The offset to manage.
  • $theTypeOffset: The element's offset of the type or category.
  • $theDataOffset: The element's offset of the data.
  • $theType: This parameter represents the value of the index element of the
		item, depending on the next parameter this value will be used for matching
		items in the list:
	 
    • NULL: An empty type means that we are looking for the item lacking
			the index element.
    • array: If you provide an array, it means that you are operating on a
			list of items: depending on the next parameter this will mean either
			retrieving the data elements of the items matching the array, deleting these
			items, or adding/replacing the items; in this lastcase, this means that the
			next parameter must also be an array and that each of its elements will be
			associated to the corresponding index element.
    • other: Any other value will be considered as the index to retrieve,
			remove or add/replace. You MUST be able to cast this value to a
			string.
  • $theData: This parameter represents the item's data element, or the
		operation to be performed:
	 
    • NULL: This indicates that we want to retrieve the data of the item
			with index matching the previous parameter.
    • FALSE: This indicates that we want to remove the item matching the
			index provided in the previous parameter.
    • other: Any other value indicates that we want to add or replace the
			data element of the item matching the previous parameter. Note that if the
			previous parameter is an array, this one must also be an array matching the
			latter's count.
  • $getOld: Determines what the method will return:
	 
    • TRUE: Return the value of the offset before it was eventually
			modified.
    • FALSE: Return the value of the offset after it was eventually
			modified.
The method will return the matched element's value of the data offset.
