Graph unit object.

This class represents the ancestor of coded unit objects that exist within a graph, they add two default properties:

  • kTAG_LINK_IN: This property contains the list of incoming references or links, it is a list of object identifiers or object references.
  • kTAG_LINK_OUT: This property contains the list of outgoing references or links, it is a list of object identifiers or object references.

RelatedFrom (line 103)

Manage incoming references.

This method can be used to handle incoming references, it accepts the following parameters:

  • $theValue: Reference or object. This parameter represents the reference, it may be a scalar, representing either the referenced object identifier, a structure representing an object reference, or the referenced object itself.
  • $theOperation: The operation to perform:
    • NULL: Return the element matched by the previous parameters.
    • FALSE: Delete the element matched by the previous parameters and return it.
    • other: Any other value means that we want to add to the list the element provided in the previous parameters, either appending it if there was no matching element, or by replacing a matching element. The method will return either the replaced element or the new one.
  • $getOld: Determines what the method will return when deleting or replacing:
    • TRUE: Return the deleted or replaced element.
    • FALSE: Return the replacing element or NULL when deleting.

  • see: kTAG_LINK_IN
  • access: public
  • uses: CPersistentUnitObject::_ManageObjectList()

_PrepareCommit (line 197)

Normalise before a store.

We overload this method to check if the object in initialised, if this is not the case we raise an exception.

We also scan the incoming and outgoing references to commit any elements that are actual instances and convert them to references: we discriminate such elements by selecting only objects derived from the CPersistentUnitObject class.
