# UnserialiseData( reference & $theElement )

Unserialise provided data element.

Unserialise provided data element.

We CContainer::UnserialiseData() implement this method to convert all standard CDataType types into custom Mongo data types.

In this class we parse the following types and kTAG_TYPE offsets:

  • CDataTypeMongoId CDataTypeMongoId object or kTYPE_MongoId kTYPE_MongoId offset: We return a MongoId object.
  • CDataTypeMongoCode CDataTypeMongoCode object or kTYPE_MongoCode kTYPE_MongoCode offset: We return a MongoCode object.
  • CDataTypeStamp CDataTypeStamp object or kTYPE_STAMP kTYPE_STAMP offset: We return a MongoDate object.
  • CDataTypeRegex CDataTypeRegex object or kTYPE_REGEX kTYPE_REGEX offset: We return a MongoRegex object.
  • CDataTypeInt32 CDataTypeInt32 object or kTYPE_INT32 kTYPE_INT32 offset: We return a MongoInt32 object.
  • CDataTypeInt64 CDataTypeInt64 object or kTYPE_INT64 kTYPE_INT64 offset: We return a MongoInt64 object.
  • CDataTypeBinary CDataTypeBinary object or kTYPE_BINARY kTYPE_BINARY offset: We return a MongoBinData object.

# _Commit( reference & $theObject, reference & $theIdentifier, reference & $theModifiers )

Commit provided object.

Commit provided object.

We implement this method to handle MongoCollection object stores, this method will store the object in the current container.

The method will check if the current container is a MongoCollection, if this is not the case, it will raise an kERROR_INVALID_STATE exception.

If the provided modifiers indicate a kFLAG_PERSIST_MODIFY modify operation, the method will return the modified object, in all other cases the method will return the object identifier.

# _Load( reference & $theIdentifier, reference & $theModifiers )

Load object.

Load object.

We implement this method to handle MongoCollection object stores, this method will retrieve the object from the current container.

The Load() caller will have resolved references andeventually extracted the identifier from the provided parameter.

If the provided parameter is an array, the method will assume it is a query; if not, the method will assume it is the value to match against the kTAG_LID local identifier.

The method will use the findOne method to retrieve the object.

# _PrepareCommit( reference & $theObject, reference & $theIdentifier, reference & $theModifiers )

Prepare before a _Commit() commit.

Prepare before a _Commit() commit.

We CContainer::_PrepareCommit() overload this method to handle the identifier: if provided, it means that that is to become the object's unique kTAG_LID identifier; if not provided and the object has an kTAG_LID identifier, we use that one.

We also raise an exception if the provided object is not either an array or an ArrayObject.
