Class COntologyNode

Ontology graph node.

This class implements an ontology graph node.

The class is derived from CGraphNode CGraphNode, it adds another required property, the Term() term. This class will wrap the array access framework to the combination of the Node() node properties and the Term() term elements, except that the Term() term elements will be read-only.

This class introduces a new kind of container: it must be an array of two elements structured as follows:

  • kTAG_NODE kTAG_NODE: This element should hold the nodes container, it must be a Everyman\Neo4j\Client instance.
  • kTAG_TERM kTAG_TERM: This element should hold the terms container, it must be a CContainer CContainer instance.

Note that the class will not cast to an array correctly, you must use the getArrayCopy() getArrayCopy method to get an array, if you know how to solve this, please do it!

public COntologyEdge
# RelateTo( reference $theContainer, mixed $thePredicate, mixed $theObject = NULL, boolean $doDuplicate = FALSE )

Create a graph edge.

Create a graph edge.

This method can be used to create a graph edge or relation between the current node and an object node using a predicate node. This method consideres the triplet subject node/predicate term/object node as the relationship identifier when considering duplicate relationships. For that reason the subject and object terms of the relationship must relate to _IsCommitted() committed nodes.

The method accepts the following parameters:

  • $theContainer: The graph and term containers as an array:
    • kTAG_NODE kTAG_NODE: This element should hold the nodes container, it must be a Everyman\Neo4j\Client instance.
    • kTAG_TERM kTAG_TERM: This element should hold the terms container, it must be a CContainer CContainer instance.
  • $thePredicate: The predicate term:
    • COntologyNode: The node Term() term kTAG_GID global COntologyTerm::GID() identifier.
    • COntologyTerm COntologyTerm: The term kTAG_GID global identifier will be used as the COntologyEdge node COntologyEdge::Type() type.
    • Everyman\Neo4j\Relationship: The relationship's type will be used as the predicate, all other elements of the provided edge node will be ignored.
    • string: Any other type will be converted to string and will be used as the COntologyEdge node COntologyEdge::Type() type.
  • $theObject: The destination node or relationship object node:
    • COntologyNode: The method will use it's Node() node.
    • Everyman\Neo4j\Node: The method will use it as the relationship object.
    • integer: The method will search for the node corresponding to the provided number, if the node was not found, the method will raise an exception.
    • other: Any other type will raise an exception.
  • $doDuplicate: This flag determines whether cuplicate relationships are allowed: if FALSE, if a relationship with the same subject and object nodes, along with the same predicate term are found, it will be considered a duplicate, and this method will actually return that relationship.

The method will return a COntologyEdge COntologyEdge object, or raise an exception if the operation was not successful.

# _ManagePropertyArray( string $theOffset, mixed $theValue = NULL, mixed $theOperation = NULL, boolean $getOld = FALSE )

Manage scalar property.

Manage scalar property.

This method will set, retrieve and delete array element properties, it accepts the following parameters:

  • $theOffset: The property offset to manage.
  • $theValue: This parameter represents either the value to add, or the index of the element to operate on:
    • NULL: This value indicates that we want to operate on all elements, which means that we are retrieving the full list or deleting it.
    • array: This value indicates that we want to replace the whole list, this will only be tested if the next parameter evaluates to TRUE.
    • other: Any other type represents either the new value to be added or the index to the value to be returned or deleted. It must be possible to cast this value to a string, this is what will be used to compare elements.
  • $theOperation: This parameter represents the operation to be performed, it will be evaluated as a boolean and its scope depends on the value of the previous parameter:
    • NULL: Return the element or list.
    • FALSE: Delete the element or list.
    • TRUE: Add the element or list. Note that with this value, if you provide NULL in the previous parameter, it will be equivalent to deleting the whole list.
  • $getOld: Determines what the method will return:
    • TRUE: Return the element or list before it was eventually modified.
    • FALSE: Return the element or list after it was eventually modified.
