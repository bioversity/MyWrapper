Class CMongoDataWrapperClient

MongoDB data wrapper client.

This class represents a MongoDB web-services data wrapper client, it facilitates the job of requesting information from servers derived from the CMongoDataWrapper CMongoDataWrapper class.

This class adds the following properties to its CWrapperClient ancestor:

  • NoResponse() No response: This kAPI_OPT_NO_RESP property represents a switch that, if on, prevents the response from being sent. This can be useful when you are only interested in the status of the operation and not in the response.

The class also adds the following new operations:

  • kAPI_OP_GET_ONE kAPI_OP_GET_ONE: This is the tag that represents the findOne Mongo operation, it will return the first matched object.
  • kAPI_OP_GET_OBJECT_REF kAPI_OP_GET_OBJECT_REF: This tag defines a web service that returns an object by reference. It is equivalent to the kAPI_OP_GET_ONE kAPI_OP_GET_ONE operation, except that instead of using the query provided in the kAPI_DATA_QUERY kAPI_DATA_QUERY parameter, it will try to extract an identifier from the object provided in the kAPI_DATA_OBJECT kAPI_DATA_OBJECT parameter.

# NoResponse( integer $theValue = NULL, boolean $getOld = FALSE )

Manage no response switch.

Manage no response switch.

This method can be used to manage the no kAPI_OPT_NO_RESP response switch, it accepts a boolean which represents either the on/off value, or the requested operation:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • other: Set the value with the provided parameter.

The second parameter is a boolean which if TRUE will return the old value when replacing values; if FALSE, it will return the currently set value.
