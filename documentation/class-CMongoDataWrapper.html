Class CMongoDataWrapper

Mongo data wrapper.

This class overloads its CDataWrapper ancestor to implement a web-service that uses a MongoDB data store to manage objects.

This class implements the various elements declared in its CDataWrapper ancestor and adds the following options:

  • kAPI_OP_GET_ONE kAPI_OP_GET_ONE: This kAPI_OPERATION operation is equivalent to the kAPI_OP_GET kAPI_OP_GET operation, except that it will only return the first found element. It is equivalent to the Mongo findOne() method.
  • kAPI_OP_GET_OBJECT_REF kAPI_OP_GET_OBJECT_REF: This kAPI_OPERATION operation will return an object referenced by an identifier provided in the kAPI_DATA_OBJECT kAPI_DATA_OBJECT parameter. It is equivalent to the kAPI_OP_GET_ONE kAPI_OP_GET_ONE operation, except that instead of using the query provided in the kAPI_DATA_QUERY kAPI_DATA_QUERY parameter, it will try to extract an identifier from the object provided in the kAPI_DATA_OBJECT kAPI_DATA_OBJECT parameter. Remember to CDataType::SerialiseObject() serialise the reference before providing it to the wrapper.

This class also implements a static interface that can be used to UnserialiseObject() unserialise data flowing to and from the service parameters and the Mongo container.
