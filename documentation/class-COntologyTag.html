Class COntologyTag

Ontology tag.

A data element tagged by an instance of the COntologyTerm COntologyTerm class will have all its metadata stored in that term, but there are cases in which one term is not enough to describe the full metadata of a data element: for instance, a trait may be measured using different methods and workflows, and this data may also be measured in different units. For this reason this class extends the term COntologyTermObject base class to handle this case.

Instances of this class contain a list of ontology COntologyTerm terms whose elements are related between each other by predicates, which are also COntologyTerm terms, this kTAG_TERM attribute

This path or chain of COntologyTerm terms represents the unique identifier of this class instances and the tags with which data elements can be described.

The path root is a COntologyTerm term whose COntologyNode::Kind() kind must be kTYPE_TRAIT kTYPE_TRAIT which represents what the data element is and the path leaf is a COntologyTerm term whose COntologyNode::Kind() kind must be kTYPE_MEASURE kTYPE_MEASURE which represents the type or scale of the data; all the COntologyTerm terms found between these two describe how the data was measured or obtained. This path or chain represents the descriptors of data in this library and the container in which these objects are stored represents the data dictionary.

No two objects may exist with the same Path() path.

All instances of this class are uniquely GID() identified by a combination of two elements:

  • NS() Namespace: There is one instance of this class which represents the default namespace of all objects derived from this class, its GID() identifier is a '@' character (which is forbidden in other related term classes).
  • Code() Code: The aforementioned namespace contains an integer field which represents a sequence counter which will be used as the object's Code() code.

The class features the following properties:

  • kTAG_PATH kTAG_PATH: This Path() attribute holds the list of COntologyTerm term COntologyTerm::GID() identifiers representing the tag path, the attribute is a string formed by the concatenation of all the COntologyTerm term COntologyTerm::GID() identifiers structured as follows: TERM/PREDICATE/TERM/PREDICATE/.../TERM where all items are COntologyTerm term COntologyTerm::GID() identifiers. This property represents the object's unique identifier, no two records can have this same value.
  • kTAG_TERM kTAG_TERM: This Term() attribute holds the list of COntologyTerm terms featured in the Path() path as an array of COntologyTerm term kTAG_LID identifiers in which odd elements represent the subjects and objects of the relationships and the even elements the relationship predicates.
  • kTAG_UID kTAG_UID: This attribute holds the hashed value of the Path() path: it will be used to detect duplicate records.
  • kTAG_REF_COUNT kTAG_REF_COUNT: This RefCount() attribute holds the count of data instances that refer to the current tag, or the number of data instances tagged by the current path. This attribute is required and is initialised to 0.

public mixed
# Term( mixed $theValue = NULL )

Manage terms.

Manage terms.

This method can be used to add COntologyTerm term elements of the path, it accepts a single parameter which represents either the elements to be added or the operation to be performed:

  • NULL: This value indicates that we want to retrieve the whole list.
  • FALSE: This value indicates that we want to delete the whole list.
  • array: The elements of the array will be added to the list by recursing this method.
  • other: Any other type will be analysed as follows:
    • COntologyTerm COntologyTerm: The term's kTAG_GID identifier will be used.
    • COntologyEdge COntologyEdge: The edge elements will be handled as follows:
      • The current list is empty: The three terms will make the list.
      • The current list exists: The method will check if the edge's subject matches the last element in the list: if this is the case, the method will add the predicate and object terms; if not, it will raise an exception.
    • COntologyEdgeIndex COntologyEdgeIndex: The same treatment as for the COntologyEdge COntologyEdge.
    • other: Any other type will be cast to a string and interpreted as the COntologyTerm term's kTAG_GID identifier.

All elements will be appended to the list in the order they were provided, it must be noted that the class expects this list to be a sequence of term/predicate/term elements, this method is not responsible for checking this.

The method will prevent modifications if the object is _IsCommitted() committed, because this would change the object's unique kTAG_UID identifier.

The method will return the current full list.

Get new sequence.

Get new sequence.

This method will return the next sequence number and increment the existing one. The sequence is stored in a singleton record identified by the kTAG_SINGLETON_ID kTAG_SINGLETON_ID character, this record contains a single kTAG_COUNT attribute which represents the next available sequence number.

When retrieving the sequence, if the main record does not exist, this method will create one with a first value of 1.
