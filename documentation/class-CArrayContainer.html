Class CArrayContainer

Array persistent data store.

This class extends its CContainer ancestor to implement a concrete object store instance that uses arrays or ArrayObject objects to store data.

# _Commit( reference & $theObject, reference & $theIdentifier, reference & $theModifiers )

Commit provided object.

Commit provided object.

We implement this method to handle array or ArrayObject data stores and we ensure provided options are followed.

When kFLAG_PERSIST_MODIFY modifying the contents of the object, we perform the following checks:

  • Both matched object and provided object are arrays: this is the default scenario. If the provided object element is NULL, the corresponding element in the matched object will be deleted; if not NULL, the provided object element will replace the eventual existing one or be set in the matched object.
  • Matched object is array and provided object is scalar: the provided object will be appended to the matched object.
  • Matched object is scalar and provided object is array: we transform the matched object by appending the existing element to an empty array, and we set all the elements of the provided element into the newly created array.
  • Matched object and provided object are scalar: we transform the matched object by appending the existing element to an empty array, and we append the provided object to it.

By default the object must be an array or ArrayObject, any other type will raise an kERROR_INVALID_PARAMETER exception.

The provided identifier will be cast to a string. If it is NULL, it means that the object is to be appended in the container and the method assumes the Commit() caller has determined that it is an kFLAG_PERSIST_INSERT insert operation.

Although the Commit() caller accepts the kFLAG_PERSIST_DELETE delete option, in this class we do not, so we shall raise an kERROR_INVALID_PARAMETER exception.
