Class CContact

Contact.

This class overloads its CEntity ancestor to implement a contact entity.

Contacts are entities that need to be tracked by Mail() mail, Phone() telephone or Email() e-mail.

The class features the following properties:

  • kOFFSET_MAIL kOFFSET_MAIL: The contact Mail() mailing address list.
  • kOFFSET_PHONE kOFFSET_PHONE: The contact Phone() telephone list.

These three properties are managed as an array of elements discriminated by the kTAG_KIND kTAG_KIND offset, no two elements may share the same kTAG_KIND kind.

The object inherits the _IsInited() inited kFLAG_STATE_INITED status, it is the responsibility of concrete classes to implement this feature.

Manage mailing address.

This method can be used to manage the contact kOFFSET_MAIL mailing address, it manages an array of structures with the following offsets:

  • kTAG_KIND kTAG_KIND: The mailing address kind, this could be home, work or other. This element represents the array key, although technically it is implemented as an element to allow searching on all values.
  • kTAG_DATA kTAG_DATA: The mailing address structure or string, this element should hold the actual address.

The parameters to this method are:

  • $theValue: The value or operation:
    • NULL: Return the current value selected by the second parameter.
    • FALSE: Delete the value selected by the second parameter.
    • other: Set value selected by the second parameter.
  • $theType: The element type, kind or index:
    • NULL: This value indicates that the adress has no type or kind, in general, when adding elements, this case applies to default elements.
    • other: All other types will be interpreted as the kind or type of the address.
  • $getOld: Determines what the method will return:
    • TRUE: Return the value before it was eventually modified.
    • FALSE: Return the value after it was eventually modified.
