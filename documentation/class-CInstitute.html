Class CInstitute

Institute.

This class overloads its CContact ancestor to implement an institute contact entity.

Institutes feature specific properties that are added to the inherited ones:

  • kOFFSET_ACRONYM kOFFSET_ACRONYM: This offset represents the institute list of Acronym() acronyms.
  • kOFFSET_URL kOFFSET_URL: This offset represents the institute list of URLs or web pages.

The object is considered _IsInited() initialised only if it has its Code() code, as its CEntity ancestor, its Name() name, Password() password and Email() e-mail address.

If the Code() code has not been explicitly set, _PrepareCommit() before Commit() committing the object it will be set to the value of the Email e-mail. Also in that phase, the kENTITY_USER kENTITY_USER constant will be set in the user Type() type.

The Email() e-mail in this class is a scalar property, in other classes it will probably be a list of different e-mail types. In this class we want to link a single user with a single e-mail, possibly not shared by any other user, that is why we link by default the user Code() code and Email() e-mail.

The unique identifier of this class is composed by the default _index() index of thew object, prefixed by the kENTITY_INST kENTITY_INST token and the kTOKEN_CLASS_SEPARATOR kTOKEN_CLASS_SEPARATOR token, this allows institutes and other types of CEntity entities to share the same Code() code; this is enforced both in the HashIndex() HashIndex method, to which you only need to pass the user Code() code, and in the protected _PrepareCommit() _PrepareCommit method which will place the resulting string in the global kTAG_GID identifier.

Manage entity acronyms.

This method can be used to handle the institute kOFFSET_ACRONYM acronyms list, it uses the standard accessor CAttribute::ManageArrayOffset() method to manage the list of acronyms.

Each element of this list should indicate an acronym by which one refers to the current institute, the nature and specifics of these elements is the responsibility of concrete classes.

Manage institute urls.

Manage institute urls.

This method can be used to manage the institute kOFFSET_URL URL or web pages list, the method expects the following parameters:

  • $theValue: The value or operation:
    • NULL: Return the current value selected by the second parameter.
    • FALSE: Delete the value selected by the second parameter.
    • other: Set value selected by the second parameter.
  • $theType: The element type, kind or index:
    • NULL: This value indicates that the URL has no type or kind, in general, when adding elements, this case applies to default elements.
    • other: All other types will be interpreted as the kind or type of the URL.
  • $getOld: Determines what the method will return:
    • TRUE: Return the value before it was eventually modified.
    • FALSE: Return the value after it was eventually modified.
