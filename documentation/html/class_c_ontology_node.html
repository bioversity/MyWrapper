Create node property indexes.

This method will save node indexes after the node was committed, there are two main indexes for node properties:

  • kINDEX_NODE_NODE: This index (NodeIndex) links the node to its properties through the following keys:
    • kTAG_TYPE: This key links the current node to its type, which may either be inherited from its term or have been explicitly set.

    • kTAG_DOMAIN: This key links the current node to its domains, which may either be inherited from its term or have been explicitly set.
    • kTAG_CATEGORY: This key links the current node to its categories, which may either be inherited from its term or have been explicitly set.

Manage scalar property.

This method will set, retrieve and delete array element properties, it accepts the following parameters:

  • $theOffset: The property offset to manage.
  • $theValue: This parameter represents either the value to add, or the index of the element to operate on:
    • NULL: This value indicates that we want to operate on all elements, which means that we are retrieving the full list or deleting it.
    • array: This value indicates that we want to replace the whole list, this will only be tested if the next parameter evaluates to TRUE.
    • other: Any other type represents either the new value to be added or the index to the value to be returned or deleted. It must be possible to cast this value to a string, this is what will be used to compare elements.
  • $theOperation: This parameter represents the operation to be performed, it will be evaluated as a boolean and its scope depends on the value of the previous parameter:
    • NULL: Return the element or list.
    • FALSE: Delete the element or list.
    • TRUE: Add the element or list. Note that with this value, if you provide NULL in the previous parameter, it will be equivalent to deleting the whole list.
  • $getOld: Determines what the method will return:
    • TRUE: Return the element or list before it was eventually modified.
    • FALSE: Return the element or list after it was eventually modified.

COntologyNode::GetRelated (   $theContainer,
  $thePredicate = NULL,
  $theDirection = 'all' 
)

Return related edges.

This method will return all nodes that are related to the current one in an array of edge nodes whose elements are structured as follows:

  • Key: The edge ID.
  • Value: The edge object.

The method accepts the following parameters:

  • $theContainer: The graph and term containers as an array:
    • kTAG_NODE: This element should hold the nodes container, it must be a Everyman instance.
    • kTAG_TERM: This element should hold the terms container, it must be a CContainer instance.
  • $thePredicate: The predicate terms as an array of the following types:
    • COntologyNode: The node term global identifier.
    • COntologyTerm: The term global identifier will be used as the node type.
    • Everyman: The relationship's type will be used as the predicate, all other elements of the provided edge node will be ignored.
    • string: Any other type will be converted to string and will be used as the node type.
  • $theDirection: The direction of the relation:
    • out: Outgoing relations, the current node is the subject.
    • in: Incoming relations, the current node is the object.
    • all: Incoming and outgoing relations (default), the current node is the object or the subject.

COntologyNode::RelateTo (   $theContainer,
  $thePredicate,
  $theObject = NULL 
)

Create a graph edge.

This method can be used to create a graph edge or relation between the current node and an object node using a predicate node. This method will not duplicate relationships between the same nodes and the predicate term: it uses the kINDEX_NODE_TERM relationship index and its kTAG_EDGE_NODE key to locate existing relationships. For that reason the subject and object terms of the relationship must relate to committed nodes.

The method accepts the following parameters:

  • $theContainer: The graph and term containers as an array:
    • kTAG_NODE: This element should hold the nodes container, it must be a Everyman instance.
    • kTAG_TERM: This element should hold the terms container, it must be a CContainer instance.
  • $thePredicate: The predicate term:
    • COntologyNode: The node term global identifier.
    • COntologyTerm: The term global identifier will be used as the node type.
    • Everyman: The relationship's type will be used as the predicate, all other elements of the provided edge node will be ignored.
    • string: Any other type will be converted to string and will be used as the node type.
  • $theObject: The destination node or relationship object node:
    • COntologyNode: The method will use it's node.
    • Everyman: The method will use it as the relationship object.
    • integer: The method will search for the node corresponding to the provided number, if the node was not found, the method will raise an exception.
    • other: Any other type will raise an exception.

The method will return a COntologyEdge object, or raise an exception if the operation was not successful.

Parameters:
reference$theContainerObject container.
mixed$thePredicatePredicate.
mixed$theObjectObject.
