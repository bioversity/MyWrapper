Return the object's unique identifier.

This method can be used to return a string value that represents the object's unique identifier. When saving the object for the first time, if this method returns a value, this will be used as the object's ID.

If this method returns NULL, it is assumed that the container will provide a default unique value.

Manage a list of objects.

This method can be used to manage a list of objects derived from this class, in particular, an array in which each element is either a string representing the identifier of the object, or the object itself.

Each element of the array is identified either by the value itself, if it is a string representing the object identifier, or by the identifier if the element is an object itself.

The array is numerically indexed and this method will ensure it remains an array.

The parameters to this method are:

  • $theOffset: This parameter represents the offset in the current object that holds the list of objects. This value may not be empty.
  • $theValue: This parameter may represent either the identifier of an element in the list, if the operation involves retrieving or deleting, or the actual contents if the operation involves adding or replacing into the list. Values provided as instances derived from this class will be converted to the identifier they hold if the operation involves retrieving or deleting; if these objects do not have this identifier, the method will attempt to use the result of the _id method; if also this returns a NULL value we shall raise an exception. If you provide an array in this parameter, the method will assume you want to use the elements of the array as keys or values, in that case it will send each element to this method using the same other parameters.
  • $theOperation: The operation to perform:
    • NULL: Return the element identified by the second parameter.
    • FALSE: Delete the element identified by the second parameter, if the next parameter evaluates to TRUE, the method will return the deleted element if available.
    • other: Any other value means that we want to add/replace the element provided in the second parameter. This means that the method will scan all the elements of the array and replace the element matching the identifier, or append the element if there is no match.
  • $getOld: Determines what the method will return when deleting elements:
    • TRUE: Return the value of the offset before it was eventually modified or deleted.
    • FALSE: Return the value of the offset after it was eventually modified or deleted.

Since the list managed by this method is a numeric keyed array, it is important that elements of the array have the identifier, or deleting elements from the list may prove problematic.

In general, objects holding such lists will have all their elements stored as instances derived from this class committed and converted to a string containing the object identifier.

Return object index.

This method is used to determine an object index, the method accepts a single parameter that may be:

  • CPersistentUnitObject: In this case we interpret the parameter to be an instance of the object for which we want to retrieve the identifier:
    • kTAG_ID_NATIVE: We first check whether the object has that offset and use it is so.
    • _id: We then check the result of this method and use it if not NULL.
  • string: In this case we interpret the parameter to be the identifier of an object and cast it to a string.
  • other: Any other value will be cast to a string.

CPersistentUnitObject::_PrepareStore ( &$  theContainer,
&$  theIdentifier 
) [protected]

Normalise before a store.

The duty of this method is to ensure that the parameters provided to the store operation are correct.

In this class we ensure that the container is a ArrayObject or a CContainer derived instance and we ensure the identifier is filled in the case it was not provided:

  • Get kTAG_ID_NATIVE: If the object features the kTAG_ID_NATIVE offset, it will be preferred. This is necessary, because we don't want the object identifier to change in time.
  • Use the :id method: We use the value returned by the _id protected method, this will be the case when saving new objects.

We also handle here the other two default offsets:

  • kTAG_CLASS: We set this offset with the current object's name. Note that we overwrite old values.
  • kTAG_VERSION: If not set, we initialise this value to zero, if already set, we increment it.
