CWarehouseWrapper::_Handle_GetEdges ( )
protected

Handle get-edges request.

This method will return the following structure:

  • kAPI_RESPONSE_TERMS: The list of terms related to the list of subject and object nodes and the list of predicate terms as follows:
    • Index: The term identifier.
    • Value: The term properties.
  • kAPI_RESPONSE_NODES: The list of subject and object nodes as follows:
    • Index: The node ID.
    • Value: The node properties.
  • kAPI_RESPONSE_EDGES: The list of edges as an array structured as follows:
    • Index: The edge identifier.
    • Value: An array structured as follows:
      • kAPI_RESPONSE_SUBJECT: The subject node ID.
      • kAPI_RESPONSE_PREDICATE: The predicate term identifier.
      • kAPI_RESPONSE_OBJECT: The object node ID.

The method will interpret the contents of the kAPI_OPT_IDENTIFIERS parameter depending on whether the kAPI_OPT_DIRECTION parameter was provided or not:

  • kAPI_OPT_DIRECTION not provided: In this case the method will treat the kAPI_OPT_IDENTIFIERS parameter elements as a list of edge identifiers to be matched.
  • kAPI_OPT_DIRECTION provided: In this case the method will treat the kAPI_OPT_IDENTIFIERS parameter elements as a list of node identifiers for which we want to retrieve connected edges in the direction provided in the kAPI_OPT_DIRECTION parameter:
    • kAPI_DIRECTION_IN: The service will return all edges that point to the nodes provided in the kAPI_OPT_IDENTIFIERS parameter.
    • kAPI_DIRECTION_OUT: The service will return all edges pointing from the nodes provided in the kAPI_OPT_IDENTIFIERS parameter.
    • kAPI_DIRECTION_ALL: The service will return all edges connected in any way to the nodes provided in the kAPI_OPT_IDENTIFIERS parameter.

If the kAPI_OPT_PREDICATES parameter was provided, only those edges whose type matches any of the predicate term identifiers provided in that parameter will be selected.

If the kAPI_OPT_IDENTIFIERS parameter was not provided, the method will return the above structure with no content.

CWarehouseWrapper::_Handle_GetNodes ( )
protected

Handle get-nodes request.

This method expects the kAPI_OPT_IDENTIFIERS parameter to hold a list of node IDs, the method will query these nodes and return the following structure:

  • kAPI_RESPONSE_TERMS: The list of terms related to the list of nodes as follows:
    • Index: The term identifier.
    • Value: The term properties.
  • kAPI_RESPONSE_NODES: The list of nodes as follows:
    • Index: The node ID.
    • Value: The node properties.

If the kAPI_OPT_IDENTIFIERS parameter was not provided, the method will return the above structure with no content.

&  $theEdges,
  $theLevel

protected

Traverse graph.

This method will traverse the graph in the direction set in the kAPI_OPT_DIRECTION parameter and collect all found edge elements in the provided parameters.

The parameters to this method are:

  • &$theList: List of current level node identifiers.
  • &$theTerms: Reference to the list of term identifiers.
  • &$theNodes: Reference to the list of nodes.
  • &$theEdges: Reference to the list of edge elements.
  • $theLevel: Current traversal depth level, 0 means the lowest.

The method will return the number of processed elements count.

Parameters:
reference&$theListCurrent level node identifiers.
reference&$theTermsList of term identifiers.
reference&$theNodesList of nodes.
reference&$theEdgesList of edge references.
integer$theLevelDepth level.
