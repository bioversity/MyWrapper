Instantiate class.

The constructor will instantiate an object reference in three ways:

  • Empty reference: Both parameters are omitted or both are NULL. In this case you will have to build the reference by yourself.
  • Object reference: The first parameter is provided, the second is not. In this case we assume the first parameter is already an object reference.
  • Object: Both parameters are provided, in this case we assume the first parameter is the object to be referenced and the second parameter is the collection from which it was taken; in this case we also record the object class in the reference.

In the presence of both parameters, the provided collection takes the precedence over the referenced collection (and database).

When providing either a reference or an object, the class reference will be set only if the provided reference or object has it.

The parameters to this method are:

  • $theReference: Either a reference or an object.
  • $theContainer: The collection in which the referenced object resides.

Manage referenced object class.

This method can be used to manage the referenced object's class, it uses the standard accessor method to manage the offset:

  • $theValue: The value or operation:
    • NULL: Return the current value.
    • FALSE: Delete the value.
    • other: Set value.
  • $getOld: Determines what the method will return:
    • TRUE: Return the value before it was eventually modified.
    • FALSE: Return the value after it was eventually modified.
