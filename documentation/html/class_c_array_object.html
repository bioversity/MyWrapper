Manage an array offset.

This library implements a standard interface for managing object properties using methods, this method extends the approach to offset members that are in the form of arrays, in which it should be possible to cast the values to strings and these strings should be unique within the list:

  • $theOffset: The offset to manage.
  • $theValue: This parameter represents either the value to add, or the index of the element to operate on:
    • NULL: This value indicates that we want to operate on all elements, which means that we are either retrieving the full list or deleting it.
    • array: This value indicates that we want to replace the whole list, this will only be tested if the next parameter evaluates to TRUE.
    • other: Any other type represents either the new value to be added or the index to the value to be returned or deleted. It must be possible to cast this value to a string, this is what will be used to compare elements.
  • $theOperation: This parameter represents the operation to be performed, it will be evaluated as a boolean and its scope depends on the value of the previous parameter:
    • NULL: Return the element or list.
    • FALSE: Delete the element or list.
    • TRUE: Add the element or list. Note that with this value, if you provide NULL in the previous parameter, it will be equivalent to deleting the whole list.
  • $getOld: Determines what the method will return:
    • TRUE: Return the element or list before it was eventually modified.
    • FALSE: Return the element or list after it was eventually modified.

Manage a typed array offset.

This library implements a standard interface for managing object properties using methods, this method extends the approach to offset members that are in the form of arrays, in which each element is itself an array of two items:

  • kTAG_TYPE: This element represents the qualifier of the item, it provides a type or qualification for the next element. It must be possible to cast this value to a string.
  • kTAG_DATA: This element represents the element data.

No two elements of the list can share the same type. You may have one element without type, this one may be considered the default element.

This method is intended for managing list elements rather than the list itself, for the latter purpose use the offset management methods.

The parameters to this method are:

  • $theOffset: The offset to manage.
  • $theType: This parameter represents the value of the type element of the item, depending on the next parameter this value will be used for matching items in the list:
    • NULL: An empty type means that we are looking for the item lacking the kTAG_TYPE tag.
    • array: If you provide an array, it means that you are operating on a list of items: depending on the next parameter this will mean either retrieving the data elements of the items matching the array, deleting these items, or adding/replacing the items; in this last case, this means that the next parameter must also be an array and that each of its elements will be associated to the corresponding type element.
    • other: Any other value will be considered as the type to retrieve, remove or add/replace. You MUST be able to cast this value to a string.
  • $theData: This parameter represents the item's data element, or the operation to be performed:
    • NULL: This indicates that we want to retrieve the data of the item with type matching the previous parameter.
    • FALSE: This indicates that we want to remove the item matching the type provided in the previous parameter.
    • other: Any other value indicates that we want to add or replace the data element of the item matching the previous parameter. Note that if the previous parameter is an array, this one must also be an array matching the latter's count.
  • $getOld: Determines what the method will return:
    • TRUE: Return the element or list before it was eventually modified.
    • FALSE: Return the element or list after it was eventually modified.

CArrayObject::offsetGet ( $  theOffset)

Return a value at a given offset.

This method should return the value corresponding to the provided offset.

This method is overloaded to prevent notices from being triggered when seeking non-existing offsets.

In this class no offset may have a NULL value, if this method returns a NULL value, it means that the offset doesn't exist.
