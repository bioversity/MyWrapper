COntologyTermObject::_CheckReference (   $theReference)
protected

Normalise reference parameter.

This method can be used to normalise a parameter that is supposed to be a reference to another term, that is, a binary string hash of the term's identifier converted to a binary standard type.

The method will perform the following conversions:

  • CDataTypeBinary: This is the default data type for the identifier.
  • MongoBinData: This type will be converted to the standard CDataTypeBinary type.
  • COntologyTermObject: Objects of the same class will have their identifier extracted.
  • NULL: NULL data will simply be passed.
  • other: Any other data type is assumed to be the term's identifier, so it will be hashed into a binary string and converted to the standard CDataTypeBinary type.
  • array: Arrays cannot be converted to string, so the method will raise an exception.

COntologyTermObject::Definition (   $theValue = NULL,
  $theLanguage = NULL,
  $getOld = FALSE 
)

Manage term definition.

This method can be used to manage the term definition, it manages an array of structures with the following offsets:

  • kTAG_LANGUAGE: The definition's language, this element represents the code of the language in which the next element is expressed in.
  • kTAG_DATA: The term definition or description.

The parameters to this method are:

  • $theValue: The definition or operation:
    • NULL: Return the current value selected by the second parameter.
    • FALSE: Delete the value selected by the second parameter.
    • other: Set value selected by the second parameter.
  • $theLanguage: The definition's language code:
    • NULL: This value indicates that the definition has no language, in general, when adding elements, this case applies to default elements.
    • other: All other types will be interpreted as the language code.
  • $getOld: Determines what the method will return:
    • TRUE: Return the value before it was eventually modified.
    • FALSE: Return the value after it was eventually modified.

Manage synonyms.

This method can be used to manage the term synonyms list, these elements are strings that can be considered synonyms of the current term.

This property is organised as an array of items structured as follows:

  • kTAG_KIND: The synonym kind, its value is provided in the $theType parameter.
  • kTAG_DATA: The synonym values organised as an array of strings.

The method expects the following parameters:

  • $theValue: The synonym value.
