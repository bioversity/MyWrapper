COntologyEdge::_IndexTerms ( Everyman\Neo4j\Client  $theContainer)
protected

Create node indexes.

This method will save node indexes after the node was committed, it will perform the following selections:

  • kINDEX_EDGE_TERM: The term global identifier (RelationshipIndex). container, it must be a Everyman instance.
  • kINDEX_EDGE_NAME: The term names in all languages (RelationshipIndex).
  • kINDEX_EDGE_TERMS: The node relations, this index records the relation terms, that is, the combination of the subject term, predicate term and the object term, this can be used to retrieve existing relations.

The following index tags are set:

  • kTAG_GID: The term global identifier.
  • kTAG_NAME: The term names.
  • kTAG_EDGE_TERM: The relationships between terms taken from the node's kTAG_EDGE_TERM property, formatted as a SUBJECT/PREDICATE/OBJECT string, in which each element is the global term identifier.
  • kTAG_NODE: The relationships between nodes, expressed as a SUBJECT/PREDICATE/OBJECT string, in which the subject and object are the node identifiers and the predicate is the global term identifier.

Normalise parameters of a create.

In this class we first check whether the container has the following structure:

  • kTAG_NODE: This element should hold the nodes container, it must be a Everyman instance.
  • kTAG_TERM: This element should hold the terms container, it must be a CContainer instance.

If the container has the correct structure the node container will be passed to the parent method and the method will check if the term container is a COntologyTermObject.

COntologyEdge::ObjectTerm (   $theValue = NULL,
  $getOld = FALSE 
)

Manage object term.

This method can be used to manage the object node term reference, it uses the standard accessor method to manage the property:

  • $theValue: The value or operation:
    • NULL: Return the current value.
    • FALSE: Delete the value.
    • COntologyTermObject: Set value.
    • other: Raise exception.
  • $getOld: Determines what the method will return:
    • TRUE: Return the value before it was eventually modified.
    • FALSE: Return the value after it was eventually modified.

The method will also set the dirty status and the inited status if the node is provided.
