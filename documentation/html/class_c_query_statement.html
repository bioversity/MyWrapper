Instantiate class.

The constructor can be used to instantiate a statement either by providing an existing statement structure, or by providing the statement elements:

  • $theSubject: The statement subject:
    • array or ArrayObject: Structures are interpreted as built statements, so the method will scan the structure and load the corresponding elements.
    • string: Any other type will be converted to a string and interpreted as the statement subject, or data element key.
  • $thePredicate: The statement operator or predicate:
    • kOPERATOR_DISABLED: Disabled, it means that the current statement is disabled; the remaining parameters are ignored.
    • kOPERATOR_EQUAL: Equality (=), this operator implies that the method expects the next two parameters.
    • kOPERATOR_EQUAL_NOT: Inequality (!=), negates the kOPERATOR_EQUAL operator; this operator implies that the method expects the next two parameters.
    • kOPERATOR_LIKE: Like, it is an accent and case insensitive equality filter, this operator implies that the method expects the next two parameters.
    • kOPERATOR_LIKE_NOT: The negation of the LIKE operator, this operator implies that the method expects the next two parameters.
    • kOPERATOR_PREFIX: Starts with, or prefix match, this operator implies that the method expects the next two parameters.
    • kOPERATOR_CONTAINS: Contains, selects all elements that contain the match string, this operator implies that the method expects the next two parameters.
    • kOPERATOR_SUFFIX: Ends with, or suffix match, this operator implies that the method expects the next two parameters.
    • kOPERATOR_REGEX: Regular expression, this operator implies that the method expects the next two parameters.
    • kOPERATOR_LESS: Smaller than (<), this operator implies that the method expects the next two parameters.
    • kOPERATOR_LESS_EQUAL: Smaller than or equal (<=), this operator implies that the method expects the next two parameters.
    • kOPERATOR_GREAT: Greater than (>), this operator implies that the method expects the next two parameters.
    • kOPERATOR_GREAT_EQUAL: Greater than or equal (>=), this operator implies that the method expects the next two parameters.
    • kOPERATOR_IRANGE: Range inclusive, matches >= value <=; this operator implies that the method expects the next three parameters.
    • kOPERATOR_ERANGE: Range exclusive, matches > value <, this operator implies that the method expects the next three parameters.
    • kOPERATOR_NULL: Is NULL or element is missing.
    • kOPERATOR_NOT_NULL:Not NULL or element exists.
    • kOPERATOR_IN: In, or belongs to set, this operator implies that the method expects the next two parameters.
    • kOPERATOR_NI: Not in, the negation of kOPERATOR_IN, this operator implies that the method expects the next two parameters.
    • kOPERATOR_ALL: All, or match the full set, this operator implies that the method expects the next two parameters.
    • kOPERATOR_NALL: Not all, the negation of the kOPERATOR_ALL operator, this operator implies that the method expects the next two parameters.
    • kOPERATOR_EX: Expression, indicates a complex expression, this operator implies that the method expects the next two parameters.
  • $theType: The statement object data type, this qualifies all remaining parameters. The allowed values are:
    • kTYPE_STRING: String, we assume in UTF8 character set, the string is expected in the next parameter.
    • kTYPE_INT32: 32 bit signed integer, the number is expected in the next parameter, either as an integer, float or string; once received, it will be converted to a CDataTypeInt32 object.
    • kTYPE_INT64: 64 bit signed integer, the number is expected in the next parameter, either as an integer, float or string; once received, it will be converted to a CDataTypeInt64 object.
    • kTYPE_FLOAT: Floating point number, the number is expected in the next parameter, either as an integer, float or string.
    • kTYPE_DATE: A string date, it is treated as a string date with a YYYYMMDD format in which month and day may be omitted.
    • kTYPE_TIME: A string time, it is treated as a string time with a YYYY-MM-DD HH:MM:SS format in which all elements are required; this element will be converted to a kTYPE_STAMP data type.
    • kTYPE_STAMP: A timestamp, optionally including microseconds, elements of this type will be converted to CDataTypeStamp objects.
    • kTYPE_BOOLEAN: An on/off switch, it will be converted to a 1/0 pair.
    • kTYPE_BINARY: A binary string, the string will be converted to a CDataTypeBinary object.
  • $theObject: The statement object data, it should reflect the data type provided in the previous parameter.
  • $theRange: The statement range end or second element. This value must also reflect the data type provided in the previous parameter and will be automatically set if you provided a range and forgot to set it.

Create an all statement.

This method can be used to instantiate an all query statement which will test whether all the subject values can be found in the provided object, which will be interpreted as a list of values.

If the provided object is not an array or an ArrayObject, the method will add it to a newly created array.

The statement uses the kOPERATOR_ALL operator and this method accepts the following parameters:

  • $theSubject: Statement subject.
  • $theObject: Statement object, or members list.
  • $theType: Statement object data type.

The method will return an instance of this class or raise an exception on errors.

  $getOld = FALSE 
)

Manage predicate.

This method can be used to manage the query operator or predicate, it accepts a parameter which represents either the statement predicate or the requested operation, depending on its value:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • other: Set the value with the provided parameter:
    • kOPERATOR_DISABLED: Disabled, it means that the current statement is disabled.
    • kOPERATOR_EQUAL: Equality (=).
    • kOPERATOR_EQUAL_NOT: Inequality (!=), negates the kOPERATOR_EQUAL operator.
    • kOPERATOR_LIKE: Like, it is an accent and case insensitive equality filter.
    • kOPERATOR_LIKE_NOT: The negation of the LIKE operator.
    • kOPERATOR_PREFIX: Starts with, or prefix match.
    • kOPERATOR_CONTAINS: Contains, selects all elements that contain the match string.
    • kOPERATOR_SUFFIX: Ends with, or suffix match.
    • kOPERATOR_REGEX: Regular expression.
    • kOPERATOR_LESS: Smaller than (<).
    • kOPERATOR_LESS_EQUAL: Smaller than or equal (<=).
    • kOPERATOR_GREAT: Greater than (>).
    • kOPERATOR_GREAT_EQUAL: Greater than or equal (>=).
    • kOPERATOR_IRANGE: Range inclusive, matches >= value <=.
    • kOPERATOR_ERANGE: Range exclusive, matches > value <.
    • kOPERATOR_NULL: Is NULL or element is missing.
    • kOPERATOR_NOT_NULL:Not NULL or element exists.
    • kOPERATOR_IN: In, or belongs to set.
    • kOPERATOR_NI: Not in, the negation of kOPERATOR_IN.
    • kOPERATOR_ALL: All, or match the full set.
    • kOPERATOR_NALL: Not all, the negation of the kOPERATOR_ALL operator.
    • kOPERATOR_EX: Expression, indicates a complex expression.
    If the provided value does not match any of the above, the method will raise an exception.

The second parameter is a boolean which if TRUE will return the old value when replacing values; if FALSE, it will return the currently set value.

Manage data type.

This method can be used to manage the query data type or object data type, it accepts a parameter which represents either the data type in which the object is expressed, or the requested operation, depending on its value:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • other: Set the value with the provided parameter:
    • kTYPE_STRING: String, we assume in UTF8 character set.
    • kTYPE_INT32: 32 bit signed integer.
    • kTYPE_INT64: 64 bit signed integer.
    • kTYPE_FLOAT: Floating point number.
    • kTYPE_DATE: A string date, it means a string date with a YYYYMMDD format in which month and day may be omitted.
    • kTYPE_TIME: A string time, it is treated as a string time with a YYYY-MM-DD HH:MM:SS format in which all elements are required.
    • kTYPE_STAMP: A timestamp, optionally including microseconds.
    • kTYPE_BOOLEAN: An on/off switch.
    • kTYPE_BINARY: A binary string.
    If the provided value does not match any of the above, the method will raise an exception.

The second parameter is a boolean which if TRUE will return the old value when replacing values; if FALSE, it will return the currently set value.
