Class CUser

User.

This class overloads its CEntity ancestor to implement a user entity.

Users are entities that are used for authentication purposes, they share the same attributes as their parent CEntity class and add one required element:

  • kOFFSET_PASSWORD kOFFSET_PASSWORD: This offset represents the user access password, it will be used in the authentication process. The class features a member accessor Password() method to manage this property.

The object is considered _IsInited() initialised only if it has its Code() code, as its CEntity ancestor, its Name() name, Password() password and Email() e-mail address.

If the Code() code has not been explicitly set, _PrepareCommit() before Commit() committing the object it will be set to the value of the Email e-mail. Also in that phase, the kENTITY_USER kENTITY_USER constant will be set in the user Type() type.

The Email() e-mail in this class is a scalar property, in other classes it will probably be a list of different e-mail types. In this class we want to link a single user with a single e-mail, possibly not shared by any other user, that is why we link by default the user Code() code and Email() e-mail.

Note: this class enforces the _IsEncoded() encoded Status() status kFLAG_STATE_ENCODED flag, because the object identifier is a binary string, so always use complex data type instances derived from the CDataType standard types.

The unique identifier of this class is composed by the default _index() index of the object, prefixed by the kENTITY_USER kENTITY_USER token and the kTOKEN_CLASS_SEPARATOR kTOKEN_CLASS_SEPARATOR token, this allows users and other types of CEntity entities to share the same Code() code; this is enforced both in the HashIndex() HashIndex method, to which you only need to pass the user Code() code, and in the protected _PrepareCommit() _PrepareCommit method which will place the resulting string in the global kTAG_GID identifier.
