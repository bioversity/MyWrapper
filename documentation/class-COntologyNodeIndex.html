Class COntologyNodeIndex

Node index.

This class implements a graph node index, it can be used to replicate a Neo4j node to a MongoDB collection.

Neo4j uses Lucene as its standard indexing tool, in this library we use MongoDB as the standard database, so it is a logical choice to use Mongo as the indexing mechanism for Neo4j. Also, by storing nodes in Mongo, by dumping the database contents we also have the graph structure with it.

The class features a single data member, the Node() node which contains a Neo4j node reference. When __construct() instantiating this class you are required to provide a persistent instance of a node.

The contents of the object will be the node properties as will be stored in the Mongo kDEFAULT_CNT_NODES default collection for nodes:

  • kTAG_LID kTAG_LID: This offset will hold the node ID.
  • kTAG_DATA kTAG_DATA: This offset will hold the node properties.

The class provides a single member accessor method: Node() Node, which can be used to set the referenced node.

Note that the class will not cast to an array correctly, you must use the getArrayCopy() getArrayCopy method to get an array, if you know how to solve this, please do it!

# __construct( mixed $theNode, mixed $theContainer = NULL )

Instantiate class.

Instantiate class.

The constructor can be used to instantiate an object with a node, there is no provision for instantiating an empty object.

The method expects the following parameters:

  • $theNode: The referenced node, this data can come in three flavours:
    • Neo4j node: This is the expected format, the node must be persistent, that is, it must have an ID. In this case the second parameter is ignored.
    • integer or numeric string: The value refers to the node ID, in this case the node will have to be loaded from a Neo4j client, which must be provided in the next parameter.
    • CMongoQuery query: In this case we assume the value represents a CMongoQuery query that will be sent to the index, only the first matching record will be used.
  • $theContainer: The nodes container, depending on the first parameter:
    • Neo4j node: The parameter will be ignored.
    • integer or numeric string: The parameter is expected to be an instance of a Neo4j client, it will be used to locate the node in the graph.
    • array: The parameter is expected to have two elements:
      • kTAG_NODE kTAG_NODE: The Neo4j client (graph container).
      • kTAG_TERM kTAG_TERM: The Mongo container (index container), which can take the following types:
        • MongoDB: The database will be used to instantiate a MongoCollection named as the kDEFAULT_CNT_NODES kDEFAULT_CNT_NODES tag.
        • MongoCollection: The provided collection will be used.
        • CMongoContainer CMongoContainer: The referenced Mongo CContainer::Container() collection will be used (be careful not to send the terms collection).

If any of the above conditions is not met, or if the node was not found, an exception will be raised.

Commit the object into a container.

Commit the object into a container.

This method should be used to commit the object to a container, the method accepts two parameters:

  • $theContainer: This parameter represents the index container, the value will be fed to the _ResolveIndexContainer() _ResolveIndexContainer protected method which should return a CMongoContainer CMongoContainer.
  • $theModifiers: This parameter represents the commit operation options, only the following options are considered:
    • kFLAG_PERSIST_REPLACE kFLAG_PERSIST_REPLACE: The provided object will be kFLAG_PERSIST_INSERT inserted, if the identifier doesn't match any container elements, or it will kFLAG_PERSIST_UPDATE replace the existing object. As with kFLAG_PERSIST_UPDATE update, it is assumed that the provided object's attributes will replace all the existing object's ones.
    • kFLAG_PERSIST_DELETE kFLAG_PERSIST_DELETE: This option assumes you want to remove the object from the container, although the container features a specific CContainer::Delete() method for this purpose, this option may be used to implement a deleted state, rather than actually removing the object from the container.

# _QueryNode( mixed $theNode, mixed $theContainer )

Query node.

Query node.

This method can be used to locate a node in the provided container, it accepts two parameters:

  • $theNode: The node reference:
    • integer or numeric string: The value refers to the node ID, in this case the node will have to be loaded from a Neo4j client, which must be provided in the next parameter.
    • CMongoQuery query: In this case we assume the value represents a CMongoQuery query that will be sent to the index, only the first matching record will be used.
  • $theContainer: The nodes container:
    • Neo4j client: The first parameter is interpreted as the node ID.
    • array: The parameter is expected to have two elements:
      • kTAG_NODE kTAG_NODE: The Neo4j client (graph container).
      • kTAG_TERM kTAG_TERM: The Mongo container (index container), which can take the following types:
        • MongoDB: The database will be used to instantiate a MongoCollection named as the kDEFAULT_CNT_NODES kDEFAULT_CNT_NODES tag.
        • MongoCollection: The provided collection will be used.
        • CMongoContainer CMongoContainer: The referenced Mongo CContainer::Container() collection will be used (be careful not to send the terms collection).

The method will return the located node or NULL if not found.

If the located node has an ID of zero, meaning that it is the default Neo4j node, the method will raise an exception, because most likely the node parameter was a string that resolved to zero.

If any of the provided parameters is incorrect, the method will raise an exception.

# _LocateNode( mixed $theNode, mixed $theContainer )

Locate node in graph.

Locate node in graph.

This method can be used to locate a node in the provided graph container, it accepts two parameters:

  • $theNode: The node reference, it can be either an integer or a numeric string referring to the seeked node ID.
  • $theContainer: The graph container, it must be a Neo4j client.

The method will return the located node or NULL if not found.

If the located node has an ID of zero, meaning that it is the default Neo4j node, the method will raise an exception, because most likely the node parameter was a string that resolved to zero.

If any of the provided parameters is incorrect, the method will raise an exception.
