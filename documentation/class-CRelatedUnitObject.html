Class CRelatedUnitObject

Related unit objects ancestor.

This class extends its CPersistentUnitObject parent to add two properties: a list of related objects and a pointer to a valid object.

This object introduces the concept of object reference, that is, a structure that can be used to refer to other objects. The Relate() Relate method can be used to manage a list of object references in the kTAG_REFS kTAG_REFS offset, this list can take two forms: it can be an array of object references, or an array of predicate/object pairs that can constitute a graph.

This class also features a kTAG_VALID property that can be Valid() used to refer to a valid object: in other words, objects do not get deleted, they simply point to the Valid() valid object, that way one can implement a system that maintains referential integrity.

To supplement the last property, this class implements a static ValidObject() method that can be used to return the valid object, objects that are obsolete or deleted may Valid() point to the valid object, this method will follow the links until it reaches an object that is valid.

We declare the class abstract because none of the CPersistentUnitObject parent abstract methods are here explicitly implemented.

# Relate( mixed $theObject, mixed $thePredicate = NULL, mixed $theOperation = NULL, boolean $getOld = FALSE )

Manage object references.

Manage object references.

This method can be used to manage relationships between the current object and other objects, it is represented as a list of subject/predicate/object relationships in which the subject is the current object, and the list of predicate/object pairs will be stored in the kTAG_REFS kTAG_REFS offset.

The method accepts the following parameters:

  • $theObject: Relation object, it should either be the relation object itself, or an object reference to the relation's object. Before the current object is Commit() committed, all elements provided as instances derived from the CRelatedUnitObject CRelatedUnitObject class will also be _ParseReferences() committed and converted to object references. This parameter is passed through a protected CPersistentUnitObject::NormaliseRelatedObject() method that derived classes can use to validate and normalise relation objects.
  • $thePredicate: Relation predicate, this parameter represents the kind or predicate of the relation, depending on whether it is provided or not, each element of the relations list will take the following form:
    • NULL: If the predicate is omitted, the element of the relation will only have the object parameter.
    • FALSE The element will be an array composed of one element, kTAG_DATA kTAG_DATA, which will hold the firat parameter.
    • other The element will be an array composed of two elements:
      • kTAG_KIND kTAG_KIND: This offset represents the reference predicate, it will hold the value of this parameter.
      • kTAG_DATA kTAG_DATA: This offset represents the reference object, it will hold the value of the first parameter.
      This parameter is passed through a protected CPersistentUnitObject::NormaliseRelatedPredicate() method that derived classes can use to validate and normalise relation predicates.
  • $theOperation: The operation to perform:
    • NULL: Return the element matched by the previous parameters.
    • FALSE: Delete the element matched by the previous parameters and return it.
    • other: Any other value means that we want to add to the list the element provided in the previous parameters, either appending it if there was no matching element, or by replacing a matching element. The method will return either the replaced element or the new one.
  • $getOld: Determines what the method will return when deleting or replacing:
    • TRUE: Return the deleted or replaced element.
    • FALSE: Return the replacing element or NULL when deleting.

Manage valid reference.

Manage valid reference.

This method can be used to handle the valid object's kTAG_LID identifier, it uses the standard accessor CAttribute::ManageOffset() method to manage the kTAG_VALID offset.

Objects derived from this class should be persistent, in other words, it is not an option to delete such objects: by creating a new object and referencing it from the old one, we maintain the original reference and point to the valid object.

# ValidObject( mixed $theContainer, mixed $theIdentifier, bitfield $theModifiers = kFLAG_DEFAULT )

Return a valid object.

Return a valid object.

This method can be used to instantiate a valid object, this means that if we provide the identifier of an expired or obsolete object that has a reference to a Valid() valid object, this method will return the valid one.

The method expects the same parameters as the NewObject() NewObject static method. If the Valid() valid chain is recursive, the method will raise an exception.

The method will return the first object that does not have a Valid() valid kTAG_VALID reference, or NULL if the object was not found.
