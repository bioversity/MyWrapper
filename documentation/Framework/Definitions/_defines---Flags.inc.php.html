kFLAG_MODIFIER_HASH = 0x20000000 (line 296)

Modifier: hash.

If the flag is set, the string will be hashed into a 32 character hex string; this option is mutually exclusive with the kFLAG_MODIFIER_HEX and kFLAG_MODIFIER_HEXEXP flags.

This modification will be applied after the eventual UTF-8, trim, NULL, lowercase and url or HTML conversions.

kFLAG_MODIFIER_HASH_BIN = 0x60000000 (line 312)

Modifier: binary hash.

If the flag is set, the string will be hashed and the resulting string will be a 16 character binary string. Note that if this flag is set it is assumed that kFLAG_MODIFIER_HASH is also set and this option is mutually exclusive with the kFLAG_MODIFIER_HEX and kFLAG_MODIFIER_HEXEXP flags.

This modification will be applied after the eventual UTF-8, trim, NULL, lowercase and url or HTML conversions.

kFLAG_MODIFIER_HEXEXP = 0x18000000 (line 282)

If the flag is set the string will be converted to a hexadecimal string suitable to be used in an expression (0x00...). Persist: Insert.

kFLAG_PERSIST_MASK = 0x000000F0 (line 150)

Persist mask.

This value masks the access flags that imply modification of the collection.

kFLAG_PERSIST_MODIFY = 0x00000060 (line 112)

Persist: Modify.

This bitfield value indicates that we intend to modify an existing object, the difference with the update operation is that while the latter will replace the whole object, in this case only the matching object attributes will be updated and the unmatched ones will be left untouched. This option applies when adding or modifying a subset of an object.

kFLAG_PERSIST_REPLACE = 0x00000030 (line 123)

Persist: Replace.

This bitfield value indicates that we intend to either insert an object, if it doesn't already exist, or update an existing object. In the latter case, this operation assumes that the provided object will replace the entire contents of the existing one.

kFLAG_PERSIST_UPDATE = 0x00000020 (line 101)

Persist: Update.

This bitfield value indicates that we intend to update an object in a container, this means that the object must exist, or the operation must fail. This operation assumes that the provided object will replace the entire contents of the existing one.

kFLAG_PERSIST_WRITE_MASK = 0x00000070 (line 143)

Write mask.

kFLAG_STATE_COMMITTED = 0x00000004 (line 62)

State: Committed.

This bitfield value indicates that the object has been either loaded from a persistent container, or that it has been saved to a persistent container. If used in combination with the dirty flag, it allows committing the object only if it was modified.

If the flag is off, this means that the object was not instantiated from a persistent container, or that it was not committed to it.

kFLAG_STATE_CUSTOM = 0x00000008 (line 69)

State: Custom.

This bitfield value indicates a custom state.

kFLAG_STATE_DIRTY = 0x00000002 (line 49)

State: Dirty.

This bitfield value indicates that the object has been modified. In general this state is only triggered by modifications to persistent data members; run time members should not be included. Methods that serialise or modify the contents of the object should respectively reset and set this flag.

If the flag is not set, this means that the object has not been modified;

kFLAG_STATE_INITED = 0x00000001 (line 37)

State: Initialised.

This bitfield value indicates that the object has been initialised, this means that all required data members are present. In general, this means that elements comprising the object's CObject::_Index() are not missing.

If this flag is not set, it means that the object lacks required elements, thus it will not work correctly, or it cannot be stored persistently.

kFLAG_STATE_MASK = 0x0000000F (line 80)

State mask.

This value masks all the state flags.
