Common virtual ancestor.

This class represents the ancestor of most entity mapped classes, it maps objects to an array by deriving from ArrayObject and it defines a couple of general purpose utility static methods.

The library uses the following conventions:

Naming standards

  • Classes: All class names in this library start with a capital (C), for instance CAnotherClass, followed by a name starting with another capital letter.
  • Public methods: All public method names should begin with a capital letter, for instance AnotherMethod().
  • Protected and private methods: All protected and private method names should begin with an underscore, for instance _AnotherMethod(). Another case in which names would start with an underscore is with core static methods.
  • Members: All members should start with lowercase (m), followed by a capital letter. For instance mMember.
  • Static members: All static members should begin with lowercase (s), so a static member could be CSomeClass::$sStaticMember.
  • Definitions: All definitions should start with a (k) and be followed by a code and ending with an uppercase name, for instance kTAG_DOMAIN would be the definition of a domain tag.
  • Method arguments: All method arguments should start with either the, for parameters holding miscellaneous values, such as $theVariable,, or with a lowercase verb, such as is, has or do, for parameters holding a flag value, such as isProtected.
  • Local variables: All local variables should be in lowercase, for instance $local_counter.

In general, abstract classes implement public interfaces which call a protected implementation. Usually the public methods should not be overridden, while derived classes may implement custom behaviours in the protected interface.

This class implements the following interfaces:

  • Offsets: In this class there cannot be an offset with a NULL value, the offset itself should be deleted in that case. Because of this we also override the inherited behaviour by suppressing notices and warnings when getting non-existant offsets.
  • JSON encoding: Derived classes will use JSON for web-services, so we provide two static methods to encode and decode JSON strings allowing for exceptions on errors.
  • String formatting: We provide a generalised static method to format strings which accepts a bitfield parameter that indicates which operation to perform, such as UTF8 encode, left and right trim, NULL handling, case insensitive conversion, URL, HTML and HEX encoding and hashing.
  • Time formatting: We provide a generalised static method to display duration strings.

static StringNormalise (line 435)

Normalise string.

This method can be used to format a string, the provided modifiers bitfield determines what manipulations are applied:

  • kFLAG_MODIFIER_UTF8: Convert the string to the UTF8 character set.
  • kFLAG_MODIFIER_LTRIM: Apply left trimming to the string.
  • kFLAG_MODIFIER_RTRIM: Apply right trimming to the string.
  • kFLAG_MODIFIER_TRIM: Apply both left and right trimming to the string.
  • kFLAG_MODIFIER_NULL: If this flag is set and the resulting string is empty, the method will return NULL.
    • kFLAG_MODIFIER_NULLSTR: If this flag is set and the resulting string is empty, the method will return the 'NULL' string; this option implies that the kFLAG_MODIFIER_NULL is also set.
  • kFLAG_MODIFIER_NOCASE: Set the string to lowercase, this is the default way to generate a case insensitive string.
  • kFLAG_MODIFIER_URL: URL-encode the string; note that this option and kFLAG_MODIFIER_HTML are mutually exclusive.
  • kFLAG_MODIFIER_HTML: HTML-encode the string; note that this option and kFLAG_MODIFIER_URL are mutually exclusive.
  • kFLAG_MODIFIER_HEX: Convert the string to hexadecimal; note that this option and hashing are mutually exclusive.
    • kFLAG_MODIFIER_HEXEXP: Convert the string to a hexadecimal expression; note that this option implies kFLAG_MODIFIER_HEX, and this option and hashing are mutually exclusive.
  • kFLAG_MODIFIER_HASH: If this bit is set the resulting string will be hashed using the md5 algorithm resulting in a 32 character hexadecimal string; this option is mutually exclusive with the kFLAG_MODIFIER_MASK_HEX option.
    • kFLAG_MODIFIER_HASH_BIN: If this bit is set, the resulting value should be a 16 character binary string; if the bit is OFF, the resulting value should be a 32 character hexadecimal string.

The order in which these modifications are applied are as stated.

  • see: kFLAG_MODIFIER_HASH, kFLAG_MODIFIER_HASH_BIN

_ManageOffset (line 620)

Manage an offset.

This library implements a standard interface for managing object properties using methods, this method extends the approach to offset members:

  • $theOffset: The offset to manage.
  • $theValue: The value or operation:
    • NULL: Return the offset's current value.
    • FALSE: Delete the offset.
    • other: Any other type represents the offset's new value.
  • $getOld: Determines what the method will return:
    • TRUE: Return the value of the offset before it was eventually modified.
    • FALSE: Return the value of the offset after it was eventually modified.

  • access: protected
