CArrayContainer

Description
Description | Vars | Methods (details)

Array persistent data store.

This class extends its ancestor to implement a concrete object store instance consisting of an array or ArrayObject. All other concrete instances of persistent object stores derive from this class, so that they might fall back onto a default working implementation.

_Commit (line 252)

Commit provided object.

We implement this method to handle array or ArrayObject stores and we ensure provided options are followed.

By default the object must be an array or ArrayObject, any other type will raise an exception.

The provided identifier will be cast to a string. If it is NULL, it means that the object is to be appended in the container and the method assumes the caller has determined that it is an insert operation.

Although the caller accepts the delete option, in this class we do not, so we shall raise an exception.

_PrepareStore (line 432)

Normalise before a store.

We overload this method to perform the following operations:

  • Call the parent method which will:
    • Ensure the identifier is provided if the operation is not an insert.
    • Ensure the method has the correct options.
    • Ensure the current object has a container.
  • Check for object in container if required.
  • Initialise identifier if required.

In derived classes you should handle your custom containers or delegate to the parent method.

In this class we do not check the identifier.

Any errors should raise an exception.

  • see: kERROR_OPTION_MISSING kERROR_INVALID_PARAMETER
