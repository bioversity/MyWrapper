Unit objects ancestor.

A unit object represents a main class, an object that should not be mixed with objects of other classes. In general, if an object is to refer to a unit object, it should use an object reference rather than embedding the object.

This class overloads its ancestor by adding two default offsets:

  • kTAG_CLASS: This offset will be automatically filled with the object's class name when committing, this is used to instantiate the correct class in mixed classes collections.
  • kTAG_VERSION: This offset will be automatically incremented each time the object is updated.

These offsets should be managed by this class and should be considered read-only attributes.

This class declares a static method that will make use of the class information to instantiate an object of the correct class.

Derived classes share a common identification workflow applied when saving a new object:

  • Object ID: If set among the object's offsets, this will be the value used to uniquely identify the object.
  • Object identifier: If the object ID was not explicitly set, a protected method, id(), will be used to get a string that represents the object's unique identifier. This string will be hashed and the resulting 16 character binary string will become the object's unique identifier.
  • Mongo default: If the previous method returns NULL, this is the indication that we want Mongo to assign a default identifier.

_id (line 177)

Return the object's unique identifier.

This method can be used to return a string value that represents the object's unique identifier, when saving the object for the first time, if this method returns a value, this will be hashed into a 16 character binary string and set as the object's ID.

If this method returns NULL, it is assumed that we want MongoDB to assign a default ID.

If the object already has an id offset, this method will not be used.

By default we let the system choose an identifier.

_PrepareStore (line 221)

Normalise parameters of a store.

We overload this method to perform the following actions:

  • Call parent method: The parent method will check if the container is of the correct type and set the object ID with the value provided in the identifier parameter if not NULL.
  • Manage ID: This class introduces the unique identifier concept. If the identifier parameter is empty, the method will check if the _id() method returns a value: in this case it will both set the identifier parameter and the object ID to the binary hash of that value.
  • Manage class: If not already set, this method will record the object's class among the offsets.
  • Manage version: If set, the method will increment its value, if not set, it will initialise it to zero. record the object's class among the offsets.
