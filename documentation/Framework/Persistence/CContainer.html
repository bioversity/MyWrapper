Persistent objects data store ancestor.

This abstract class is the ancestor of all persistent object data stores, it implements the common interfaces shared by all concrete instances of object data stores.

The class is designed to work with one object at the time, which means that it expects and returns one object, it does not handle object collections.

Persistence operations are performed through three methods:

  • Commit: This method will insert, replace or modify objects in the current object store.
  • Load: This method will retrieve objects from the current object store.
  • Delete: This method will remove objects from the current object store.

The class features a member that holds the native data store.

Commit (line 240)

Commit provided object.

This method can be used to commit the provided object to the current data store, it expects three parameters:

  • $theObject: The object to be committed.
  • $theIdentifier: This parameter is expected to be the object's unique identifier within the container, it will be the access key to the object once committed. The default value is NULL, this will generally be the case when inserting objects that expect their identifier to be computed by the container itself; in all other cases the parameter is required.
  • $theModifiers: This parameter represents the commit operation options, by default we assume it is a bitfield where the following values apply:
    • kFLAG_PERSIST_INSERT: The provided object will be inserted in the container, it is assumed that no other element in the container shares the same identifier, in that case the method must raise an exception.
    • kFLAG_PERSIST_UPDATE: The provided object will replace the existing object. In this case the method expects the container to have an entry with the same key as the provided identifier, if this is not the case the method must raise an exception. With this option it is assumed that the provided object's attributes will replace all the existing object's ones.
    • kFLAG_PERSIST_MODIFY: The provided object is assumed to contain a subset of an existing object's attributes, these provided attributes will be appended or replace the existing ones. In this case the method expects the container to have an entry with the same key as the provided identifier, if this is not the case the method must raise an exception.
    • kFLAG_PERSIST_REPLACE: The provided object will be inserted, if the identifier doesn't match any container elements, or it will replace the existing object. As with update, it is assumed that the provided object's attributes will replace all the existing object's ones.
    • kFLAG_PERSIST_DELETE: This option assumes you want to remove the object from the container, although this operation has its own public method, derived classes can use this option to implement a deleted state, rather than actually removing the object from the container.

The method will return the object's key within the container or raise an exception if the operation was not successful.

This method relies on a virtual method that will perform the actual operation, here we only perform the following controls:

  • Check identifier: by default we assume that all operations except insert require the identifier to be provided. If you handle default values for identifiers, you should overload this method.
  • Commit: we call the protected interface to perform the requested operation, this interface must be implemented by derived classes.

By default we assume the provided object to be either an array or an ArrayObject.

The actual operation will be performed by a protected method.

Container (line 149)

Manage persistent container.

This method can be used to manage the persistent container, it accepts a single parameter which represents either the container or the requested operation, depending on its value:

  • NULL: Return the current value.
  • other: Set the value with the provided parameter.

The second parameter is a boolean which if TRUE will return the old value when setting a new value; if FALSE, it will return the newly set value.

One is not allowed to delete the container.

In derived classes you should overload this method to check if the provided container is of the correct type.

Decode (line 472)

Decode provided data.

This method should be used on objects encoded with the Encode method, its duty is to restore the provided object to the state it was when provided to the Encode method.

The duty of this method is to scan the provided data for elements that need to be decoded, these will be in general of a type known to the current container, when decoded, these custom data types will be converted to an array structured as follows:

  • kTAG_TYPE: This offset represents the data type in which the data contained in the next offset is encoded.
  • kTAG_DATA: This offset represents the actual data, it may be a scalar or an array containing the subcomponents of the data.

The method accepts a single parameter which must be an array or ArrayObject instance, it will perform the conversion on the object itself. If you need to restore later the object to the state in which it was provided to this method, you should use the counter method Encode.

The method will traverse the provided object selecting only scalar elements which will be passed by reference to the protected method that will check the element and perform the conversion if necessary; structured elements will be recursed.

Encode (line 392)

Encode provided data.

This method can be used to encode the provided object into a format suitable for the current container, the method expects the provided data to be an object or data that will have to be stored in the current container.

The duty of this method is to scan the provided data for elements that need to be encoded to be compatible with the current container, these elements are expected to be either arrays or ArrayObjects and they should have have the following structure:

  • kTAG_TYPE: This offset represents the data type in which the data contained in the next offset is encoded.
  • kTAG_DATA: This offset represents the actual data, it may be a scalar or an array containing the subcomponents of the data.

The method accepts a single parameter which must be an array or ArrayObject instance, it will perform the encoding on the object itself. If you need to restore later the object to the state in which it was provided to this method, you should use the counter method Decode.

The method will traverse the provided object selecting only structures, if the structure contains both the type and data offsets, it will pass the corresponding object element by reference to the protected method that will take care of performing the conversion and replacing the provided element; if the element does not have the required offsets, the method will recurse on that element.

_Decode (line 704)

Decode provided data element.

This method should convert the provided scalar into a structure containing the type and the normalised data, and replace the provided reference with this structure.

This method is called by a public interface which traverses an object and provides this method with all scalar elements.

This method should select all data types that are considered custom and convert the element into an array in which the data type is set in the type offset, and the normalised data in the data offset.

For instance, a binary data object would be converted into an array where the type offset would be set as binary and the data offset would be set with the hexadecimal representstion of that data.

The conversion is performed on the provided element itself.
