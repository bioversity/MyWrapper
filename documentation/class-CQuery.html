Class CQuery

Query.

This class implements a query.

The main goal of this class is to provide a common framework and format to exchange data store queries or filters.

The query is an array structured as follows:

  • index: The index of the root element expresses a boolean condition which qualifies its content:
    • kOPERATOR_AND kOPERATOR_AND: AND.
    • kOPERATOR_NAND kOPERATOR_NAND: Not AND.
    • kOPERATOR_OR kOPERATOR_OR: OR.
    • kOPERATOR_NOR kOPERATOR_NOR: Not OR.
    The value of this root element is an array that can be of two types:
    • Query statement: A query statement defines a filter structured as follows:
      • kAPI_QUERY_SUBJECT kAPI_QUERY_SUBJECT: The subject field. It refers to the object element that we are filtering.
      • kAPI_QUERY_OPERATOR kAPI_QUERY_OPERATOR: The filter operator. This element is required and can take the following values:
        • kOPERATOR_DISABLED kOPERATOR_DISABLED: Disabled, it means that the filter is disabled.
        • kOPERATOR_EQUAL kOPERATOR_EQUAL: Equality (=).
        • kOPERATOR_EQUAL_NOT kOPERATOR_EQUAL_NOT: Inequality (!=), negates the kOPERATOR_EQUAL kOPERATOR_EQUAL operator.
        • kOPERATOR_LIKE kOPERATOR_LIKE: Like, it is an accent and case insensitive equality filter.
        • kOPERATOR_LIKE_NOT kOPERATOR_LIKE_NOT: The negation of the kOPERATOR_LIKE LIKE operator.
        • kOPERATOR_PREFIX kOPERATOR_PREFIX: Starts with, or prefix match.
        • kOPERATOR_CONTAINS kOPERATOR_CONTAINS: Contains, selects all elements that contain the match string.
        • kOPERATOR_SUFFIX kOPERATOR_SUFFIX: Ends with, or suffix match.
        • kOPERATOR_REGEX kOPERATOR_REGEX: Regular expression.
        • kOPERATOR_LESS kOPERATOR_LESS: Smaller than (\<).
        • kOPERATOR_LESS_EQUAL kOPERATOR_LESS_EQUAL: Smaller than or equal (\<=).
        • kOPERATOR_GREAT kOPERATOR_GREAT: Greater than (\>).
        • kOPERATOR_GREAT_EQUAL kOPERATOR_GREAT_EQUAL: Greater than or equal (\>=).
        • kOPERATOR_IRANGE kOPERATOR_IRANGE: Range inclusive, matches \>= value \<=.
        • kOPERATOR_ERANGE kOPERATOR_ERANGE: Range exclusive, matches \> value \<.
        • kOPERATOR_NULL kOPERATOR_NULL: Is NULL or element is missing.
        • kOPERATOR_NOT_NULL kOPERATOR_NOT_NULL:Not NULL or element exists.
        • kOPERATOR_IN kOPERATOR_IN: In, or belongs to set.
        • kOPERATOR_NI kOPERATOR_NI: Not in, the negation of kOPERATOR_IN kOPERATOR_IN.
        • kOPERATOR_ALL kOPERATOR_ALL: All, or match the full set.
        • kOPERATOR_NALL kOPERATOR_NALL: Not all, the negation of the kOPERATOR_ALL kOPERATOR_ALL operator.
        • kOPERATOR_EX kOPERATOR_EX: Expression, indicates a complex expression.
      • kAPI_QUERY_TYPE kAPI_QUERY_TYPE: The data type of the kAPI_QUERY_DATA kAPI_QUERY_DATA element:
        • kTYPE_STRING kTYPE_STRING: String, we assume in UTF8 character set.
        • kTYPE_INT32 kTYPE_INT32: 32 bit signed integer.
        • kTYPE_INT64 kTYPE_INT64: 64 bit signed integer.
        • kTYPE_FLOAT kTYPE_FLOAT: Floating point number.
        • kTYPE_DATE kTYPE_DATE: A date.
        • kTYPE_TIME kTYPE_TIME: A date and time.
        • kTYPE_STAMP kTYPE_STAMP: A native timestamp.
        • kTYPE_BOOLEAN kTYPE_BOOLEAN: An on/off switch.
        • kTYPE_BINARY kTYPE_BINARY: A binary string.
        • kTYPE_ENUM kTYPE_ENUM: An enumerated value.
        • kTYPE_ENUM_SET kTYPE_ENUM_SET: An enumerated set of values.
      • kAPI_QUERY_DATA kAPI_QUERY_DATA: The statement test data.
    • Nested query condition: A nested structure as the current one.

# AppendStatement( array $theStatement, string $theCondition = kOPERATOR_AND )

Append statement.

Append statement.

This method will append the provided statement to the query, the second parameter represents the condition.

Appended statements are merged at the condition level: if the condition exists at any level, the statement is appended to that condition; if the condition does not exist, it is created. Obviously, kOPERATOR_AND AND and kOPERATOR_NAND NAND are treated as equivalent, as well as kOPERATOR_OR OR and kOPERATOR_NOR NOR.

If you provide an object derived from CQuery CQuery as the statement, the method will ignore the condition parameter and append the provided query to the current object.

# _AppendStatement( reference & $theQuery, array $theCondition, string $theStatement )

Append statement.

Append statement.

This method will append the provided statement to the current query.

Statements of the same type, kOPERATOR_AND AND and kOPERATOR_NAND NAND, or kOPERATOR_OR OR and kOPERATOR_NOR NOR, will be added at the same level. If the top level is an kOPERATOR_OR OR or kOPERATOR_NOR NOR and the provided statement is an kOPERATOR_AND AND or kOPERATOR_NAND NAND, the latter will be promoted to the top level.

The parameters to this method are:

  • &$theQuery: Query receiving statement.
  • $theCondition: Statement condition.
  • $theCondition: Boolean statement condition code.
