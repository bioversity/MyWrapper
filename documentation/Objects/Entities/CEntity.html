Entity ancestor.

This class is the ancestor of entity classes in this library, entities are either institutions or individuals, this class implements the elements common to both:

  • kTAG_NAME: This offset represents the entity full name.
  • kTAG_CODE: The user code. This required property represents the user code. If empty, by default it will be initialised as the user's e-mail. Although no two users should share the same value, in this class we do not link the object ID to this property. The class features a member accessor method to manage this property.
  • kOFFSET_PASSWORD: This offset represents the user access password, it will be used in the authentication process. This property is not associated with any specific offset, but it is stored The class features a member accessor method to manage this property.
  • kTAG_NAME: This offset represents the user full name. The class features a member accessor method to manage this property.
  • kOFFSET_EMAIL: This offset represents the user e-mail. The class features a member accessor method to manage this property.

All the above attributes are required prior to committing an object: the object's inited status depends on having all of the above offsets set.

By default, the object's unique identifier will be provided by the system, whenever a new user is created, we shall use the identifier provided by MongoDB; this value should never be changed.

This class also features a static method that should return the default collection in which to store such objects.
