Manage terms.

This method can be used to add \hyperlink{class_c_ontology_term}{term} elements of the path, it accepts a single parameter which represents either the elements to be added or the operation to be performed\-:


\begin{DoxyItemize}
\item {\itshape N\-U\-L\-L}\-: This value indicates that we want to retrieve the whole list. 
\item {\itshape F\-A\-L\-S\-E}\-: This value indicates that we want to delete the whole list. 
\item {\itshape array}\-: The elements of the array will be added to the list by recursing this method. 
\item {\itshape other}\-: Any other type will be analysed as follows\-: 
\begin{DoxyItemize}
\item {\itshape \hyperlink{class_c_ontology_term}{C\-Ontology\-Term}}\-: The term's \hyperlink{}{identifier} will be used. 
\item {\itshape \hyperlink{class_c_ontology_edge}{C\-Ontology\-Edge}}\-: The edge elements will be handled as follows\-: 
\begin{DoxyItemize}
\item {\itshape The current list is empty}\-: The three terms will make the list. 
\item {\itshape The current list exists}\-: The method will check if the edge's subject matches the last element in the list\-: if this is the case, the method will add the predicate and object terms; if not, it will raise an exception. 
\end{DoxyItemize}
\item {\itshape \hyperlink{class_c_ontology_edge_index}{C\-Ontology\-Edge\-Index}}\-: The same treatment as for the \hyperlink{class_c_ontology_edge}{C\-Ontology\-Edge}. 
\item {\itshape other}\-: Any other type will be cast to a string and interpreted as the \hyperlink{class_c_ontology_term}{term}'s \hyperlink{}{identifier}. 
\end{DoxyItemize}
\end{DoxyItemize}

All elements will be appended to the list in the order they were provided, it must be noted that the class expects this list to be a sequence of term/predicate/term elements, this method is not responsible for checking this.

The method will prevent modifications if the object is \hyperlink{class_c_persistent_object_a6520a7bcecf3f39fd61ec6d08f736e77}{committed}, because this would change the object's unique \hyperlink{}{identifier}.

The method will return the current full list.
