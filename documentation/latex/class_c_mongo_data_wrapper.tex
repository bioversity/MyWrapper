This method is called by a static \hyperlink{class_c_mongo_data_wrapper_a0d37f7b47e1a1ac48846f6d10d08d846}{interface} which traverses an object and provides this method with all scalar elements.

This method should process all data types that cannot be fully represented in J\-S\-O\-N or those that need to be converted from a scalar to a structure, the method will replace the referenced element with an array in which the data type is set in the \hyperlink{}{type} offset, and the normalised data in the \hyperlink{}{data} offset.

Serialise provided object.

This method should be used on objects containing custom data types that cannot be converted to \hyperlink{class_c_object_a660f66b5be21d0cd1156463bcf75e575}{J\-S\-O\-N}, its duty is to convert special types into a format that can then be restored using the counter-\/method \hyperlink{class_c_mongo_data_wrapper_aff82f7c69fa3641daf93d1f375e33501}{Unserialise\-Object}.

The method will scan the provided data elements, array or Array\-Object elements will be recursed, scalar elements will be sent to another public \hyperlink{class_c_mongo_data_wrapper_a4a23d055b9735b8ba53102e619f94ffb}{method} that will take care of converting the data if necessary, the converted data will be an array structured as follows\-:


\begin{DoxyItemize}
\item {\itshape \hyperlink{}{k\-T\-A\-G\-\_\-\-T\-Y\-P\-E}\/}\-: This offset represents the data type in which the data contained in the next offset is encoded. 
\item {\itshape \hyperlink{}{k\-T\-A\-G\-\_\-\-D\-A\-T\-A}\/}\-: This offset represents the actual data, it may be a scalar or an array containing the subcomponents of the data. 
\end{DoxyItemize}

The method accepts a single parameter which must be an array or Array\-Object instance, it will perform the conversion on the object itself. If you need to restore later the object to the state in which it was provided to this method, you should use the counter method \hyperlink{class_c_mongo_data_wrapper_aff82f7c69fa3641daf93d1f375e33501}{Unserialise\-Object}.

Unserialise provided object.

This method should be used on objects converted using the \hyperlink{class_c_mongo_data_wrapper_a0d37f7b47e1a1ac48846f6d10d08d846}{Serialise\-Object} method, its duty is to convert the elements of the object that were converted to arrays back to the data type indicated in the \hyperlink{}{k\-T\-A\-G\-\_\-\-T\-Y\-P\-E} offset.

The method will scan the provided structure and select all elements which are either arrays or Array\-Objects having both the \hyperlink{}{k\-T\-A\-G\-\_\-\-T\-Y\-P\-E} and \hyperlink{}{k\-T\-A\-G\-\_\-\-D\-A\-T\-A} offsets, these elements will be sent to the \hyperlink{class_c_mongo_data_wrapper_a581d27ffad3ff0a68c5bd6123abece01}{Unserialise\-Data} method that will take care of converting the array back into the data type indicated in the \hyperlink{}{k\-T\-A\-G\-\_\-\-T\-Y\-P\-E} offset using the data in the \hyperlink{}{k\-T\-A\-G\-\_\-\-D\-A\-T\-A} offset.

The method accepts a single parameter which must be an array or Array\-Object instance, it will perform the encoding on the object itself. If you need to restore later the object to the state in which it was provided to this method, you should use the counter method \hyperlink{class_c_mongo_data_wrapper_a0d37f7b47e1a1ac48846f6d10d08d846}{Serialise\-Object}.

The \hyperlink{class_c_mongo_data_wrapper_a581d27ffad3ff0a68c5bd6123abece01}{Unserialise\-Data} method is not implemented in this class, if in derived classes you want to take advantage of this feature you must implement it.
