Commit provided object.

We implement this method to handle array or Array\-Object data stores and we ensure provided options are followed.

When \hyperlink{}{modifying} the contents of the object, we perform the following checks\-:


\begin{DoxyItemize}
\item {\itshape Both matched object and provided object are arrays}\-: this is the default scenario. If the provided object element is {\itshape N\-U\-L\-L}, the corresponding element in the matched object will be deleted; if not {\itshape N\-U\-L\-L}, the provided object element will replace the eventual existing one or be set in the matched object. 
\item {\itshape Matched object is array and provided object is scalar}\-: the provided object will be appended to the matched object. 
\item {\itshape Matched object is scalar and provided object is array}\-: we transform the matched object by appending the existing element to an empty array, and we set all the elements of the provided element into the newly created array. 
\item {\itshape Matched object and provided object are scalar}\-: we transform the matched object by appending the existing element to an empty array, and we append the provided object to it. 
\end{DoxyItemize}

By default the object must be an array or Array\-Object, any other type will raise an \hyperlink{}{exception}.

The provided identifier will be cast to a string. If it is {\itshape N\-U\-L\-L}, it means that the object is to be appended in the container and the method assumes the \hyperlink{class_c_container_a4847dc676d1f7704e75f8981e927508a}{caller} has determined that it is an \hyperlink{}{insert} operation.

Although the \hyperlink{class_c_container_a4847dc676d1f7704e75f8981e927508a}{caller} accepts the \hyperlink{}{delete} option, in this class we do not, so we shall raise an \hyperlink{}{exception}.
