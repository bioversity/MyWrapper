Class CPersistentObject

Persistent unit objects ancestor.

This class is the ancestor of all persistent objects in this library, it implements the common interfaces and workflow that persistent objects will use to be __construct() instantiated, __construct() retrieved or Commit() stored in object CContainer containers.

This class interacts closely with CContainer CContainer derived classes so that concrete classes derived from this one need only to be concerned with their specific functionality, while persistent issues are handled foth by this class and by the concrete classes derived from CContainer CContainer.

The class declares three main persistence operations whose workflow is as follows:

  • __construct() Create: The __construct() constructor can instantiate an object from data, as its CArrayObject ancestor, a protected _Create() method may be overloaded by derived classes to implement specific functionality.
  • __construct() Retrieve: The __construct() constructor is also used to retrieve objects stored in a CContainer container, this operation follows these steps:
    • _PrepareLoad() Prepare: This step is used to check parameters and prepare the resources needed to locate and retrieve the object.
    • _Load() Find: In this step the object will be searched for and retrieved from the CContainer container.
  • Commit() Store: This operation will store the object in a CContainer container, the followed steps are:
    • _PrepareCommit() Prepare: This step is used to check parameters and prepare the resources needed to save the object.
    • _Commit() Store: This operation will perform the actual storage, this step should be delegated to the CContainer container.

This class implements the default behaviour, in derived classes you should overload these methods if necessary, without changing the public interface.

This class extends its CStatusObject ancestor behaviour to record persistence status and accepts only instances derived from CContainer CContainer as containers.

# __construct( mixed $theContainer = NULL, mixed $theIdentifier = NULL, bitfield $theModifiers = kFLAG_DEFAULT )

Instantiate class.

Instantiate class.

The constructor can be used to instantiate an empty object, instantiate an object from some content, or load an object from a container.

The parameters to this method are:

  • $theContainer: This parameter represents either the contents of the object, if the next parameter is missing, or the CContainer container in which the object resides. If missing, we assume we want to create an empty object.
    • NULL: In this case it is assumed you want to instantiate an empty object and the next parameter will be ignored.
    • Array or ArrayObject: In this case we assume the parameter represents either the object contents, if the next parameter is NULL, or the CContainer container in which the object is Commit() stored, in which case we interpret the next parameter to be the object's identifier.
    • Other: Any other type will raise an kERROR_UNSUPPORTED exception, or it should be handled by derived classes.
  • $theIdentifier: This parameter represents the key or query that will be used to locate the object in the container provided in the first parameter.
    • NULL: This indicates that the first parameter represents the object contents.
    • other: Any other type is considered as a key or query used to locate the object in the container provided in the first parameter.
  • $theModifiers: This bitfield parameter can be used to provide a series of options to the object. The available options can be consulted in Flags.inc.php this file. Most of these options are managed automatically and should not be provided explicitly. This class handles one option that has the following meaning:
    • kFLAG_STATE_ENCODED kFLAG_STATE_ENCODED: If the option is on, the object will be passed through a CContainer::UnserialiseObject() conversion process that will convert all CDataType standard data types into native container custom data types CContainer::_PrepareCommit() before Commit() committing the object; the opposite CDataType::SerialiseObject() conversion process will take place CContainer::_FinishLoad() after __construct() loading the object from a container. It is a good idea to keep this flag ON, because the converted object will be compatible with any CContainer container and it will make data transfers from one database to the other easy. This process is automatic and transparent.

Depending on the results of this method we set the _IsCommitted() committed status kFLAG_STATE_COMMITTED flag as follows:

  • Empty object: By omitting the first parameter we indicate that we want to start from scratch, the object will not have its _IsCommitted() committed kFLAG_STATE_COMMITTED flag set.
  • Filled object: In this case we provide in the $theContainer parameter the object's contents. In this case we do not consider the object as persistent, so the object will not have its _IsCommitted() committed kFLAG_STATE_COMMITTED flag set.
  • Selected object: In this case we search the $theContainer with the provided $theIdentifier as a key or query, and if:
    • Found: We set the _IsCommitted() committed kFLAG_STATE_COMMITTED flag on.
    • Not found: We instantiate an empty object and ignore the _IsCommitted() committed kFLAG_STATE_COMMITTED flag.

The _IsInited() inited status kFLAG_STATE_INITED flag is not set by default, this will prevent the object from being Commit() stored and it is the responsibility of derived concrete instances to manage this status.

This method takes advantage of a protected interface which should be overloaded by derived classes instead of overloading this method:

  • Retrieve object: If the second parameter was provided, it implies that the object is to be retrieved from the first parameter which represents the CContainer container in which the object contents are stored.
    • _PrepareLoad() _PrepareLoad(): This method should check and normalise the container and identifier. In this class we ensure that the container is derived from CContainer CContainer.
    • _Load() _Load(): This method will delegate the CContainer container the responsibility of locating and retrieving the object using the provided identifier.
  • _Create() _Create(): This method should instantiate the object from the contents returned by _Load() _Load or by the data provided in the first parameter in the case the second parameter was omitted.

Derived classes should overload the above interface rather than overloading this method and should do so only if necessary, because this workflow should be sufficient for persisting objects.

# Commit( mixed $theContainer = NULL, mixed $theIdentifier = NULL, bitfield $theModifiers = kFLAG_PERSIST_REPLACE )

Commit the object into a container.

Commit the object into a container.

This method should be used to commit the object to a container, the method accepts two parameters:

  • $theContainer: This parameter represents the container in which the object is to be stored. By default we enforce concrete instances of CContainer CContainer, derived classes may overload the protected interface to initialise this value.
  • $theIdentifier: This parameter represents the value by which the current object will be uniquely identified in the first parameter. By default this value is required, derived classes may overload the protected interface to initialise this value.
  • $theModifiers: This parameter represents the commit operation options, by default we assume you want to kFLAG_PERSIST_REPLACE replace an object, but you may change this value if you want to perform specific commit operations:
    • kFLAG_PERSIST_INSERT kFLAG_PERSIST_INSERT: The provided object will be inserted in the container, it is assumed that no other element in the container shares the same identifier, in that case the container CContainer::Commit() method must raise an kERROR_DUPLICATE exception.
    • kFLAG_PERSIST_UPDATE kFLAG_PERSIST_UPDATE: The provided object will replace the existing object. In this case the method expects the container to have an entry with the same key as the provided identifier, if this is not the case the container CContainer::Commit() method must raise an kERROR_NOT_FOUND exception. With this option it is assumed that the provided object's attributes will replace all the existing object's ones.
    • kFLAG_PERSIST_MODIFY kFLAG_PERSIST_MODIFY: The provided object is assumed to contain a subset of an existing object's attributes, these provided attributes will be appended or replace the existing ones. In this case the method expects the container to have an entry with the same key as the provided identifier, if this is not the case the method must raise an kERROR_NOT_FOUND exception.
    • kFLAG_PERSIST_REPLACE kFLAG_PERSIST_REPLACE: The provided object will be kFLAG_PERSIST_INSERT inserted, if the identifier doesn't match any container elements, or it will kFLAG_PERSIST_UPDATE replace the existing object. As with kFLAG_PERSIST_UPDATE update, it is assumed that the provided object's attributes will replace all the existing object's ones.
    • kFLAG_PERSIST_DELETE kFLAG_PERSIST_DELETE: This option assumes you want to remove the object from the container, although the container features a specific CContainer::Delete() method for this purpose, this option may be used to implement a deleted state, rather than actually removing the object from the container.
    • kFLAG_STATE_ENCODED kFLAG_STATE_ENCODED: If the option is on, the object will be passed through a CContainer::UnserialiseObject() conversion process that will convert all CDataType standard data types into native container custom data types CContainer::_PrepareCommit() before committing the object. It is a good idea to keep this flag ON, because the converted object will be compatible with any CContainer container and it will make data transfers from one database to the other easy. This process is automatic and transparent.

The method will only operate if either the _IsCommitted() committed kFLAG_STATE_COMMITTED flag is not set, or the _IsDirty() dirty kFLAG_STATE_DIRTY flag is set, if none of these two conditions are satisfied, the method will do nothing.

The method will by default set the _IsCommitted() committed kFLAG_STATE_COMMITTED flag and reset the _IsDirty() dirty kFLAG_STATE_DIRTY flag if the operation was successful.

This method should raise an exception if any error occurs.

The method should return the unique identifier of the object or NULL if the operation was not performed.

In derived classes you should not overload this method, instead, you should overload the protected interface:

  • _PrepareCommit() _PrepareCommit(): This method can be used to initialise or check the parameters and resources.
  • _Commit() _Commit(): This method will perform the actual commit.
  • _FinishCommit() _FinishCommit(): This method can be used to perform eventual post-flight adjustments.

# _IsCommitted( mixed $theState = NULL )

Manage committed status.

Manage committed status.

This method can be used to get or set the object's committed state.

A committed object is one that has either been loaded from a container or committed to a container. This state indicates that the object is persistent. This state, combined with the _IsDirty() dirty status, can determine if an object needs to be committed in a container or not.

The method features a single parameter:

  • NULL: The method will return TRUE if the object is committed, or FALSE if the object is not committed.
  • TRUE: The method will set the object to committed.
  • FALSE: The method will reset the object's committed state.

In all cases the method will return the state after it was eventually modified.

# _IsEncoded( mixed $theState = NULL )

Manage encoded status.

Manage encoded status.

This method can be used to get or set the object's encoded state.

This flag determines whether custom data types are to be converted to CDataType standard types when working with the object. This is especially useful when transmitting the object through the network or when managing custom database types.

The conversion to CDataType standard types is CDataType::SerialiseObject() done when __construct() loading the object from a CContainer container and the opposite is CContainer::UnserialiseObject() done before Commit() storing the object into a CContainer container, these operations are performed by the CContainer container object that takes care of object persistence.

This kFLAG_STATE_ENCODED kFLAG_STATE_ENCODED flag can be provided at __construct() instantiation or directly to the persistence methods, although it is a better idea to provide it in the __construct() constructor to have this conversion happen transparently.

The method features a single parameter:

  • NULL: The method will return TRUE if the object is encoded, or FALSE if the object is not encoded.
  • TRUE: The method will set the object to encoded.
  • FALSE: The method will reset the object's encoded state.

In all cases the method will return the state after it was eventually modified.

Create object.

Create object.

The duty of this method is to instantiate an object with the provided data.

The method expects one parameter which is passed by reference:

# _Commit( reference & $theContainer, reference & $theIdentifier, reference & $theModifiers )

Store object in container.

Store object in container.

The duty of this method is to store the current object in the container provided in the first parameter with as key the value provided in the second parameter with options provided in the third parameter, please refer to Commit() this documentation for a reference of these parameters. Note that in this method all three parameters are passed by reference.

This class supports CContainer CContainer derived instances and will delegate the operation to the container. Note that by default we call the container's CContainer::Commit() commit method with the kFLAG_PERSIST_REPLACE kFLAG_PERSIST_REPLACE option, which will kFLAG_PERSIST_INSERT insert the object if new or kFLAG_PERSIST_UPDATE replace the eventual existing object.

The method expects all parameters to have been previously _PrepareCommit() checked, its main duty is to perform the actual storage. In derived classes you should intercept custom containers, or call the parent method.

There might be cases in which you do not want the commit operation to occur: in those cases you can clear all the persistence kFLAG_PERSIST_MASK flags in the modifier parameter, in that case the operation will not occur.

Note: the duty of this method is to store only the array part of the object, properties should be ignored.
